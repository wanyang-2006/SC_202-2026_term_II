# 第4章 互斥同步与通信

## 一、本章考试定位

本章是期末考试的核心应用章。根据 `2024操作系统复习范围.docx`，第4章重点为 `4.1、4.2、4.3`，并明确把“互斥算法”列为计算类内容，把“信号量机制、管程机制”列为应用题内容。

常见题型包括名词解释、简答题、判断题、概念辨析题、PV 操作应用题、管程应用题和并发程序分析题。考试中既可能问概念，也可能要求写同步互斥伪代码。

## 二、核心概念

并发进程是可以在同一时间间隔内同时推进的多个进程。并发提高了资源利用率，但也会导致执行顺序不可预测，从而产生与时间有关的错误。

前趋图是有向无环图，用来描述进程或语句之间的先后关系。图中节点表示一个程序段或进程，边表示前趋关系。

临界资源是一次只允许一个进程使用的资源。访问临界资源的程序段称为临界区。多个进程不能同时进入关于同一临界资源的临界区。

互斥是指多个进程不能同时进入同一临界区。互斥解决的是“不能同时做”的问题。

同步是指多个进程为了协调推进速度，在某些点相互等待和唤醒。同步解决的是“必须按某种先后顺序做”的问题。

信号量是 Dijkstra 提出的同步机制。它是一个特殊变量，只能通过 P 操作和 V 操作访问。P 操作通常表示申请资源或等待条件，V 操作通常表示释放资源或发出信号。

管程是一种高级同步机制，把共享变量、对共享变量的操作过程以及初始化语句封装在一个模块中。管程内部一次只允许一个进程进入，从结构上保证互斥。

进程通信是进程之间交换信息的机制。低级通信主要包括同步与互斥，高级通信包括共享内存、消息传递和信箱等方式。

## 三、重要原理与机制

并发程序产生错误的根本原因是多个进程对共享变量的访问发生了不正确的交叉。要保证结果可再现，必须控制进程进入临界区的顺序。

临界区互斥机制应满足三个基本要求：空闲让进、忙则等待、有限等待。有的教材还强调让权等待，即进程不能进入临界区时应释放处理机。

软件互斥算法通过设置标志变量来控制进入临界区，不依赖特殊硬件。它能体现互斥思想，但容易出现忙等、复杂且难以推广。

硬件互斥方法包括关中断、test-and-set、swap 等。硬件原子指令能保证检查和修改不可被打断，适合实现底层锁，但可能造成忙等。

信号量机制可以表示互斥，也可以表示同步。互斥信号量初值通常为 1，表示同一时刻只允许一个进程进入临界区；同步信号量初值通常为 0，表示某事件尚未发生。

PV 操作必须成对理解。P 操作在进入临界区或等待条件前执行，V 操作在离开临界区或条件满足后执行。P、V 顺序写错会造成死锁、违背同步顺序或互斥失效。

管程把共享数据和操作封装起来，避免 PV 操作分散在各进程中导致难以维护。管程一般配合条件变量实现等待和唤醒。

## 四、典型算法或计算方法

### 临界区互斥模板

算法思想：用一个互斥信号量保护临界资源。

```text
semaphore mutex = 1;

Process Pi:
while (true) {
    P(mutex);
    临界区;
    V(mutex);
    剩余区;
}
```

易错点：`P(mutex)` 必须在临界区前，`V(mutex)` 必须在临界区后；互斥信号量初值通常为 1。

### 前趋关系同步模板

若要求 S1 必须在 S2 前完成：

```text
semaphore s = 0;

P1:
S1;
V(s);

P2:
P(s);
S2;
```

算法思想：前趋动作完成后 V，后继动作开始前 P。

### 生产者-消费者问题模板

```text
semaphore mutex = 1;
semaphore empty = n;
semaphore full = 0;

Producer:
while (true) {
    生产一个产品;
    P(empty);
    P(mutex);
    放入缓冲区;
    V(mutex);
    V(full);
}

Consumer:
while (true) {
    P(full);
    P(mutex);
    取出产品;
    V(mutex);
    V(empty);
    消费产品;
}
```

易错点：对同一缓冲区的访问要用 `mutex` 互斥；`empty` 和 `full` 控制同步；一般先 P 资源信号量，再 P 互斥信号量，避免死锁。

### 读者-写者问题思想

多个读者可以同时读，写者必须互斥访问。常用一个计数器记录读者数量，用互斥信号量保护计数器，用写互斥信号量控制写者和第一/最后一个读者。

### 管程题解思想

1. 把共享数据定义在管程内部。
2. 把对共享数据的操作写成管程过程。
3. 用条件变量表示不能继续执行的条件。
4. 条件不满足时 wait，条件满足或状态改变后 signal。

## 五、易混淆知识点辨析

互斥与同步：互斥强调不能同时进入临界区；同步强调多个进程按一定先后关系推进。

临界资源与临界区：临界资源是被访问的资源；临界区是访问该资源的程序段。

忙等与阻塞等待：忙等进程持续占用 CPU 检查条件；阻塞等待进程放弃 CPU，等待被唤醒。

P 操作与 V 操作：P 通常表示申请、等待、减一；V 通常表示释放、通知、加一。

信号量与管程：信号量灵活但易错；管程封装性强，更易维护。

低级通信与高级通信：同步互斥只能传递简单控制信息；消息传递、共享内存等可以传递大量数据。

## 六、期末常考简答题

### 问题：什么是临界区？临界区管理应满足哪些原则？

答：
1. 临界区是进程中访问临界资源的程序段。
2. 空闲让进：临界区空闲时，应允许一个请求进入的进程进入。
3. 忙则等待：已有进程在临界区时，其他进程必须等待。
4. 有限等待：等待进入临界区的进程不能无限期等待。
5. 让权等待：不能进入临界区的进程应释放处理机，避免忙等浪费。

### 问题：什么是进程互斥？为什么需要互斥？

答：
1. 进程互斥是指多个进程不能同时进入关于同一临界资源的临界区。
2. 并发进程可能同时访问共享变量或文件。
3. 若访问顺序不受控制，会产生与时间有关的错误。
4. 互斥保证共享资源一次只被一个进程访问，使结果可再现。

### 问题：什么是进程同步？

答：
1. 进程同步是指一组并发进程为协调推进速度，在某些点相互等待和唤醒。
2. 同步用于保证多个进程按正确的先后关系执行。
3. 例如生产者必须先放入产品，消费者才能取出产品。
4. 信号量、管程和消息传递都可以实现同步。

### 问题：信号量机制如何实现互斥和同步？

答：
1. 信号量是只能通过 P、V 操作访问的特殊变量。
2. 实现互斥时，设互斥信号量初值为 1，进入临界区前 P，离开临界区后 V。
3. 实现同步时，设同步信号量初值通常为 0，前趋进程完成后 V，后继进程执行前 P。
4. P、V 操作必须是原子操作，否则会破坏同步互斥效果。

### 问题：管程相对信号量有什么优点？

答：
1. 管程把共享变量和访问过程集中封装，结构清晰。
2. 管程内部自动保证互斥，一次只允许一个进程进入。
3. 条件变量可用于表达等待条件和唤醒关系。
4. 相比 PV 操作分散在各进程中，管程更易于编写、检查和维护。

## 七、典型题解题模板

题型识别方法：看到“多个进程共享缓冲区、读者写者、哲学家、司机售票员、前趋关系、PV 操作、管程”等关键词，就是同步互斥应用题。

解题步骤：
1. 找共享资源，确定是否需要互斥信号量。
2. 找先后关系，确定同步信号量及初值。
3. 先写信号量定义和初值。
4. 再分别写各进程伪代码。
5. 检查 P/V 是否成对，是否可能死锁，是否满足题目并发要求。

常见错误：
1. 互斥信号量初值写成 0。
2. 同步信号量初值随意设置。
3. P/V 顺序颠倒，导致死锁。
4. 忘记保护共享计数器。
5. 把互斥问题写成同步问题，或把同步问题只写一个 mutex。
