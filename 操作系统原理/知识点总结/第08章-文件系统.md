# 第8章 文件系统

## 一、本章考试定位

本章是期末简答题和计算题常考章。根据 `2024操作系统复习范围.docx`，第8章重点为 `8.1、8.3、8.7、8.8、8.9`。历年试题中出现过文件系统、文件目录、目录文件、文件顺序存取与随机存取、多级索引 i-node 最大文件长度和实际占用空间计算。

常见题型包括名词解释、简答题、选择判断、概念辨析题和文件索引结构计算题。

## 二、核心概念

文件是具有符号名、在逻辑上具有完整意义的信息项序列。文件系统是操作系统中负责文件存储、组织、访问、共享和保护的程序集合。

文件逻辑结构是用户看到的文件组织形式，如记录式文件和流式文件。记录式文件由记录组成，流式文件由字节流组成。

文件物理结构是文件在磁盘块上的存放方式，常见有顺序结构、链接结构和索引结构。它决定文件访问效率、空间开销和动态增长能力。

文件目录是文件名和文件控制信息的集合，用于实现按名存取。目录项通常包含文件名和指向文件控制块或 i-node 的信息。

FCB 是文件控制块，是文件存在的标志，保存文件名、文件类型、位置、大小、权限、时间等管理信息。

i-node 是类 UNIX 文件系统中记录文件属性和数据块地址的数据结构。目录项通常保存文件名和 i-node 号，真正的文件信息存放在 i-node 中。

打开文件表用于管理已打开文件。系统打开文件表全系统共享，保存 FCB 主部、共享计数、读写状态等；用户打开文件表每个进程一张，保存打开方式、读写指针和系统表入口。

日志结构文件系统通过先记录日志或把更新顺序写入磁盘，提高崩溃恢复能力和写性能。

## 三、重要原理与机制

文件系统要解决的问题包括：用户如何按名访问文件、文件如何组织在外存上、多个进程如何共享文件、如何保护文件安全、如何提高文件系统效率。

顺序结构要求文件占用连续磁盘块。它访问速度快，支持顺序和直接访问，但文件动态增长困难，容易产生外部碎片。

链接结构允许文件块分散存放，每块保存下一块指针。它适合文件动态增长，空间利用较好，但随机访问速度慢。

索引结构为文件建立索引块，索引块中保存数据块地址。它既支持顺序访问，也支持随机访问，适合大文件，但需要额外索引空间。

文件共享可通过目录项共享、链接和 i-node 等机制实现。硬链接通常多个目录项指向同一 i-node；符号链接保存被链接文件路径。

文件保护关注防止非授权访问，常见方法包括访问控制矩阵、访问控制表、用户权限位和口令。文件安全关注防止数据丢失，常见方法是备份和恢复。

打开文件机制的目的是减少反复查目录开销。文件打开后，系统在打开文件表中保存文件控制信息和读写指针，后续读写可直接使用表项。

## 四、典型算法或计算方法

### 多级索引最大文件长度

题型识别：题目给出 i-node 中直接索引、一次间接、二次间接、三次间接项数，数据块大小和地址项大小，要求最大文件长度。

解题步骤：
1. 计算一个索引块能存放的地址数：`地址数 = 数据块大小 / 每个磁盘地址大小`。
2. 直接索引可表示的数据块数 = 直接索引项数。
3. 一次间接可表示的数据块数 = 地址数。
4. 二次间接可表示的数据块数 = 地址数平方。
5. 三次间接可表示的数据块数 = 地址数三次方。
6. 最大文件长度 = 可表示总数据块数 * 数据块大小。

### 文件实际占用空间计算

解题步骤：
1. 先计算文件数据需要的数据块数，向上取整。
2. 判断需要使用哪些直接索引和间接索引。
3. 计算额外占用的索引块数量。
4. 实际占用空间 = 数据块空间 + 索引块空间。
5. 若题目说明“不包括 i-node 块”，则不要把 i-node 本身算入。

### 文件访问方式判断

顺序访问适合磁带、日志文件和连续读写。随机访问适合数据库、索引文件和需要按位置访问的文件。

## 五、易混淆知识点辨析

文件逻辑结构与物理结构：逻辑结构是用户看到的记录或字节组织；物理结构是文件块在磁盘上的存放方式。

顺序访问与顺序结构：顺序访问是一种访问方式；顺序结构是一种磁盘存放方式。

FCB 与目录项：FCB 保存文件控制信息；目录项通常包含文件名和指向 FCB 或 i-node 的指针。

目录文件与普通文件：目录文件内容是目录项集合，用于管理文件名和文件控制信息；普通文件内容是用户数据或程序。

硬链接与符号链接：硬链接共享同一 i-node；符号链接是一个保存路径名的特殊文件。

缓存与缓冲：缓存强调保存常用数据以提高命中率；缓冲强调协调速度差异和批量传输。

## 六、期末常考简答题

### 问题：什么是文件系统？

答：
1. 文件系统是操作系统中负责管理文件的程序和数据结构集合。
2. 它为用户提供按名存取文件的能力。
3. 它管理文件的逻辑结构、物理存放、目录、共享、保护和安全。
4. 它把外存空间组织成用户可理解和可操作的文件。

### 问题：常见的文件物理结构有哪些？各有什么特点？

答：
1. 顺序结构：文件占用连续磁盘块，访问速度快，支持随机访问，但增长困难。
2. 链接结构：文件块可离散存放，空间利用率高，增长方便，但随机访问慢。
3. 索引结构：用索引块记录数据块地址，既支持顺序访问也支持随机访问，但有索引空间开销。

### 问题：文件目录的作用是什么？

答：
1. 文件目录保存文件名与文件控制信息之间的对应关系。
2. 它支持用户按文件名查找文件。
3. 它组织文件之间的层次关系，方便分类管理。
4. 它还可保存或间接指向文件权限、位置和大小等信息。

### 问题：打开文件表有什么作用？

答：
1. 打开文件表记录已经打开文件的相关信息。
2. 它避免每次读写文件都重新查找目录。
3. 系统打开文件表保存全局文件信息和共享计数。
4. 用户打开文件表保存每个进程的打开方式、读写指针和系统表入口。

### 问题：文件顺序存取和随机存取有什么区别？

答：
1. 顺序存取按文件中信息的先后顺序依次访问。
2. 随机存取可以根据记录号、字节偏移或关键字直接访问指定位置。
3. 顺序存取适合日志、磁带和顺序处理文件。
4. 随机存取适合数据库、索引文件和需要频繁定位的文件。

## 七、典型题解题模板

题型识别方法：看到“i-node、直接索引、一次间接、二次间接、三次间接、数据块大小、磁盘地址大小、最大文件长度、实际占用空间”，就是文件索引计算题。

解题步骤：
1. 统一单位，把 KB、MB、GB 转成字节或块数。
2. 计算每个索引块可存地址项数。
3. 分别计算直接、一次间接、二次间接、三次间接覆盖的数据块数。
4. 求最大文件长度时，只算可寻址数据块容量。
5. 求实际占用空间时，额外加上用到的索引块。

常见错误：
1. 忘记文件大小除以块大小要向上取整。
2. 把索引块也当成数据块内容。
3. 题目说不包括 i-node，却把 i-node 块算入。
4. 一次、二次、三次间接索引覆盖块数计算混淆。
5. 单位换算错误。
