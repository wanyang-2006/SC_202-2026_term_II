# 第6章 存储管理

## 一、本章考试定位

本章是期末考试的重点章，既考概念，也考地址转换计算。根据 `2024操作系统复习范围.docx`，第6章重点为 `6.1--6.5`，并明确列出“地址定位”为计算题。历年试题中出现过地址映射、段式地址变换、分页与分段区别、静态页面管理等内容。

常见题型包括名词解释、选择判断、简答题、概念辨析题和地址转换计算题。

## 二、核心概念

存储管理是操作系统对内存资源进行分配、回收、共享、保护和扩充的功能。它的目标是提高内存利用率，支持多道程序运行，并保护进程之间互不干扰。

逻辑地址是程序中使用的地址，也称相对地址、虚拟地址或程序地址。物理地址是内存中实际存储单元的地址，也称绝对地址或实地址。

地址映射是把逻辑地址转换为物理地址的过程。它通常需要硬件支持，如基址寄存器、限长寄存器、页表、段表和快表。

重定位是将程序中的逻辑地址转换为实际物理地址的过程。静态重定位在程序装入时完成，动态重定位在程序运行时完成。

内部碎片是已分配分区内部未被使用的空间。外部碎片是已分配分区之间存在的、难以利用的小空闲区。

页式存储管理把进程逻辑空间划分为固定大小的页，把内存划分为同样大小的页框。页和页框大小相等，通过页表完成地址转换。

段式存储管理按程序的逻辑结构划分为若干段，每段长度可不同。段式管理便于共享和保护，但可能产生外部碎片。

段页式存储管理先把进程划分为段，再把每段划分为页。它结合了段式便于共享保护和页式便于内存分配的优点。

## 三、重要原理与机制

存储管理需要解决的问题包括：为进程分配内存、回收内存、支持地址转换、保护进程空间、实现共享和扩充内存容量。

内存分区可按分区时刻分为静态分区和动态分区，也可按大小分为等长分区和异长分区。静态等长分区适合页式管理，动态异长分区适合界地址或段式管理。

动态分区分配算法包括首次适应、最佳适应和最坏适应。首次适应从低地址开始寻找第一个足够大的空闲区；最佳适应选择最小但足够大的空闲区；最坏适应选择最大的空闲区。

紧凑是处理外部碎片的一种方法，通过移动已占用区域，把分散空闲区合并为一个大空闲区。它能提高可用空间连续性，但移动开销很大。

页式管理解决连续内存分配困难的问题。由于页框固定大小，外部碎片基本消除，但可能产生最后一页未用满的内部碎片。

段式管理符合用户程序的逻辑结构。每段可以独立保护、共享和动态增长，但段长度不同，分配时容易产生外部碎片。

快表是页表或段表项的高速缓存，用于加快地址转换。若快表命中，可快速得到页框号或段基址；若未命中，需要访问内存中的页表或段表。

## 四、典型算法或计算方法

### 动态分区分配

首次适应：
1. 空闲分区按地址递增排列。
2. 从头查找第一个大小满足请求的空闲区。
3. 分配后剩余部分仍作为空闲区。

最佳适应：
1. 查找所有满足请求的空闲区。
2. 选择大小最接近请求的空闲区。
3. 易产生许多很小的外部碎片。

最坏适应：
1. 查找最大空闲区。
2. 从最大空闲区中分割出所需空间。
3. 保留较大的剩余空闲区，但大空闲区消耗快。

### 页式地址转换

题型识别：给出页大小、逻辑地址、页表，要求求物理地址。

解题步骤：
1. 页号 `p = 逻辑地址 / 页大小`，页内位移 `d = 逻辑地址 % 页大小`。
2. 查页表得到页框号 `f`。
3. 物理地址 = `f * 页大小 + d`。
4. 若页号越界或页表项无效，则产生地址越界或缺页异常。

### 段式地址转换

题型识别：逻辑地址写成 `[段号, 段内地址]`，并给出段表。

解题步骤：
1. 查段表，找到段号对应的段长和段基址。
2. 判断段内地址是否小于段长。
3. 若合法，物理地址 = 段基址 + 段内地址。
4. 若段号不存在或段内地址越界，产生越界中断。

### 段页式地址转换

解题步骤：
1. 逻辑地址分为段号、段内页号和页内位移。
2. 先查段表，找到该段页表地址和段长信息。
3. 再查页表，找到页框号。
4. 物理地址 = 页框号 * 页大小 + 页内位移。

## 五、易混淆知识点辨析

逻辑地址、虚拟地址、物理地址：逻辑地址和虚拟地址通常指程序看到的地址；物理地址是内存实际地址。

分页与分段：分页是固定大小，面向系统管理；分段是长度可变，面向程序逻辑结构。

页与页框：页属于进程逻辑空间，页框属于物理内存空间，二者大小相同。

内部碎片与外部碎片：内部碎片在已分配区域内部；外部碎片在已分配区域之间。

静态重定位与动态重定位：静态重定位在装入时完成，程序运行时地址不再改变；动态重定位在运行时完成，便于移动和换入换出。

页表与段表：页表记录页号到页框号的映射；段表记录段号到段基址、段长和保护信息的映射。

## 六、期末常考简答题

### 问题：存储管理的主要功能有哪些？

答：
1. 存储分配和回收：为进程分配内存并在进程结束后回收。
2. 地址映射：把逻辑地址转换为物理地址。
3. 存储保护：防止进程越界访问或非法操作。
4. 存储共享：允许多个进程共享代码或数据。
5. 存储扩充：通过内外存结合形成虚拟存储体系。

### 问题：什么是地址映射？

答：
1. 地址映射是把程序中的逻辑地址转换为内存物理地址的过程。
2. 程序编译后通常使用逻辑地址，不能直接确定实际内存位置。
3. 操作系统和硬件通过基址寄存器、页表或段表完成转换。
4. 地址映射还能配合越界检查和访问权限检查实现存储保护。

### 问题：分页存储管理的基本思想是什么？

答：
1. 把进程逻辑空间划分为大小相等的页。
2. 把物理内存划分为同样大小的页框。
3. 进程的各页可装入不连续的页框中。
4. 通过页表记录页号到页框号的映射。
5. 分页减少外部碎片，但可能产生内部碎片。

### 问题：分段存储管理的基本思想是什么？

答：
1. 按程序逻辑结构把进程划分为若干段，如代码段、数据段、栈段。
2. 每段有独立的段名或段号，长度可以不同。
3. 通过段表记录段基址、段长和保护信息。
4. 分段便于共享和保护，但容易产生外部碎片。

### 问题：分页和分段有什么区别？

答：
1. 页大小固定，段长度可变。
2. 分页面向系统内存管理，分段面向用户程序逻辑结构。
3. 分页地址是一维逻辑地址划分页号和页内位移，分段地址通常是段号和段内地址。
4. 分页容易产生内部碎片，分段容易产生外部碎片。
5. 分段更便于共享和保护，分页更便于离散分配内存。

## 七、典型题解题模板

题型识别方法：题目出现“逻辑地址、页号、页内位移、页表、段表、段长、基址、地址变换”等关键词，就是地址转换题。

分页题步骤：
1. 明确页大小。
2. 逻辑地址除以页大小得到页号和页内位移。
3. 查页表得到页框号。
4. 计算物理地址。
5. 检查页号和页表项是否合法。

分段题步骤：
1. 根据段号查段表。
2. 比较段内地址和段长。
3. 合法则段基址加段内地址。
4. 不合法则写明越界中断或地址错误。

常见错误：
1. 把页号和页内位移写反。
2. 段式地址转换忘记检查段长。
3. 把页大小单位 KB 和字节混淆。
4. 把页号直接当作页框号。
5. 分页和分段的碎片类型混淆。
