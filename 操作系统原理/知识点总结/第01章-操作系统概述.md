# 第1章 操作系统概述

## 一、本章考试定位

本章是期末考试的基础概念章，常用于单选、判断、名词解释和简答题。根据 `2024操作系统复习范围.docx`，第1章重点落在 `1.4` 操作系统类型和 `1.5` 操作系统运行环境；历年试题与试题解析中也出现过操作系统定义、接口形式、SPOOLing、系统调用、中断环境等知识点。

本章一般不作为大型计算题出现，但会为后续“中断、进程、存储保护、I/O 管理”等章节提供概念基础。

## 二、核心概念

操作系统是位于硬件层之上、其他软件层之下的系统软件。它负责管理计算机系统中的硬件和软件资源，合理组织程序运行，并向用户和应用程序提供方便的接口。

操作系统的两个基本作用：一是资源管理者，管理处理机、存储器、设备、文件和作业；二是用户与计算机之间的接口，使用户不必直接面对复杂硬件。

并发是指多个程序在同一时间间隔内都在向前推进，宏观上同时进行，微观上可能交替执行。并行是指多个程序在同一时刻真正同时执行，通常需要多个处理器。单处理机系统可以实现并发，但不能实现真正的并行。

共享是指系统资源可供多个并发进程共同使用。共享分为互斥共享和同时共享。打印机这类独占设备通常需要互斥共享；可重入代码、只读文件等可以被多个进程同时共享。

虚拟是指通过某种技术把一个物理实体变成多个逻辑实体，或把有限资源表现为更大的逻辑资源。虚拟存储器、虚拟设备和虚拟机都是典型例子。

异步是指并发程序的执行以不可预知的速度向前推进，程序何时获得 CPU、何时等待 I/O，通常无法由用户精确控制。

系统调用是操作系统提供给程序员的接口。用户程序不能直接执行特权指令，需要通过系统调用请求操作系统完成打开文件、创建进程、申请设备等服务。

用户接口包括命令接口、图形用户界面、作业控制语言和系统调用接口。命令接口和图形界面主要面向普通用户，系统调用主要面向程序员。

## 三、重要原理与机制

操作系统需要引入，是因为裸机难以直接使用，硬件资源也难以被多个程序公平、高效、安全地共享。操作系统通过抽象和管理，把复杂硬件包装成文件、进程、地址空间、设备等更易使用的对象。

多道批处理系统通过在内存中同时存放多个作业，使 CPU 和 I/O 设备交替忙碌，提高资源利用率。缺点是交互性差，用户不能及时干预作业运行。

分时系统把 CPU 时间划分为很短的时间片，多个终端用户轮流获得系统服务。它强调多路性、交互性、独占性和及时性，适合程序调试和人机交互。

实时系统要求系统在规定时间内响应外部事件。它强调及时性和可靠性，资源利用率不是首要目标。实时控制、航班订票、工业控制都属于典型场景。

SPOOLing 是假脱机技术，利用磁盘上的输入井和输出井，把独占的慢速设备改造成逻辑上的共享设备。它解决 CPU 与慢速 I/O 设备速度不匹配的问题，提高设备利用率。

操作系统运行环境包括定时装置、系统栈、特权指令与非特权指令、处理机状态、地址映射机构、存储保护设施和中断系统。这些硬件支持保证操作系统能控制资源并保护自身。

处理机状态通常分为管态和目态。操作系统内核运行在管态，可以执行特权指令；普通用户程序运行在目态，只能执行非特权指令。用户程序需要操作系统服务时，通过访管指令或系统调用进入内核。

## 四、典型算法或计算方法

本章通常没有独立大型计算题，但可能考 SPOOLing 工作过程或操作系统类型判断。

SPOOLing 题解思路：

1. 先判断题目是否涉及独占设备、慢速外设、输入井或输出井。
2. 写出组成：输入井、输出井、预输入进程、缓输出进程、井管理程序或作业调度相关程序。
3. 写出作用：提高 I/O 速度，把独占设备改造为共享设备，实现虚拟设备。
4. 易错点：SPOOLing 不等于简单缓冲；它通常以磁盘为后援存储器，并有专门进程管理输入输出。

## 五、易混淆知识点辨析

并发与并行：并发强调同一时间间隔内多个活动推进；并行强调同一时刻多个活动同时执行。

系统调用与库函数：系统调用是操作系统内核提供的服务入口；库函数是高级语言库提供的函数，可能封装系统调用，也可能完全在用户态完成。

命令接口与系统调用：命令接口给终端用户使用，如 shell 命令；系统调用给程序使用，如 `open`、`read`、`fork`。

分时系统与实时系统：分时系统强调交互和公平响应；实时系统强调在限定时间内完成任务。

SPOOLing 与缓冲：缓冲主要解决速度匹配；SPOOLing 不仅缓冲，还用磁盘和专门进程把独占设备虚拟成共享设备。

## 六、期末常考简答题

### 问题：什么是操作系统？它有哪些基本功能？

答：
1. 操作系统是控制和管理计算机系统硬件与软件资源、合理组织程序运行、方便用户使用计算机的系统软件。
2. 它的基本功能包括处理机管理、存储管理、设备管理、文件管理和作业管理。
3. 它向用户提供命令接口或图形界面，向程序员提供系统调用接口。
4. 它通过资源抽象和资源分配，提高系统效率并保护系统安全。

### 问题：操作系统有哪些基本特征？

答：
1. 并发性：多个程序在同一时间间隔内都向前推进。
2. 共享性：系统资源可被多个并发进程共同使用。
3. 虚拟性：通过软件技术把物理资源抽象成多个逻辑资源。
4. 异步性：并发程序以不可预知的速度推进，执行顺序不固定。

### 问题：分时系统和实时系统有什么区别？

答：
1. 分时系统主要面向多个交互用户，强调多路性、交互性和及时响应。
2. 实时系统主要面向外部事件控制，强调及时性和可靠性。
3. 分时系统响应时间通常以秒级衡量，实时系统可能要求毫秒级或微秒级响应。
4. 分时系统更重视用户公平使用，实时系统更重视任务是否在截止时间内完成。

### 问题：什么是 SPOOLing 技术？它有什么作用？

答：
1. SPOOLing 即假脱机技术，是利用磁盘作为后援存储器来模拟脱机输入输出的技术。
2. 它通常由输入井、输出井、预输入进程和缓输出进程组成。
3. 它可以缓和 CPU 与慢速 I/O 设备之间的速度矛盾。
4. 它能把独占设备改造为逻辑上的共享设备，提高设备利用率。

### 问题：操作系统提供哪些用户接口？

答：
1. 命令接口，供终端用户通过命令使用系统。
2. 图形用户界面，供普通用户以窗口、图标、菜单等方式使用系统。
3. 作业控制语言，供批处理作业说明作业执行要求。
4. 系统调用接口，供程序员在程序中请求操作系统服务。

## 七、典型题解题模板

题型识别方法：看到“操作系统特征、类型、接口、管态目态、SPOOLing、系统调用”等关键词，一般属于本章概念题或简答题。

解题步骤：
1. 先给定义，不要只举例。
2. 再写组成或特点。
3. 最后写作用、优缺点或适用场景。

常见错误：
1. 把并发写成同一时刻同时执行。
2. 把库函数等同于系统调用。
3. 把 SPOOLing 简化为普通缓冲。
4. 忘记操作系统的管理功能包括处理机、存储、设备、文件和作业。
