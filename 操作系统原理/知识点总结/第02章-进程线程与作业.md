# 第2章 进程线程与作业

## 一、本章考试定位

本章是操作系统期末的高频基础章。常见题型包括单选、判断、名词解释、简答题和概念辨析题。根据 `2024操作系统复习范围.docx`，第2章重点为 `2.1、2.2、2.3`，即多道程序设计、进程和线程。

历年试题中出现过 Process、Thread、PCB、进程与线程区别、作业调度与进程调度区别、进程状态转换等内容。本章本身计算题不多，但会与第3章处理机调度、第4章同步互斥、第5章死锁结合成综合题。

## 二、核心概念

多道程序设计是指在内存中同时存放多道相互独立的程序，使它们在操作系统管理下交替运行。它的目标是提高 CPU、内存和外设的利用率。

程序是静态的指令集合，通常存放在外存中。进程是程序在某个数据集合上的一次运行活动，是动态概念。一个程序可以对应多个进程，一个进程至少包含程序、数据和进程控制块。

进程是资源分配和独立运行的基本单位。它具有动态性、并发性、独立性、异步性和结构性。进程的存在由 PCB 标志，没有 PCB 就没有进程。

PCB 是进程控制块，是操作系统为管理进程而设置的数据结构。它保存进程标识符、进程状态、程序计数器、寄存器内容、调度信息、资源清单、打开文件等信息。

进程状态通常包括运行态、就绪态和阻塞态。运行态表示进程正在占用 CPU；就绪态表示进程已具备运行条件，只等待 CPU；阻塞态表示进程正在等待某事件或资源，暂时不能运行。

线程是进程中的一个执行流，是处理机调度的基本单位。线程共享所属进程的地址空间和资源，但拥有自己的程序计数器、寄存器和栈。

作业是用户要求计算机完成的一项完整任务，是从用户角度看的宏观执行单位。进程是从操作系统角度看的微观执行单位。

## 三、重要原理与机制

引入进程是为了解决多道程序并发执行时的描述、管理和调度问题。程序本身是静态的，不能反映并发执行过程中的状态变化，因此操作系统用进程表示正在运行的程序。

进程状态转换由操作系统完成。就绪到运行通常由调度程序选择；运行到就绪可能因为时间片用完或被高优先级进程抢占；运行到阻塞通常因为等待 I/O、等待资源或等待事件；阻塞到就绪是因为等待事件发生。

进程上下文是进程运行所依赖的全部环境，包括用户级上下文、寄存器上下文和系统级上下文。进程切换时，操作系统需要保存当前进程上下文，并恢复新进程上下文，因此进程切换有较大开销。

引入线程是为了减少并发执行单位切换和通信的开销。线程比进程轻量，同一进程内线程共享地址空间，通信方便，创建和切换速度较快。但线程共享资源也带来同步互斥问题。

作业调度从后备作业队列中选择作业进入内存运行，属于高级调度。进程调度从就绪队列中选择进程占用 CPU，属于低级调度。两者服务对象、调度频率和影响范围不同。

## 四、典型算法或计算方法

本章主要是概念题，少量题会涉及状态转换分析。

进程状态转换分析步骤：

1. 判断进程当前状态：是否占用 CPU、是否具备运行条件、是否等待事件。
2. 判断事件类型：获得 CPU、时间片到、申请 I/O、I/O 完成、资源满足。
3. 写出状态转换方向。
4. 注意阻塞态不能直接变成运行态，通常先进入就绪态，再由调度程序选择运行。

作业与进程关系题模板：

1. 作业是用户提交的任务集合，是宏观概念。
2. 进程是程序的一次执行，是系统分配资源和调度的对象。
3. 一个作业进入内存后，可以产生一个或多个进程。
4. 作业调度决定哪些作业进入内存，进程调度决定哪个进程占用 CPU。

## 五、易混淆知识点辨析

程序与进程：程序是静态指令集合；进程是动态执行过程。程序无状态，进程有状态和 PCB。

进程与线程：进程是资源分配单位，线程是处理机调度单位。进程间切换开销大，线程间切换开销小。同一进程内线程共享地址空间，进程之间一般相互独立。

作业与进程：作业从用户角度描述一次任务，进程从系统角度描述运行实体。一个作业可对应多个进程。

就绪与阻塞：就绪是只缺 CPU；阻塞是缺少某事件或资源，即使 CPU 空闲也不能运行。

进程切换与模式切换：进程切换更换当前运行进程，需要保存和恢复上下文；模式切换只是同一进程在用户态和核心态之间切换。

## 六、期末常考简答题

### 问题：为什么要引入进程？

答：
1. 多道程序并发执行时，程序的执行顺序和速度不再固定，单纯用程序无法描述运行状态。
2. 操作系统需要记录每个运行实体的状态、资源和现场信息。
3. 进程为程序的并发执行提供了独立的管理单位。
4. 通过进程，操作系统可以实现调度、同步、通信和资源分配。

### 问题：进程与程序有什么区别和联系？

答：
1. 程序是静态的指令集合，进程是程序的一次动态执行过程。
2. 程序可以长期保存，进程有创建、运行和撤销的生命周期。
3. 程序没有并发性，进程可以与其他进程并发执行。
4. 一个程序可以对应多个进程，一个进程通常包含程序、数据和 PCB。

### 问题：PCB 的作用是什么？通常包含哪些内容？

答：
1. PCB 是进程存在的唯一标志，操作系统通过 PCB 管理进程。
2. PCB 保存进程标识符、进程状态、程序计数器和寄存器现场。
3. PCB 保存调度信息，如优先级、队列指针和时间片信息。
4. PCB 保存资源信息，如内存区域、打开文件和 I/O 状态。

### 问题：进程的三种基本状态及转换原因是什么？

答：
1. 运行态：进程正在占用 CPU 执行。
2. 就绪态：进程具备运行条件，只等待处理机。
3. 阻塞态：进程等待 I/O、资源或事件，暂不能执行。
4. 就绪到运行由调度引起；运行到就绪由时间片到或抢占引起；运行到阻塞由等待事件引起；阻塞到就绪由等待事件完成引起。

### 问题：进程和线程有什么区别？

答：
1. 进程是资源分配的基本单位，线程是处理机调度的基本单位。
2. 进程拥有独立地址空间，线程共享所属进程的地址空间。
3. 进程切换开销较大，线程创建和切换开销较小。
4. 线程间通信方便，但需要注意共享数据的同步互斥。

## 七、典型题解题模板

题型识别方法：看到“PCB、状态转换、进程与程序、进程与线程、作业与进程”等关键词，通常考概念辨析或简答。

解题步骤：
1. 先写定义。
2. 再从静态/动态、资源/调度、地址空间、生命周期等角度比较。
3. 如果问状态转换，要画或描述三态模型。
4. 如果问原因，要说明引入该概念解决了什么管理问题。

常见错误：
1. 把进程说成程序本身。
2. 把线程说成资源分配单位。
3. 认为阻塞进程在 CPU 空闲时可以直接运行。
4. 忘记 PCB 是进程存在的标志。
