# 第3章 中断与处理机调度

## 一、本章考试定位

本章是期末考试的重点章。根据 `2024操作系统复习范围.docx`，第3章重点为 `3.1、3.2、3.3、3.4`，其中“作业调度”被明确列为计算题。历年试题中出现过中断、Interrupt、中断过程、作业调度与进程调度区别、FCFS、RR 等调度计算。

常见题型包括名词解释、简答题、选择判断、调度计算题和综合分析题。中断部分偏概念和过程，调度部分偏算法与计算。

## 二、核心概念

中断是指处理机运行过程中出现某一事件，必须暂停当前程序，转去处理该事件，处理完后再返回原程序继续执行的过程。操作系统常被称为中断驱动的系统，因为很多系统服务、I/O 完成和异常处理都由中断触发。

中断系统由中断装置和中断处理程序组成。中断装置是硬件机构，负责发现中断、识别中断源、保存必要现场并引出中断处理程序；中断处理程序是软件，负责分析原因并完成处理。

中断源是引起中断的事件。中断字保存与中断事件相关的信息。中断向量通常保存中断处理程序入口地址及其运行环境。

强迫性中断是当前程序不希望发生但必须处理的中断，如时钟中断、I/O 中断、硬件故障和程序异常。自愿性中断是程序主动请求操作系统服务产生的中断，如系统调用或访管指令。

处理机调度是操作系统按照一定策略从就绪队列中选择一个进程或作业使用处理机。调度目标包括提高 CPU 利用率、吞吐量，缩短周转时间和响应时间，并兼顾公平性。

周转时间是作业从提交到完成所经历的时间。带权周转时间是周转时间与实际运行时间之比，用来衡量用户等待相对于服务时间的程度。

## 三、重要原理与机制

中断机制的作用是让 CPU 能及时响应外部事件和异常情况，使操作系统能够获得控制权。没有中断，多道程序设计、分时系统、I/O 并发和系统调用都难以实现。

中断处理的一般过程包括：响应中断、保存现场、分析中断原因、转入中断处理程序、处理中断、恢复现场、返回被中断程序。保存现场是为了保证中断处理后原程序能继续正确执行。

中断嵌套是指高优先级中断可以打断低优先级中断处理程序。它需要中断优先级、中断屏蔽和系统栈支持。系统栈用于保存嵌套中断现场、函数调用参数和返回地址。

时钟中断用于实现时间片轮转、统计系统时间、更新调度参数和唤醒定时等待进程。I/O 中断用于通知设备传输完成或出现错误。

处理机调度需要回答三个问题：按什么原则分配 CPU、何时重新分配 CPU、怎样完成上下文切换。调度算法不同，系统响应时间、平均周转时间和公平性也不同。

调度级别通常包括高级调度、中级调度和低级调度。高级调度又称作业调度，决定哪些作业进入内存；低级调度又称进程调度，决定哪个就绪进程占用 CPU。

## 四、典型算法或计算方法

### FCFS 先来先服务

算法思想：按照作业或进程到达的先后顺序调度。

解题步骤：
1. 按到达时间排序。
2. 依次计算开始时间和完成时间。
3. 周转时间 = 完成时间 - 到达时间。
4. 带权周转时间 = 周转时间 / 运行时间。

优点是公平、实现简单；缺点是短作业可能等待很久，平均周转时间不一定好。

### SJF 短作业优先

算法思想：优先选择估计运行时间最短的作业。

解题步骤：
1. 在已到达作业中选择运行时间最短者。
2. 计算开始、完成、周转和带权周转时间。
3. 若题目说明不可抢占，则运行后不被打断。

优点是平均等待时间和平均周转时间较短；缺点是长作业可能饥饿，且运行时间不容易准确预知。

### HRRN 高响应比优先

算法思想：选择响应比最高的作业。

公式：响应比 = (等待时间 + 服务时间) / 服务时间。

解题步骤：
1. 每次调度前计算所有已到达作业的等待时间。
2. 按公式计算响应比。
3. 选择响应比最大者运行。

优点是兼顾短作业和长作业，等待越久优先级越高。

### RR 时间片轮转

算法思想：按时间片轮流为就绪进程分配 CPU。

解题步骤：
1. 按到达顺序进入就绪队列。
2. 每个进程最多运行一个时间片。
3. 未完成则回到队尾，完成则记录完成时间。
4. 最后计算周转时间和带权周转时间。

易错点：新到达进程和时间片结束进程入队顺序要按题目约定处理；时间片过大接近 FCFS，过小切换开销增大。

### 优先级调度

算法思想：选择优先级最高的进程运行。优先级可以静态确定，也可以动态调整。

易错点：必须看清题目规定“数字越大优先级越高”还是“数字越小优先级越高”，以及是否可抢占。

## 五、易混淆知识点辨析

中断与异常：中断常由外部事件引起，如时钟或 I/O；异常常由当前指令执行引起，如越界、除零、缺页。

强迫性中断与自愿性中断：强迫性中断不是程序主动请求；自愿性中断是程序为了获得系统服务主动执行访管指令。

作业调度与进程调度：作业调度决定作业是否进入内存，频率较低；进程调度决定哪个就绪进程占用 CPU，频率较高。

周转时间与等待时间：周转时间是提交到完成的总时间；等待时间通常是不占用 CPU、在队列中等待的时间。

FCFS 与 RR：FCFS 一次运行到完成或阻塞；RR 按时间片轮流运行，更适合分时系统。

SJF 与 SRTN：SJF 通常不可抢占；SRTN 是最短剩余时间优先，可被新来的短作业抢占。

## 六、期末常考简答题

### 问题：什么是中断？为什么说操作系统是中断驱动的？

答：
1. 中断是 CPU 暂停当前程序、转去处理某事件、处理完后再返回原程序的过程。
2. 操作系统通过中断获得处理机控制权，处理 I/O 完成、时钟、异常和系统调用。
3. 多道程序、分时系统、设备管理和系统调用都依赖中断机制。
4. 因此，操作系统的许多活动都是由中断事件触发的。

### 问题：中断处理的一般过程是什么？

答：
1. 中断装置发现中断并识别中断源。
2. 硬件保存必要现场，如 PSW 和 PC。
3. 根据中断向量转入相应中断处理程序。
4. 中断处理程序保存更多现场，分析原因并处理中断。
5. 恢复现场，执行中断返回，继续原程序或重新调度。

### 问题：什么是中断向量？

答：
1. 中断向量是与某类中断对应的一组入口信息。
2. 它通常包含中断处理程序入口地址及运行所需的状态信息。
3. 系统根据中断类型查找中断向量，从而转入相应处理程序。
4. 中断向量使不同中断能被快速分派到不同处理程序。

### 问题：处理机调度算法应考虑哪些指标？

答：
1. CPU 利用率：希望处理机尽可能忙碌。
2. 吞吐量：单位时间完成的作业数量。
3. 周转时间：作业从提交到完成的时间。
4. 响应时间：用户提交请求到首次响应的时间。
5. 公平性：避免某些进程长期得不到服务。

### 问题：FCFS、SJF 和 RR 各有什么特点？

答：
1. FCFS 按到达顺序调度，公平简单，但短作业可能等待过久。
2. SJF 优先短作业，平均等待时间较短，但长作业可能饥饿。
3. RR 按时间片轮转，适合分时系统，响应较好，但时间片过小会增加切换开销。

## 七、典型题解题模板

题型识别方法：题目给出作业到达时间、运行时间、优先级、时间片，要求周转时间或平均带权周转时间，就是调度计算题。

解题步骤：
1. 画时间轴。
2. 按指定算法确定每一段 CPU 分配给谁。
3. 记录每个作业完成时间。
4. 计算周转时间 = 完成时间 - 到达时间。
5. 计算带权周转时间 = 周转时间 / 服务时间。
6. 求平均值时注意保留题目要求的小数位。

常见错误：
1. 忘记只在“已到达作业”中选择。
2. 把等待时间当成周转时间。
3. RR 中时间片结束后队列顺序处理错误。
4. 优先级调度没有看清数字大小含义。
5. 可抢占和不可抢占算法混用。
