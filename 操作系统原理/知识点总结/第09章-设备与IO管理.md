# 第9章 设备与I/O管理

## 一、本章考试定位

本章是期末简答题、概念题和计算题综合章。根据 `2024操作系统复习范围.docx`，第9章重点为 `9.1、9.2、9.4、9.9、9.10`，并明确列出“磁头调度”为计算题。简答题资料和试题解析还覆盖 I/O 控制方式、I/O 软件层次、缓冲技术、RAID、SPOOLing 等内容。

常见题型包括选择题、判断题、名词解释、简答题、磁盘调度计算题、磁盘 I/O 参数计算题和设备管理综合题。

## 二、核心概念

设备管理是操作系统对 I/O 设备进行分配、回收、控制、调度和缓冲的功能。它要解决设备速度慢、种类多、共享困难、控制复杂等问题。

I/O 设备按用途可分为存储型设备和输入输出型设备；按共享属性可分为共享设备和独占设备；按数据组织可分为块设备和字符设备。

设备控制器是 CPU 与设备之间的接口，负责接收命令、控制设备、报告状态和传输数据。

设备驱动程序是直接控制具体设备的软件模块。它屏蔽设备硬件差异，向上层提供统一接口。

设备独立性是指用户使用逻辑设备名，不直接依赖具体物理设备。这样可以提高系统灵活性，便于设备替换和分配。

缓冲是在内存中开辟区域暂存输入输出数据，用于协调 CPU 与外设速度差异，减少中断次数，提高并发性。

RAID 是独立磁盘冗余阵列技术，通过多个磁盘并行和冗余存储提高性能或可靠性。

虚拟设备是利用共享设备模拟出的逻辑独占设备。SPOOLing 是虚拟设备的典型实现。

## 三、重要原理与机制

I/O 管理需要解决速度不匹配问题。CPU 速度远高于外设，若 CPU 一直等待设备完成，会造成巨大浪费。因此操作系统使用中断、DMA、通道、缓冲和 I/O 进程等机制提高并发性。

常见 I/O 控制方式有程序查询方式、中断驱动方式、DMA 方式和通道方式。程序查询方式简单但浪费 CPU；中断方式减少等待；DMA 允许设备与内存直接传输数据；通道方式由专门 I/O 处理器控制复杂传输。

设备分配要根据设备类型采用不同策略。独占设备一次只能分配给一个进程；共享设备可按块或请求交叉使用；虚拟设备通过 SPOOLing 把独占设备改造成可共享逻辑设备。

设备调度尤其关注磁盘调度。磁盘访问时间主要由寻道时间、旋转延迟和传输时间组成，其中寻道时间常是优化重点。

缓冲技术包括单缓冲、双缓冲、循环缓冲和缓冲池。缓冲可以平滑速度差异、减少中断、提高 CPU 与设备并行度。

SPOOLing 通过输入井、输出井、预输入进程和缓输出进程，把慢速独占设备的操作转化为对磁盘文件的操作，提高设备利用率。

## 四、典型算法或计算方法

### 磁盘访问时间

常见公式：

1. 磁盘访问时间 = 寻道时间 + 旋转延迟 + 传输时间。
2. 寻道时间常表示为 `Ts = m * n + s`，其中 `n` 为跨越磁道数。
3. 传输时间常与传输字节数、转速和每道字节数有关。

易错点：题目若只问磁头移动总量，通常只需统计磁道号移动距离；若问访问时间，要看是否包含旋转延迟和传输时间。

### FCFS 磁盘调度

算法思想：按请求到达顺序服务。

解题步骤：
1. 从当前磁头位置出发。
2. 按请求序列依次移动。
3. 累加相邻磁道号差的绝对值。

优点公平简单；缺点平均寻道距离可能很大。

### SSTF 最短寻道时间优先

算法思想：每次选择距离当前磁头最近的请求。

解题步骤：
1. 从当前磁头位置开始。
2. 在未服务请求中找距离最近者。
3. 移动到该磁道并删除请求。
4. 重复直到完成。

优点减少平均寻道时间；缺点远处请求可能饥饿。

### SCAN 电梯算法

算法思想：磁头沿一个方向移动，途中服务所有请求，到达端点或最远请求后再反向。

解题步骤：
1. 明确初始磁头位置和移动方向。
2. 按该方向依次服务请求。
3. 到端点或题目指定位置后反向。
4. 累加移动距离。

易错点：是否必须走到磁盘端点，要按题目说明或课程约定。

### C-SCAN 循环扫描

算法思想：磁头只沿一个方向服务请求，到达末端后直接返回另一端，再继续同方向服务。

优点响应时间更均匀；常用于负载较重的磁盘系统。

### RAID 级别理解

RAID 0 强调条带化提高性能，无冗余。RAID 1 镜像提高可靠性。RAID 5 分布式奇偶校验，兼顾容量、性能和可靠性。

## 五、易混淆知识点辨析

块设备与字符设备：块设备以块为单位传输并支持随机访问，如磁盘；字符设备以字符流方式传输，如键盘、打印机。

独占设备与共享设备：独占设备同一时刻只能被一个进程使用；共享设备可被多个进程交叉访问。

程序查询与中断方式：查询方式 CPU 主动反复检查设备状态；中断方式设备完成后主动通知 CPU。

DMA 与通道：DMA 主要完成设备与内存之间的数据块传输；通道是更独立的 I/O 处理机，可执行通道程序。

缓冲与缓存：缓冲强调速度匹配和临时存放；缓存强调保存常用数据以提高命中率。

SPOOLing 与虚拟设备：SPOOLing 是实现虚拟设备的一种技术；虚拟设备是逻辑上提供给进程的设备抽象。

SCAN 与 C-SCAN：SCAN 往返扫描；C-SCAN 单方向服务，回到起点后继续同方向。

## 六、期末常考简答题

### 问题：I/O 软件一般分为哪几个层次？

答：
1. 中断处理程序，负责处理中断事件。
2. 设备驱动程序，负责控制具体设备。
3. 与设备无关的 I/O 软件，提供统一接口、命名、保护和缓冲等功能。
4. 用户空间 I/O 软件，为用户程序提供库函数或系统调用封装。

### 问题：常见的 I/O 控制方式有哪些？

答：
1. 程序直接控制方式：CPU 通过查询设备状态完成传输，简单但浪费 CPU。
2. 中断控制方式：设备完成操作后发出中断，CPU 不必一直等待。
3. DMA 方式：设备和内存之间直接交换数据，CPU 只负责初始化和结束处理。
4. 通道方式：由专门通道执行 I/O 程序，进一步减轻 CPU 负担。

### 问题：为什么要引入缓冲技术？

答：
1. 缓冲可以协调 CPU 与外设之间的速度差异。
2. 缓冲能减少 I/O 中断次数和 CPU 中断处理开销。
3. 缓冲可提高 CPU 与设备、设备与设备之间的并行性。
4. 在 DMA 或通道方式下，缓冲还能改善数据传输组织。

### 问题：什么是 SPOOLing？它有什么特点？

答：
1. SPOOLing 是假脱机技术，利用磁盘作为后援存储器实现输入输出管理。
2. 它由输入井、输出井、预输入进程和缓输出进程等组成。
3. 它把慢速独占设备操作转化为对磁盘区域的操作。
4. 它提高 I/O 速度和设备利用率，并实现虚拟设备功能。

### 问题：设备独立性有什么意义？

答：
1. 用户使用逻辑设备名，不必关心具体物理设备。
2. 操作系统可以根据情况把逻辑设备映射到不同物理设备。
3. 设备替换或故障转移时，不需要修改用户程序。
4. 它提高了系统的灵活性、可移植性和资源利用率。

## 七、典型题解题模板

题型识别方法：题目给出当前磁头位置、请求磁道序列、磁头移动方向和调度算法，要求求服务顺序或磁头移动总量，就是磁盘调度题。

解题步骤：
1. 写出当前磁头位置。
2. 按算法确定服务顺序。
3. 逐段计算移动距离，即相邻磁道号差的绝对值。
4. 累加总移动量。
5. 若题目要求平均寻道长度，再除以请求数。

常见错误：
1. SSTF 每一步都要重新找最近请求。
2. SCAN 没看清初始移动方向。
3. SCAN 是否走到端点没有按题目要求。
4. C-SCAN 返回过程是否计入移动距离要按题目说明。
5. 把磁头移动总量和磁盘访问时间混为一谈。
