# 第7章 虚拟存储管理

## 一、本章考试定位

本章是期末计算题重点章。根据 `2024操作系统复习范围.docx`，第7章重点为 `7.2、7.3、7.4`，并明确列出“页面调度”为计算题。历年试题中出现过 Virtual Memory、Thrashing、静态页面管理与请求页式管理区别、LRU 页面置换和缺页率计算。

常见题型包括名词解释、简答题、选择判断、页面置换计算题和虚拟存储机制分析题。

## 二、核心概念

虚拟存储是指把进程的一部分装入内存，另一部分保存在外存中，运行时根据需要动态调入内存，使用户感觉可使用的存储空间大于实际内存。

局部性原理是虚拟存储能够实现的理论基础。时间局部性指刚访问过的内容短期内可能再次访问；空间局部性指访问某地址后，附近地址也可能被访问。

请求分页存储管理是虚拟页式存储的常见形式。进程开始运行时不必把所有页面装入内存，访问页面不在内存时产生缺页中断，再由操作系统调入所需页面。

缺页中断是进程访问的页面不在内存时产生的中断。操作系统需要找到页面在外存的位置，分配页框，必要时淘汰某页，再把所需页读入内存。

页面置换是内存页框不足时，选择一个已在内存中的页面换出，以便装入新页面的过程。页面置换算法直接影响缺页率。

抖动是指页面在内存和外存之间频繁换入换出，系统大部分时间用于页面调度而不是执行用户程序，导致性能急剧下降。

工作集是某段时间窗口内进程实际访问的页面集合。工作集模型用于估计进程当前需要的页面数量，避免分配页框过少导致抖动。

## 三、重要原理与机制

虚拟存储解决的问题是：内存容量有限，不能一次装入大程序；即使能装入，程序运行时也往往只使用其中一部分。通过按需调入，可以提高内存利用率和多道程序度。

请求分页的基本工作过程是：CPU 产生逻辑地址，查页表发现页面不在内存，产生缺页中断；操作系统找到外存地址；若有空闲页框则直接装入；若没有空闲页框则按置换算法淘汰一页；必要时写回被修改页；读入所需页并更新页表；重新执行被中断指令。

虚拟存储容量受计算机地址结构和外存容量影响，但从考试角度常强调“最大虚拟存储容量由地址结构决定”，而实际可用容量还受外存和系统实现限制。

虚拟段式存储按段进行动态调入，便于共享、保护和动态链接。虚拟段页式结合段式逻辑结构和页式离散分配，适用于复杂系统。

页面置换算法的目标是降低缺页率。理想算法 OPT 缺页率最低但实际不可实现；FIFO 简单但可能出现 Belady 异常；LRU 利用局部性思想，效果较好但实现成本较高；Clock 是 LRU 的近似实现。

## 四、典型算法或计算方法

### OPT 最佳置换算法

算法思想：淘汰未来最长时间不会被访问的页面。

解题步骤：
1. 按页面访问序列逐个处理。
2. 若页面已在内存，命中，不缺页。
3. 若不在内存且有空页框，调入。
4. 若不在内存且无空页框，查看未来访问序列，淘汰最晚再用或不再使用的页面。

易错点：OPT 要看未来，只适合理论分析和考试计算。

### FIFO 先进先出算法

算法思想：淘汰最早进入内存的页面。

解题步骤：
1. 维护页面进入内存的队列。
2. 缺页且无空页框时，淘汰队首页面。
3. 新页面进入队尾。

易错点：FIFO 不考虑页面最近是否被访问，可能出现页框增多但缺页次数反而增加的 Belady 异常。

### LRU 最近最久未使用算法

算法思想：淘汰最长时间没有被访问的页面。

解题步骤：
1. 每次访问页面后，更新该页面最近访问时间。
2. 缺页且无空页框时，淘汰最近访问时间最早的页面。
3. 继续处理访问序列，统计缺页次数和缺页率。

缺页率公式：缺页率 = 缺页次数 / 页面访问总次数。

### Clock 时钟置换算法

算法思想：给每个页面设置访问位，用循环指针扫描页面。访问位为 1 时改为 0 并跳过，访问位为 0 时淘汰。

解题步骤：
1. 页面被访问后访问位置 1。
2. 需要置换时从指针位置开始扫描。
3. 遇到访问位 1，改 0，指针后移。
4. 遇到访问位 0，选择该页淘汰。

## 五、易混淆知识点辨析

虚拟存储与物理内存：虚拟存储是进程看到的逻辑存储空间；物理内存是实际 RAM。

请求分页与基本分页：基本分页通常要求页面已在内存；请求分页允许页面不在内存，访问时再调入。

缺页中断与一般中断：缺页中断由访问不在内存的页面引起，处理后通常重新执行原指令。

页面调入与页面置换：调入是把所需页装入内存；置换是在无空页框时选择旧页换出。

FIFO 与 LRU：FIFO 看进入内存时间；LRU 看最近访问时间。

抖动与缺页：缺页是正常事件；抖动是缺页过于频繁导致系统效率崩溃。

## 六、期末常考简答题

### 问题：什么是虚拟存储器？

答：
1. 虚拟存储器是把内存和外存结合起来，为用户提供比实际内存更大的逻辑存储空间的技术。
2. 程序运行时只把当前需要的部分装入内存，其余部分保存在外存。
3. 当访问的页面或段不在内存时，由操作系统动态调入。
4. 虚拟存储器以局部性原理为基础，能提高内存利用率和系统并发能力。

### 问题：请求分页系统发生缺页中断后如何处理？

答：
1. 操作系统根据页表或外存表找到所需页面在外存的位置。
2. 查找空闲页框，若有空闲页框则直接使用。
3. 若没有空闲页框，则按页面置换算法选择一页淘汰。
4. 若被淘汰页被修改过，需要写回外存。
5. 把所需页读入内存，修改页表和相关状态位。
6. 重新启动产生缺页中断的指令。

### 问题：什么是抖动？如何避免？

答：
1. 抖动是页面频繁换入换出，导致系统大部分时间用于页面调度而非执行程序的现象。
2. 抖动通常由分配给进程的页框数过少或多道程序度过高引起。
3. 可以通过降低多道程序度、增加进程驻留集大小来缓解。
4. 也可以利用工作集模型，根据进程近期访问页面集合分配页框。

### 问题：FIFO、LRU、OPT 页面置换算法有何区别？

答：
1. FIFO 淘汰最早进入内存的页面，实现简单，但可能淘汰常用页。
2. LRU 淘汰最近最久未使用的页面，利用局部性原理，效果较好。
3. OPT 淘汰未来最长时间不会访问的页面，缺页率最低，但实际无法预知未来。
4. Clock 是 LRU 的近似算法，用访问位降低实现成本。

## 七、典型题解题模板

题型识别方法：题目给出页面访问序列、页框数和页面置换算法，要求求缺页次数、缺页率或页面淘汰序列，就是页面置换题。

解题步骤：
1. 画表格，列出每次访问的页面。
2. 每一列写出访问后内存页框状态。
3. 标记缺页或命中。
4. 缺页且无空页框时，按指定算法选择淘汰页。
5. 统计缺页次数。
6. 计算缺页率 = 缺页次数 / 访问总次数。

常见错误：
1. 把命中也算成缺页。
2. LRU 中没有更新命中页面的最近访问时间。
3. FIFO 中误按最近访问时间淘汰。
4. OPT 中没有查看未来访问序列。
5. 忘记初始空页框调入也算缺页。
