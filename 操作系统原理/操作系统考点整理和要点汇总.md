# 操作系统考点整理和要点汇总

说明：本文件在 `操作系统试题详细解析.md` 的基础上整理，突出高频考点、易错点和计算题方法，适合考前快速复习。

---

## 一、操作系统概述

### 高频考点

- 操作系统定义：管理计算机系统资源、控制程序执行、为用户和应用提供接口的系统软件。
- 操作系统作用：资源管理者、用户与硬件之间的接口、扩展机器。
- 操作系统特征：并发性、共享性、虚拟性、异步性。
- 操作系统主要功能：处理机管理、存储器管理、设备管理、文件管理、作业管理。

### 易错点

- 操作系统管理的是“计算机资源”，包括硬件资源和软件资源。
- 并发不等于并行：并发是宏观上同时推进，并行是微观上真正同时执行。
- 分时系统强调交互响应，多道批处理强调系统吞吐量和 CPU 利用率。

---

## 二、进程与线程

### 高频考点

- 进程：程序在数据集合上的一次运行活动，是资源分配和调度的基本单位。
- 线程：进程中的执行流，是 CPU 调度和执行的基本单位。
- `PCB`：进程控制块，是进程存在的唯一标志。
- 进程三态：就绪、运行、阻塞。
- 上下文切换：保存当前现场，恢复下一个进程/线程现场。

### 进程与程序

| 比较项 | 程序 | 进程 |
| --- | --- | --- |
| 性质 | 静态 | 动态 |
| 内容 | 指令集合 | 程序、数据、PCB、执行状态 |
| 生命周期 | 可长期保存 | 创建、运行、阻塞、终止 |
| 并发性 | 不体现 | 可并发执行 |

### 进程与线程

| 比较项 | 进程 | 线程 |
| --- | --- | --- |
| 基本属性 | 资源分配单位 | CPU 调度单位 |
| 地址空间 | 通常独立 | 同一进程内共享 |
| 切换开销 | 大 | 小 |
| 通信 | 需要 IPC | 共享变量方便，但需同步 |

### 代表性答题模板

问“为什么引入进程”：

1. 程序是静态的，无法描述并发执行；
2. 进程能表示程序的一次动态执行；
3. 操作系统可基于进程完成资源分配、调度、同步、通信和保护。

## 三、同步、互斥与死锁

### 高频考点

- 临界区：访问临界资源的代码段。
- 信号量：只能通过 `P/V` 原语操作的同步机制。
- 原语：不可被中断、具有原子性的操作序列。
- 读者写者问题：读者优先可能导致写者饥饿；写者优先要阻止新读者插队。
- 吸烟者问题：用多个信号量精确唤醒对应进程。

### 死锁四个必要条件

1. 互斥条件；
2. 请求和保持条件；
3. 不剥夺条件；
4. 循环等待条件。

### 死锁处理方法

| 方法 | 思路 | 典型技术 |
| --- | --- | --- |
| 死锁预防 | 破坏四个必要条件之一 | 一次性分配、资源有序分配 |
| 死锁避免 | 分配前判断是否会进入不安全状态 | 银行家算法 |
| 死锁检测 | 允许死锁发生，定期检测 | 资源分配图、等待图 |
| 死锁解除 | 发现后恢复系统 | 撤销进程、剥夺资源、回滚 |

### 易错点

- 银行家算法是死锁避免，不是死锁预防。
- 不安全状态不一定已经死锁，但安全状态一定不会因当前分配序列导致死锁。
- 互斥和同步是进程间制约关系，不宜直接说成“通信方式”。

## 四、处理机调度

### 调度层次

| 调度 | 又称 | 作用 |
| --- | --- | --- |
| 作业调度 | 高级调度 | 从后备队列选择作业进入内存 |
| 内存调度 | 中级调度 | 挂起/激活进程，调节内存负载 |
| 进程调度 | 低级调度 | 从就绪队列选择进程运行 |

### 常见算法

- `FCFS`：先来先服务，实现简单，但短作业可能等待久。
- `SJF`：短作业优先，平均等待时间短，但长作业可能饥饿。
- 优先级调度：按优先级选择，低优先级可能饥饿，可用老化机制缓解。
- 时间片轮转 `RR`：公平，适合分时系统；时间片过小切换开销大，过大退化为 FCFS。
- 多级反馈队列：综合响应时间、优先级和公平性。

### 计算题方法

周转时间：

`周转时间 = 完成时间 - 到达时间`

带权周转时间：

`带权周转时间 = 周转时间 / 实际运行时间`

时间片轮转：

1. 建立就绪队列；
2. 每次运行 `min(时间片, 剩余时间)`；
3. 未完成则回队尾；
4. 剩余时间为 0 时记录完成时刻。

## 五、存储管理

### 基本概念

- 地址空间：程序可见的逻辑地址集合。
- 存储空间：主存中的物理地址集合。
- 地址映射：逻辑地址到物理地址的转换。
- `MMU`：执行地址变换和访问保护的硬件部件。
- 虚拟内存：基于局部性原理，把内存和外存结合，为进程提供较大逻辑地址空间。

### 分页与分段

| 比较项 | 分页 | 分段 |
| --- | --- | --- |
| 划分依据 | 固定大小页 | 程序逻辑模块 |
| 单位大小 | 固定 | 可变 |
| 地址结构 | 页号 + 页内位移 | 段号 + 段内位移 |
| 碎片 | 主要有内部碎片 | 主要有外部碎片 |
| 优点 | 便于离散分配 | 便于共享和保护 |

### 分页地址变换

步骤：

1. `页号 = 逻辑地址 // 页大小`；
2. `页内位移 = 逻辑地址 mod 页大小`；
3. 查页表，若页不在内存则缺页中断；
4. `物理地址 = 块号 * 块大小 + 页内位移`。

### 分段地址变换

步骤：

1. 检查段号是否存在；
2. 检查段是否在内存；
3. 检查段内位移是否小于段长；
4. `物理地址 = 段基址 + 段内位移`。

### 页面置换算法

- `OPT`：淘汰未来最长时间不用的页，理论最优。
- `FIFO`：淘汰最早进入内存的页，可能出现 Belady 异常。
- `LRU`：淘汰最近最长时间未使用的页，符合局部性，属于栈式算法。
- `LFU`：淘汰访问次数最少的页。

### 易错点

- 页大小固定，段大小可变。
- 静态重定位发生在程序装入时；动态重定位发生在程序执行过程中。
- 虚拟存储器最大容量在理论上受地址结构限制，实际还受外存容量限制。
- 缺页中断不是程序出错，越界访问才是非法访问。

## 六、文件系统

### 高频考点

- 文件目录：实现文件按名存取。
- 目录文件：把目录项作为文件内容保存的特殊文件。
- 文件描述符：进程打开文件后用于标识打开文件的整数句柄。
- 文件物理结构：顺序结构、链接结构、索引结构。
- inode：保存文件元数据和数据块地址，目录项保存文件名到 inode 的映射。

### 顺序存取与随机存取

| 比较项 | 顺序存取 | 随机存取 |
| --- | --- | --- |
| 访问方式 | 按记录顺序读写 | 按偏移或记录号直接定位 |
| 适用介质 | 磁带、日志文件 | 磁盘、数据库文件 |
| 优点 | 实现简单 | 查询灵活 |

### 索引文件计算方法

核心公式：

`每个索引块可存地址数 = 块大小 / 地址项大小`

最大数据块数按层累加：

- 直接索引：`直接项数`
- 一级间接：`N`
- 二级间接：`N^2`
- 三级间接：`N^3`

实际占用空间：

`数据块空间 + 用到的索引块空间`

注意：题目说“不包括 inode”时，仍要计算间接索引块。

### 权限控制

文件权限控制可通过用户/用户组、权限位、访问控制表 `ACL`、打开文件检查、系统调用检查实现。普通用户通过文件名访问文件，内核通过目录项、inode、权限信息决定是否允许操作。

## 七、设备管理与磁盘调度

### 高频考点

- 设备驱动程序：向上提供统一接口，向下控制具体设备。
- 设备独立性：用户程序不依赖具体物理设备。
- 缓冲区：协调速度差，支持异步和批量传输。
- 高速缓存：保存可能再次访问的数据副本，提高命中率。
- `SPOOLing`：假脱机技术，把独占设备改造成逻辑共享设备。
- `DMA`：设备与内存直接交换数据，减少 CPU 干预。

### 磁盘调度算法

| 算法 | 规则 | 特点 |
| --- | --- | --- |
| FCFS | 按请求到达顺序服务 | 公平但移动距离可能大 |
| SSTF | 每次选择距离当前磁头最近的请求 | 平均距离短，但可能饥饿 |
| SCAN | 电梯式向一个方向扫描，到端点后反向 | 兼顾效率和公平 |
| C-SCAN | 单方向扫描，到端点后回到另一端继续 | 等待时间更均匀 |
| LOOK | 类似 SCAN，但到最远请求处就折返 | 不扫无请求端点 |
| C-LOOK | 类似 C-SCAN，但只在请求范围内循环 | 减少无效移动 |

### 磁盘调度计算套路

1. 确定当前磁头位置；
2. 根据“刚完成的位置 -> 当前正在服务的位置”判断扫描方向；
3. 写出服务序列；
4. 逐段计算绝对距离；
5. 总寻道长度除以请求数得到平均寻道长度。

### 易错点

- `SCAN` 是否扫到端点取决于教材口径；严格 `SCAN` 扫到端点，`LOOK` 不扫到端点。
- `C-SCAN` 从末端回到起点的移动是否计入总寻道量也可能因教材而异；若题目给出柱面范围，通常应计入。
- `SPOOLing` 主要提高独占设备利用率，不是单纯提高 CPU 速度。

## 八、银行家算法速记

### 基本数据结构

- `Available`：系统当前可用资源。
- `Max`：进程最大资源需求。
- `Allocation`：已分配给进程的资源。
- `Need = Max - Allocation`：进程还需要的资源。

### 安全性检查

1. `Work = Available`；
2. 找一个未完成进程，使 `Need[i] <= Work`；
3. 假设该进程完成，`Work = Work + Allocation[i]`；
4. 重复直到所有进程完成；
5. 若能完成所有进程，则安全，否则不安全。

### 资源请求检查

对进程 `Pi` 的请求 `Request[i]`：

1. 检查 `Request[i] <= Need[i]`；
2. 检查 `Request[i] <= Available`；
3. 试分配：
   - `Available -= Request[i]`
   - `Allocation[i] += Request[i]`
   - `Need[i] -= Request[i]`
4. 做安全性检查；
5. 安全才真正分配，不安全则回滚。

### 常见失分点

- 忘记先算 `Need`。
- 只检查请求量不超过可用量，忘记安全性检查。
- 请求失败后仍把资源当作已分配，导致后续题状态错误。

## 九、常见简答题模板

### 中断过程

发现中断源，响应中断；保存断点和现场；根据中断向量找到服务程序；执行中断服务程序；恢复现场；返回原程序。

### 缺页中断处理

检查地址合法性；若合法，查找空闲块；无空闲块则选择牺牲页；必要时写回；调入所需页；修改页表；重新执行被中断指令。

### 上下文切换

保存当前进程寄存器、程序计数器、栈指针和状态到 PCB；选择下一个进程；恢复其现场；切换地址空间；继续执行。开销来自保存恢复现场、调度、缓存/TLB 失效。

### SPOOLing

利用磁盘输入井/输出井和后台进程，把慢速独占设备虚拟成可共享设备，使用户进程不必直接等待实际设备完成。

### 启动过程

加电复位；执行 BIOS/UEFI；硬件自检；查找启动设备；加载引导程序；装入操作系统内核；初始化内存、设备、文件系统和系统服务。

### 原语与系统调用

原语是不可中断的原子操作，常用于内核内部同步和进程控制；系统调用是用户程序请求操作系统服务的接口。

## 十、考前优先级建议

### 必须熟练

- 进程、线程、PCB、三态转换；
- 信号量、临界区、读者写者、吸烟者问题；
- 死锁四条件、银行家算法；
- 分页/分段地址转换；
- FIFO/LRU 页面置换；
- 文件索引空间计算；
- 磁盘调度 FCFS/SSTF/SCAN/C-SCAN；
- SPOOLing、DMA、中断过程。

### 最容易出计算题

1. 银行家算法安全序列与请求判断；
2. 页面置换缺页次数和缺页率；
3. 分页/分段地址变换；
4. 多级索引最大文件长度和实际占用空间；
5. 磁盘调度服务次序、总寻道长度、平均寻道长度；
6. 时间片轮转或 FCFS 周转时间。

### 答题风格

- 名词解释：先定义，再补充作用或特点。
- 简答题：按条列写关键词，避免大段空泛表述。
- 计算题：写公式、过程和最终结果；有歧义的算法口径写明假设。
- 同步题：先说明共享资源和互斥点，再给信号量初值和 `P/V` 位置。
