# 吉林大学《操作系统》2020-2021 学年春季学期期末试卷（B）详细解析

说明：本 PDF 与 2020-2021 学年第二学期期末试卷内容基本一致，差别主要是水印/页数。本文件仍按完整试卷逐题解析。

## 一、名词解释

### 1. Interrupt（中断）
答案：中断是 CPU 在执行程序过程中，由外设事件、内部异常或软件请求触发，使 CPU 暂停当前程序，转去执行中断服务程序，处理完成后再返回原程序继续执行的机制。

解析：中断的关键动作是保存现场、识别中断源、执行服务程序、恢复现场。它使 CPU 不必轮询等待 I/O 完成，是实现并发和设备管理的重要基础。

知识点总结：中断机制；中断响应；中断向量；现场保护与恢复；I/O 控制方式。

### 2. Process（进程）
答案：进程是程序在某个数据集合上的一次动态执行过程，是资源分配和调度的基本单位。

解析：程序是静态的，进程是动态的。进程由程序段、数据段和 PCB 构成，具有动态性、并发性、独立性和异步性。

知识点总结：进程与程序区别；PCB；进程状态；资源分配单位。

### 3. Virtual Memory（虚拟存储器）
答案：虚拟存储器是通过地址转换和页面/段调入调出，把外存空间逻辑上扩展为内存，使作业拥有较大虚拟地址空间的技术。

解析：虚拟存储器基于局部性原理，只把当前需要的页或段装入内存。访问不在内存的页面时产生缺页中断，由操作系统调入。

知识点总结：虚拟存储；局部性原理；请求分页；缺页中断；地址空间。

### 4. MMU
答案：MMU 是内存管理单元，负责把逻辑地址/虚拟地址转换为物理地址，并进行越界、权限和状态检查。

解析：分页时 MMU 用页号查页表得到块号，再与页内偏移组成物理地址；分段时用段号查段表得到段基址，并检查段长。

知识点总结：地址转换；页表；段表；TLB；内存保护。

### 5. Thrashing（抖动）
答案：抖动是系统频繁缺页，CPU 大量时间用于页面换入换出，真正执行用户程序的时间很少，性能急剧下降的现象。

解析：常见原因是进程工作集大于可用内存、并发进程过多或驻留集过小。可通过减少多道程序度、增加内存、采用工作集策略等缓解。

知识点总结：缺页率；工作集；页面置换；多道程序度；系统性能。

## 二、简答题

### 1. 什么是虚拟内存？虚拟内存最大容量由什么决定？
答案：虚拟内存是把内存和外存统一组织起来，使程序按虚拟地址空间运行的存储管理技术。理论最大容量由虚拟地址位数决定，实际还受外存容量和系统实现限制。

解析：若虚拟地址为 n 位，则单进程理论虚拟地址空间最大为 2^n 个可寻址单元。物理内存容量影响驻留页数和缺页率，但不直接决定虚拟地址空间上限。

知识点总结：虚拟地址空间；地址位数；内存与外存；虚拟存储容量。

### 2. 地址空间和存储空间是什么？
答案：地址空间是程序可见的逻辑地址集合；存储空间是实际物理内存中的地址集合。

解析：地址空间面向进程，存储空间面向硬件。操作系统通过重定位、页表或段表把逻辑地址映射到物理地址。

知识点总结：逻辑地址；物理地址；地址映射；重定位。

### 3. 文件目录和目录文件是什么？
答案：文件目录是文件控制信息的集合，用于按名存取；目录文件是把目录信息作为特殊文件存放在外存上的形式。

解析：目录项通常包含文件名、类型、物理位置、长度、权限、时间戳等。目录文件由文件系统维护，其内容是目录项。

知识点总结：目录项；目录文件；按名存取；FCB；inode。

### 4. 作业调度和进程调度有什么区别？
答案：作业调度从外存后备队列中选择作业装入内存并建立进程，又称高级调度；进程调度从就绪队列中选择进程占用 CPU，又称低级调度。

解析：作业调度频率低，影响多道程序度；进程调度频率高，直接影响响应时间和 CPU 利用率。

知识点总结：高级调度；低级调度；后备队列；就绪队列；多道程序度。

### 5. 进程和线程有什么区别？
答案：进程是资源分配的基本单位，线程是 CPU 调度的基本单位。同一进程内线程共享地址空间和资源，但各自拥有运行现场和栈。

解析：进程切换开销较大；线程切换开销较小。进程间通信需要 IPC，线程间通信可直接共享变量但要注意同步互斥。

知识点总结：进程；线程；资源分配；CPU 调度；共享地址空间。

## 三、调度计算题

题目：A、B、C、D、E 同时到达，运行时间分别为 10、6、2、4、8 分钟。

### 1. FCFS
答案：A=10，B=16，C=18，D=22，E=30。

解析：执行顺序为 A(0-10)、B(10-16)、C(16-18)、D(18-22)、E(22-30)。同时到达时，周转时间等于完成时刻。

知识点总结：FCFS；周转时间；完成时刻；非抢占调度。

### 2. RR，时间片为 2 分钟
答案：A=30，B=22，C=6，D=16，E=28。

解析：轮转过程为 A、B、C完成、D、E、A、B、D完成、E、A、B完成、E、A、E完成、A完成。完成时刻分别为 C=6、D=16、B=22、E=28、A=30。

知识点总结：时间片轮转；抢占式调度；响应公平性；周转时间。

## 四、银行家算法

Available=(1,6,2,2)。

### 1. 系统是否安全？
答案：安全。一个安全序列为 P0、P3、P4、P1、P2。

解析：Work=(1,6,2,2)。P0 Need=(0,0,1,2) 可完成，Work=(1,6,5,4)；P3 可完成，Work=(1,9,8,6)；P4 可完成，Work=(1,9,9,10)；P1 可完成，Work=(2,9,9,10)；P2 可完成。所有进程均能完成，所以安全。

知识点总结：银行家算法；Available；Allocation；Need；安全序列；死锁避免。

### 2. P2 请求 (1,2,2,2) 能否立即分配？
答案：不能。

解析：请求满足 Request <= Need 且 Request <= Available，可试分配。试分配后 Available=(0,4,0,0)，各进程 Need 均不能被满足，找不到能完成的进程，系统进入不安全状态。因此银行家算法拒绝该请求。

知识点总结：资源请求算法；试分配；安全性检查；不安全状态。

## 五、请求分页地址转换

页大小为 4KB=4096B。页表：0->24，1->26，2->32 在内存；3、4 不在内存。

### 1. 逻辑地址 9016
答案：物理地址为 131896。

解析：页号=9016 div 4096=2，页内偏移=824。第 2 页在块 32 中，物理地址=32*4096+824=131896。

知识点总结：页号；页内偏移；页表；地址转换。

### 2. 逻辑地址 12300
答案：产生缺页中断，暂时不能形成物理地址。

解析：页号=12300 div 4096=3，偏移=12。第 3 页不在内存，访问时触发缺页中断，调入后才能重新转换。

知识点总结：缺页中断；请求分页；状态位；地址转换异常。

## 六、页面置换

访问串：4，3，2，1，4，3，5，4，3，2，1，5。

### 1. FIFO，4 个物理块
答案：缺页序列为 4，3，2，1，5，4，3，2，1，5；缺页次数 10；缺页率 10/12=83.33%。

解析：前四页装满后，访问 5 淘汰 4，随后 4、3、2、1、5 依次造成淘汰，命中较少。

知识点总结：FIFO；缺页序列；缺页率；先进先出队列。

### 2. LRU，4 个物理块
答案：缺页序列为 4，3，2，1，5，2，1，5；缺页次数 8；缺页率 8/12=66.67%。

解析：LRU 每次淘汰最久未被访问的页，利用了近期访问历史，因此本题优于 FIFO。

知识点总结：LRU；局部性原理；最近最久未使用；置换算法比较。

## 七、选答题 1：索引文件系统

条件：8 个直接指针，1 个一级间接，1 个二级间接。块大小 2KB，指针 4B，每个索引块含 512 个指针。

### 1. 最大文件大小
答案：537935872B，约 513.016MB。

解析：可寻址数据块数=8+512+512*512=262664。最大文件大小=262664*2048=537935872B。

知识点总结：直接索引；一级间接索引；二级间接索引；最大文件长度。

### 2. 128MB 文件实际占用空间
答案：134481920B。

解析：128MB 需要 65536 个数据块。索引开销为一级间接索引块 1 个、二级间接根索引块 1 个、二级间接下级索引块 ceil((65536-8-512)/512)=127 个。总块数=65536+129=65665，占用 65665*2048=134481920B。

知识点总结：索引块开销；实际占用空间；向上取整；多级索引。

## 七、选答题 2：读者-写者问题

### 1. 为什么原方案偏向读者？
答案：只要已有读者在读，新读者仍可继续进入；写者必须等 readcount 变为 0 才能获得 wsem，因此连续读者会使写者长期等待。

解析：R1 先到并锁住 wsem，W1 到达后阻塞；R2、R3 到达时只增加 readcount 即可进入读。只要读者不断到达，readcount 不归零，W1 就不能写。

知识点总结：读者-写者问题；读者优先；写者饥饿；信号量。

### 2. 改成写者优先
答案：增加 readTry 作为读者入口闸门，增加 writecount 统计等待/正在写的写者。第一个写者关闭 readTry，阻止新读者进入；已有读者读完后写者获得资源。

参考伪代码：

~~~text
semaphore resource=1, readTry=1, rmutex=1, wmutex=1;
int readcount=0, writecount=0;
reader: P(readTry); P(rmutex); readcount++; if(readcount==1) P(resource); V(rmutex); V(readTry); read; P(rmutex); readcount--; if(readcount==0) V(resource); V(rmutex);
writer: P(wmutex); writecount++; if(writecount==1) P(readTry); V(wmutex); P(resource); write; V(resource); P(wmutex); writecount--; if(writecount==0) V(readTry); V(wmutex);
~~~

知识点总结：写者优先；入口闸门；互斥信号量；饥饿避免；同步协议设计。
