# 吉林大学《操作系统》2019-2020 学年第二学期期末考试试题详细解析

说明：本卷来自扫描版 PDF“吉林大学《操作系统》期末试题.pdf”。第 1 页为封面，题目位于第 2-3 页。

## 一、论述题

### 1. 中断驱动 I/O 与 DMA 的 CPU 开销
答案：磁盘转速 360rpm=6 转/秒，每磁道 80 扇区，每扇区 512B，传输率为 6*80*512=245760B/s。若每字节一次中断且每次中断 2.5ms，则每秒传输导致 CPU 中断处理时间为 245760*2.5ms=614.4s，占 61440%，说明 CPU 根本无法跟上。若采用 DMA，每扇区一次中断，则每秒中断 6*80=480 次，CPU 时间为 480*2.5ms=1.2s，占 120%。

解析：题面单位写作 ms，按字面计算会得到超过 100% 的比例，含义是该方式不可行。若教材或教师原意为 2.5 微秒，则对应结果分别为 61.44% 和 0.12%。计算时应先求单位时间内传输的字节数或扇区数，再乘每次中断处理时间。

知识点总结：中断驱动 I/O；DMA；磁盘转速；数据传输率；CPU 开销。

### 2. 磁盘调度
题目：当前磁头在 143，刚完成 125，请求序列 86、147、91、177、94、150、102、175、130。由于 125 到 143，SCAN/C-SCAN 默认向柱面号增大方向移动。

答案：FCFS 顺序为 86、147、91、177、94、150、102、175、130，移动总量 565。

解析：移动量为 |143-86|+|86-147|+...+|175-130|=565。

答案：SSTF 顺序为 147、150、130、102、94、91、86、175、177，移动总量 162。

解析：每一步选择离当前磁头最近的请求：143->147->150->130->102->94->91->86->175->177。

答案：SCAN 顺序为 147、150、175、177、130、102、94、91、86。若严格扫描到磁盘末端 199 再返回，移动总量为 169；若按 LOOK 变体在最后一个请求 177 处直接返回，移动总量为 125。

答案：C-SCAN 顺序为 147、150、175、177、86、91、94、102、130。若计入从 199 回卷到 0 的移动，总量为 385；若课程不计回卷移动，则为 186。

知识点总结：FCFS；SSTF；SCAN；C-SCAN；磁头移动量；扫描方向。

### 3. 为什么引入进程？进程管理需要哪些工作？
答案：引入进程是为了描述程序的并发执行，作为资源分配和 CPU 调度的基本对象，并保证并发进程间的合作、同步与系统安全。进程管理包括创建/撤销进程，维护 PCB，管理进程状态转换，进行进程调度，完成同步互斥、通信、死锁处理和资源管理。

解析：程序本身是静态代码，不能表达并发环境中的执行状态、资源占用和调度关系。进程概念把“正在运行的程序”抽象为可管理实体。

知识点总结：进程概念；并发；PCB；进程调度；同步通信；资源管理。

### 4. 段式地址转换
段表：0 段长 15KB、基址 40KB；1 段长 8KB、基址 80KB；2 段长 10KB、基址 100KB。

答案：[0,137] -> 40KB+137=41097B；[1,9000] 越界中断；[2,3600] -> 100KB+3600=106000B；[3,230] 段号不存在，段号越界中断。

解析：段式地址先查段号，再检查段内偏移是否小于段长，最后用段基址加偏移得到物理地址。8KB=8192B，因此 [1,9000] 越界。

知识点总结：段表；段长检查；段基址；越界中断；逻辑地址到物理地址转换。

### 5. 设备分配中为什么可能出现死锁？
答案：多个进程竞争独占设备时，若进程占有一部分设备同时请求其他设备，而已占设备不能被剥夺，就可能形成循环等待，从而死锁。

解析：例如 P1 占有打印机等待磁带机，P2 占有磁带机等待打印机。设备分配若不统一顺序、不预先检查安全性，就可能满足死锁四条件。

知识点总结：设备分配；独占设备；死锁四条件；资源分配图；死锁预防。

## 二、简答题

### 6. 一般计算机的启动过程
答案：上电后 BIOS/UEFI 执行自检，初始化基本硬件；根据启动顺序读取引导扇区或 EFI 启动项；引导程序加载操作系统内核；内核初始化内存管理、进程管理、设备驱动和文件系统；挂载根文件系统，启动 init/systemd 等第一个用户态进程，最后进入登录或图形界面。

知识点总结：BIOS/UEFI；POST；bootloader；内核加载；系统初始化。

### 7. 什么是死锁？原因和必要条件是什么？
答案：死锁是多个进程因竞争资源而互相等待，若无外力干预都无法继续推进的状态。原因包括系统资源不足、资源分配策略不当和进程推进顺序不合适。必要条件是互斥、请求和保持、不剥夺、循环等待。

知识点总结：死锁定义；死锁原因；死锁四条件；死锁处理策略。

### 8. 什么是原语？它与广义指令有什么区别？
答案：原语是操作系统内核中不可分割、不可中断执行的基本操作，常用于进程控制和同步。广义指令通常指系统调用，是操作系统提供给用户程序使用的扩充指令接口。

解析：原语强调内核内部操作的原子性；广义指令强调用户态通过访管/陷入进入内核请求服务。一个系统调用内部可能由多个原语或内核步骤完成。

知识点总结：原语；原子操作；系统调用；用户态与内核态；访管指令。

### 9. CPU 有效利用率排序
答案：由小到大一般为：手工操作 < 单道批处理 < 多道批处理 < 多用户分时系统。

解析：手工操作 CPU 等待人工干预最多；单道批处理减少人工干预但 I/O 时 CPU 仍可能空闲；多道批处理通过多个作业交替运行提高 CPU 利用率；分时系统建立在多道程序基础上，并通过时间片服务多个用户。

知识点总结：操作系统发展；批处理；多道程序；分时系统；CPU 利用率。

### 10. 一般用户最常用的操作系统接口
答案：命令接口，也可包括现代系统中的图形用户接口。若按教材分类，普通用户主要使用命令接口，程序员主要使用程序接口/系统调用。

知识点总结：操作系统接口；命令接口；程序接口；GUI；系统调用。
