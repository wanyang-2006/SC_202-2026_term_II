## 一、2020-2021 学年春季学期《操作系统》期末试题 B 解析

### 第一部分：名词解释

#### 1. 原题：解释 Interrupt（中断）。

参考答案：

中断是指 CPU 在执行程序过程中，由于外部设备、内部异常或软件请求等事件的发生，暂停当前程序的执行，转去执行相应的中断处理程序，处理完毕后再返回被中断程序继续执行的机制。

解析：

答题时抓住三个关键词：事件触发、保存现场、转入处理程序。中断使操作系统能够及时响应 I/O、时钟、异常和系统调用，是多道程序设计和设备管理的重要基础。

#### 2. 原题：解释 Process（进程）。

参考答案：

进程是程序在某个数据集合上的一次运行活动，是系统进行资源分配和调度的基本单位。进程由程序段、数据段和进程控制块 `PCB` 等部分组成，具有动态性、并发性、独立性和异步性。

解析：

程序是静态的指令集合，进程是动态执行过程。考试中常考“进程与程序的区别”，核心就是动态与静态、资源分配实体与代码文件的区别。

#### 3. 原题：解释 Virtual Memory（虚拟内存）。

参考答案：

虚拟内存是在内存和外存的配合下，为进程提供一个比实际物理内存更大、更连续的逻辑地址空间的存储管理技术。它以局部性原理为基础，只把当前需要的页面或段装入内存，其余部分暂存在外存。

解析：

虚拟内存不是简单“扩大内存条”，而是通过地址映射、请求调页/调段、页面置换等机制让用户感觉拥有较大的地址空间。

#### 4. 原题：解释 MMU（内存管理单元）。

参考答案：

`MMU` 是内存管理单元，用于在程序运行时把逻辑地址或虚拟地址转换为物理地址，并配合页表、段表、快表等结构完成地址变换和访问保护。

解析：

`MMU` 常与分页、分段、虚拟存储器一起考。答题时可写出：虚拟页号查页表得到物理块号，再与页内偏移拼接形成物理地址。

#### 5. 原题：解释 Thrashing（抖动）。

参考答案：

抖动是指系统页面置换过于频繁，大量时间耗费在页面调入调出上，而真正执行用户程序的时间很少，导致系统性能急剧下降的现象。

解析：

抖动通常由分配给进程的物理块太少、并发进程过多、工作集不能驻留内存等原因引起。解决方法包括减少并发度、增加物理块、采用工作集或缺页率控制策略。

### 第二部分：简答题

#### 1. 原题：什么是虚拟内存？虚拟内存的最大容量由什么决定？

参考答案：

虚拟内存是把内存和外存统一组织起来，为进程提供较大逻辑地址空间的技术。虚拟内存的最大容量受两方面限制：一是计算机地址结构能够表示的虚拟地址空间大小，二是内存和外存可提供的实际交换/存储能力。考试单选题中如果只给“地址结构”选项，通常选地址结构。

解析：

代表性表述：虚拟存储器的理论最大容量由地址位数决定，例如 32 位地址最多可表示 `2^32` 字节地址空间；实际可用容量还会受外存容量和系统实现限制。

#### 2. 原题：什么是地址空间和存储空间？

参考答案：

地址空间是程序可使用的逻辑地址集合，也称虚拟地址空间；存储空间是主存中实际物理单元的集合，也称物理地址空间。程序访问的是逻辑地址，经过地址变换后才访问物理地址。

解析：

分页系统中，逻辑地址一般由页号和页内地址组成；物理地址一般由块号和块内地址组成。地址空间面向程序，存储空间面向硬件内存。

#### 3. 原题：什么是文件目录和目录文件？

参考答案：

文件目录是文件系统用于记录文件名、文件属性、文件物理位置等信息的数据结构；目录文件是把这些目录项作为文件内容保存起来的一类特殊文件。

解析：

文件目录的主要功能是实现按名存取。目录项通常包括文件名、文件类型、长度、起始位置、权限、时间戳或 inode 号等。

#### 4. 原题：作业调度和进程调度有什么区别？

参考答案：

作业调度又称高级调度，从外存后备作业队列中选择作业调入内存，决定哪些作业获得运行资格；进程调度又称低级调度，从就绪队列中选择一个进程分配 CPU，决定哪个进程立即运行。

解析：

作业调度频率较低，影响系统多道程度；进程调度频率较高，直接影响响应时间和 CPU 利用率。

#### 5. 原题：进程和线程有什么区别？

参考答案：

进程是资源分配的基本单位，拥有独立地址空间和系统资源；线程是 CPU 调度和执行的基本单位，同一进程内的多个线程共享进程的地址空间和资源，但各自拥有线程控制块、程序计数器、寄存器和栈等运行现场。

解析：

线程切换开销通常小于进程切换；线程之间共享数据方便，但也更需要同步和互斥。

### 第三部分：处理机调度计算题

#### 原题

五个作业 `A、B、C、D、E` 几乎同时到达。预计运行时间分别为 `10、6、2、4、8` 分钟，对应优先级分别为 `3、5、2、1、4`，其中 `5` 为最高优先级。忽略进程切换时间，分别计算：

1. 按队列 `A、B、C、D、E` 的先来先服务 `FCFS` 调度下各进程周转时间；
2. 时间片轮转 `RR` 调度下各进程周转时间，时间片为 `2` 分钟。

参考答案：

`FCFS` 下所有作业同时到达，所以周转时间等于完成时间：

| 作业 | 运行时间 | 完成时间/周转时间 |
| --- | ---: | ---: |
| A | 10 | 10 |
| B | 6 | 16 |
| C | 2 | 18 |
| D | 4 | 22 |
| E | 8 | 30 |

`RR` 时间片为 2 分钟，执行序列为：

`A B C D E A B D E A B E A E A`

完成时间为：

| 作业 | 完成时间/周转时间 |
| --- | ---: |
| A | 30 |
| B | 22 |
| C | 6 |
| D | 16 |
| E | 28 |

解析：

所有作业同时到达时，周转时间 `= 完成时间 - 到达时间 = 完成时间`。时间片轮转题的代表性方法是维护剩余运行时间，每轮最多执行一个时间片；剩余时间为 0 时记录完成时刻，不再回到队列。

### 第四部分：银行家算法

#### 原题

系统资源快照如下，资源类型为 `A、B、C、D`。

| 进程 | Allocation A B C D | Need A B C D |
| --- | --- | --- |
| P0 | 0 0 3 2 | 0 0 1 2 |
| P1 | 1 0 0 0 | 1 7 5 0 |
| P2 | 1 3 5 4 | 2 3 5 6 |
| P3 | 0 3 3 2 | 0 6 5 2 |
| P4 | 0 0 1 4 | 0 6 5 6 |

`Available = (1, 6, 2, 2)`。

1. 系统是否处于安全状态？
2. 若进程 `P2` 请求 `(1, 2, 2, 2)`，能否立即分配？

参考答案：

系统处于安全状态。一个安全序列为：

`P0 -> P3 -> P4 -> P1 -> P2`

检查过程：

| 步骤 | Work | 可完成进程 | 完成后 Work |
| --- | --- | --- | --- |
| 初始 | (1,6,2,2) | P0 | (1,6,5,4) |
| 2 | (1,6,5,4) | P3 | (1,9,8,6) |
| 3 | (1,9,8,6) | P4 | (1,9,9,10) |
| 4 | (1,9,9,10) | P1 | (2,9,9,10) |
| 5 | (2,9,9,10) | P2 | (3,12,14,14) |

对 `P2` 的请求 `(1,2,2,2)`：

- 请求量不超过 `P2` 的 `Need = (2,3,5,6)`；
- 请求量也不超过 `Available = (1,6,2,2)`；
- 试分配后 `Available = (0,4,0,0)`，`P2 Need = (1,1,3,4)`。

此时没有任何进程的 `Need` 能小于等于 `(0,4,0,0)`，系统进入不安全状态，因此不能立即分配。

解析：

银行家算法的答题步骤固定为：先判断 `Request <= Need`，再判断 `Request <= Available`，最后试分配并做安全性检查。只满足前两步还不够，必须存在安全序列。

### 第五部分：请求分页地址变换

#### 原题

系统采用请求分页存储管理。主存 `1MB`，分为 `256` 个块，每块 `4KB`。某作业页表如下，状态 `0` 表示在内存，状态 `1` 表示不在内存。

| 页号 | 块号 | 状态 |
| --- | ---: | ---: |
| 0 | 24 | 0 |
| 1 | 26 | 0 |
| 2 | 32 | 0 |
| 3 | - | 1 |
| 4 | - | 1 |

求逻辑地址 `9016` 和 `12300` 的物理地址。

参考答案：

页大小为 `4KB = 4096B`。

逻辑地址 `9016`：

- 页号 `= 9016 // 4096 = 2`；
- 页内位移 `= 9016 mod 4096 = 824`；
- 页 `2` 在内存，块号为 `32`；
- 物理地址 `= 32 * 4096 + 824 = 131896`。

逻辑地址 `12300`：

- 页号 `= 12300 // 4096 = 3`；
- 页内位移 `= 12300 mod 4096 = 12`；
- 页 `3` 不在内存，产生缺页中断，不能直接给出物理地址。

解析：

分页地址变换公式为：`逻辑地址 = 页号 * 页大小 + 页内位移`，`物理地址 = 物理块号 * 块大小 + 块内位移`。如果页表项标志位表示不在内存，应先缺页中断、调页，再重新执行地址变换。

### 第六部分：页面置换算法

#### 原题

在请求分页系统中，页面访问序列为：

`4, 3, 2, 1, 4, 3, 5, 4, 3, 2, 1, 5`

分别在 `4` 个物理块下计算：

1. `FIFO` 缺页序列和缺页率；
2. `LRU` 缺页序列和缺页率。

参考答案：

`FIFO`：

- 缺页序列：`4, 3, 2, 1, 5, 4, 3, 2, 1, 5`
- 缺页次数：`10`
- 缺页率：`10/12 = 83.33%`

`LRU`：

- 缺页序列：`4, 3, 2, 1, 5, 2, 1, 5`
- 缺页次数：`8`
- 缺页率：`8/12 = 66.67%`

解析：

`FIFO` 总是淘汰最早进入内存的页，可能淘汰仍会频繁访问的页；`LRU` 淘汰最近最长时间未被访问的页，更符合局部性原理。代表性方法是画“访问序列-内存块状态-是否缺页”表，逐列更新。

### 第七部分：二选一综合题

#### 1. 原题：文件直接索引与间接索引

某文件系统采用直接索引和间接索引。文件 inode 中有 `10` 个磁盘块指针，前 `8` 个为直接块指针，后 `2` 个分别指向一级间接块和二级间接块。每个指针 `4B`，每块 `2KB`。

1. 该方式下最大文件能占多少字节？不包括 inode 本身占用的块。
2. 一个 `128MB` 文件实际占用多少字节？不包括 inode 本身占用的块。

参考答案：

每个间接索引块可存放指针数：

`2KB / 4B = 2048 / 4 = 512`

最大可寻址数据块数：

`8 + 512 + 512 * 512 = 262664`

最大文件字节数：

`262664 * 2048 = 537935872B`

对 `128MB` 文件，按 `128MiB = 134217728B` 计算：

- 数据块数：`134217728 / 2048 = 65536`；
- 直接块使用 `8` 个；
- 一级间接覆盖 `512` 个数据块；
- 二级间接还需覆盖 `65536 - 8 - 512 = 65016` 个数据块；
- 二级间接下还需 `ceil(65016 / 512) = 128` 个一级索引块；
- 索引块数：一级间接索引块 `1` 个，二级间接根索引块 `1` 个，二级间接下一级索引块 `128` 个，共 `130` 个；
- 实际占用块数：`65536 + 130 = 65666`；
- 实际占用字节数：`65666 * 2048 = 134483968B`。

解析：

文件索引题先算“每个索引块能放多少地址”，再分直接、一级、二级逐层覆盖。实际占用空间通常包括数据块和必要的索引块，不包括 inode 时不要把 inode 块算入。

#### 2. 原题：读者写者问题

若若干进程共享一个数据区，其中一些进程只读，另一些进程更新数据。题中给出经典读者优先同步程序：

```c
semaphore x = 1, wsem = 1;
int readcount = 0;

reader() {
    P(x);
    readcount++;
    if (readcount == 1) P(wsem);
    V(x);

    READUNIT();

    P(x);
    readcount--;
    if (readcount == 0) V(wsem);
    V(x);
}

writer() {
    P(wsem);
    WRITEUNIT();
    V(wsem);
}
```

1. 用并发时间线说明该方案偏向读者。
2. 修改为写者优先。

参考答案：

该方案偏向读者。示例时间线：

1. `R1` 到达，`readcount` 从 `0` 变为 `1`，执行 `P(wsem)`，开始读；
2. `W1` 到达，请求 `P(wsem)`，因 `R1` 正在读而阻塞；
3. `R2` 到达，`readcount` 从 `1` 变为 `2`，不再执行 `P(wsem)`，可直接加入读；
4. 只要不断有新读者到达，`readcount` 就可能一直不为 `0`，`wsem` 无法释放给写者，写者可能饥饿。

写者优先的一种信号量实现：

```c
semaphore rmutex = 1;   // 保护 readcount
semaphore wmutex = 1;   // 保护 writecount
semaphore readTry = 1;  // 写者等待时阻止新读者进入
semaphore resource = 1; // 共享数据区
int readcount = 0;
int writecount = 0;

reader() {
    P(readTry);
    P(rmutex);
    readcount++;
    if (readcount == 1) P(resource);
    V(rmutex);
    V(readTry);

    READUNIT();

    P(rmutex);
    readcount--;
    if (readcount == 0) V(resource);
    V(rmutex);
}

writer() {
    P(wmutex);
    writecount++;
    if (writecount == 1) P(readTry);
    V(wmutex);

    P(resource);
    WRITEUNIT();
    V(resource);

    P(wmutex);
    writecount--;
    if (writecount == 0) V(readTry);
    V(wmutex);
}
```

解析：

读者优先的关键是“已有读者在读时，新读者不用等待写者”；写者优先的关键是“只要有写者等待，新读者先被挡在 readTry 外面”，这样等待中的写者不会被源源不断的新读者饿死。

## 二、2021-2022 学年春季学期《操作系统》期末试题 A 解析

### 第一部分：名词解释

#### 1. 原题：信号量

参考答案：

信号量是用于实现进程同步与互斥的整型变量，只能通过 `P`、`V` 原语访问。`P` 操作表示申请资源或等待条件，`V` 操作表示释放资源或通知条件成立。

解析：

信号量分为整型信号量、记录型信号量、二值信号量和计数信号量。互斥信号量初值通常为 `1`，资源信号量初值通常为可用资源数。

#### 2. 原题：线程

参考答案：

线程是进程中的一个执行流，是 CPU 调度和执行的基本单位。同一进程的线程共享进程地址空间和大部分资源，但各自有栈、寄存器和程序计数器等运行现场。

解析：

线程能提高并发性、降低切换开销，但共享地址空间也会带来同步和安全问题。

#### 3. 原题：地址映射

参考答案：

地址映射是把程序给出的逻辑地址或虚拟地址转换为主存物理地址的过程，通常由硬件 `MMU` 和操作系统页表/段表共同完成。

解析：

分页地址映射查页表，分段地址映射查段表。若页面或段不在内存，需产生缺页或缺段中断。

#### 4. 原题：PCB

参考答案：

`PCB` 是进程控制块，是操作系统为管理进程而设置的数据结构，保存进程标识、状态、优先级、程序计数器、寄存器现场、内存信息、打开文件、资源使用情况等。

解析：

`PCB` 是进程存在的唯一标志。创建进程时创建 `PCB`，撤销进程时撤销 `PCB`。

#### 5. 原题：文件系统

参考答案：

文件系统是操作系统中负责文件命名、存储、组织、访问控制、目录管理和空间管理的子系统。它向用户提供按名存取文件的逻辑接口，并把文件映射到外存物理块。

解析：

文件系统常考目录结构、文件物理结构、空闲空间管理、权限控制和 inode 等内容。

### 第二部分：简答题

#### 1. 原题：阐述产生死锁的必要条件。

参考答案：

产生死锁必须同时满足四个条件：

1. 互斥条件：资源一次只能被一个进程占有；
2. 请求和保持条件：进程已保持部分资源，又请求新的资源；
3. 不剥夺条件：进程已获得的资源不能被强行剥夺，只能主动释放；
4. 循环等待条件：存在进程资源等待环。

解析：

破坏任意一个必要条件都可预防死锁。银行家算法不是死锁预防，而是死锁避免。

#### 2. 原题：叙述中断过程。

参考答案：

中断过程一般包括：

1. 发现中断源并响应中断；
2. 关中断或屏蔽同级中断，保存断点和现场；
3. 根据中断向量找到中断服务程序入口；
4. 执行中断服务程序；
5. 恢复现场，开中断；
6. 返回被中断程序继续执行。

解析：

答题重点是“保存现场”和“恢复现场”。中断必须保证被打断程序之后还能正确继续运行。

#### 3. 原题：静态页面管理与请求页式管理有什么区别？当访问的页不在内存中应如何处理？

参考答案：

静态页面管理通常在作业运行前把所需页面一次性装入或按固定方式分配内存，运行中不主动根据访问情况调入页面；请求页式管理只在页面被访问且不在内存时才调入，能减少一次性装入的内存需求。

当访问页不在内存中时，产生缺页中断。操作系统检查访问是否合法；若合法，则从外存调入该页。如果没有空闲物理块，需要按页面置换算法选择牺牲页，必要时先写回，再把所需页调入，修改页表并重新执行被中断指令。

解析：

请求分页的核心是“按需调入”。缺页中断是可以被系统处理的内部中断，不等同于程序错误。

#### 4. 原题：高速缓存与缓冲区的区别是什么？

参考答案：

高速缓存 `Cache` 主要用于弥补高速部件和低速部件之间的速度差，保存最近可能再次使用的数据副本，以提高访问速度；缓冲区 `Buffer` 主要用于协调数据传输双方速度不匹配、批量传输或异步传输。

解析：

缓存强调“复用”和“命中率”，缓冲强调“平滑速度差”和“临时存放”。例如 CPU Cache 是缓存，I/O 输入输出缓冲区是缓冲。

#### 5. 原题：文件顺序存取与随机存取的主要区别是什么？

参考答案：

顺序存取必须按照文件中记录的先后顺序依次访问，适合磁带或日志类文件；随机存取可以根据记录号、偏移量或索引直接访问文件任意位置，适合磁盘文件和数据库文件。

解析：

顺序存取实现简单，适合批处理；随机存取灵活，适合交互式查询，但需要支持定位和索引结构。

### 第三部分：银行家算法

#### 原题

系统有三类资源 `A、B、C`，总数分别为 `17、5、20`。五个进程状态如下，`Available = (2,3,3)`。

| 进程 | Max A B C | Allocation A B C |
| --- | --- | --- |
| P1 | 5 5 9 | 2 1 2 |
| P2 | 5 3 6 | 4 0 2 |
| P3 | 4 0 11 | 4 0 5 |
| P4 | 4 2 5 | 2 0 4 |
| P5 | 4 2 4 | 3 1 4 |

1. `T0` 时刻系统是否安全？若是，给出安全序列。
2. `T0` 时刻 `P2` 请求 `(0,3,4)`，能否分配？
3. 在上一问基础上，若 `P4` 请求 `(2,0,1)`，能否分配？

参考答案：

先计算 `Need = Max - Allocation`：

| 进程 | Need A B C |
| --- | --- |
| P1 | 3 4 7 |
| P2 | 1 3 4 |
| P3 | 0 0 6 |
| P4 | 2 2 1 |
| P5 | 1 1 0 |

`T0` 安全。一个安全序列为：

`P4 -> P5 -> P1 -> P2 -> P3`

检查过程：

| 步骤 | Work | 可完成进程 | 完成后 Work |
| --- | --- | --- | --- |
| 初始 | (2,3,3) | P4 | (4,3,7) |
| 2 | (4,3,7) | P5 | (7,4,11) |
| 3 | (7,4,11) | P1 | (9,5,13) |
| 4 | (9,5,13) | P2 | (13,5,15) |
| 5 | (13,5,15) | P3 | (17,5,20) |

`P2` 请求 `(0,3,4)` 时，虽然不超过 `Need(P2) = (1,3,4)`，但超过 `Available = (2,3,3)` 中的 `C` 资源可用量，因此不能立即分配。

由于上一问不能分配，系统状态仍保持 `T0`。此时 `P4` 请求 `(2,0,1)`：

- 请求不超过 `Need(P4) = (2,2,1)`；
- 请求不超过 `Available = (2,3,3)`；
- 试分配后 `Available = (0,3,2)`，`Need(P4) = (0,2,0)`；
- 仍存在安全序列：`P4 -> P5 -> P1 -> P2 -> P3`。

因此可以分配给 `P4`。

解析：

本题第二问在资源可用量处已失败，不需要继续做安全性算法。第三问要注意“在上一问基础上”并不表示错误地把 `P2` 分配成功，而是以上一问处理结果为基础；因为 `P2` 请求被拒绝，所以仍从原状态判断 `P4`。

### 第四部分：段式存储地址变换

#### 原题

作业虚拟地址为 `24` 位，高 `8` 位为段号，低 `16` 位为段内相对地址。

1. 一个作业最多可以有多少段？
2. 每段最大长度为多少字节？
3. 段表如下，计算 `[0,430]`、`[1,50]`、`[2,30]`、`[3,70]` 的主存地址；无法变换时说明中断类型。

| 段号 | 段长 | 主存起始地址 | 是否在主存 |
| --- | ---: | ---: | --- |
| 0 | 600 | 2100 | 是 |
| 1 | 40 | 2800 | 是 |
| 2 | 100 | - | 否 |
| 3 | 80 | 4000 | 是 |

参考答案：

1. 段号占 `8` 位，最多 `2^8 = 256` 段。
2. 段内地址占 `16` 位，每段最大长度 `2^16 = 65536B = 64KB`。
3. 地址变换：

| 逻辑地址 | 判断 | 结果 |
| --- | --- | --- |
| [0,430] | 段 0 在内存，430 < 600 | `2100 + 430 = 2530` |
| [1,50] | 段 1 长 40，50 越界 | 地址越界中断 |
| [2,30] | 段 2 不在内存 | 缺段中断 |
| [3,70] | 段 3 在内存，70 < 80 | `4000 + 70 = 4070` |

解析：

分段地址变换先查段号是否合法，再检查段是否在内存，然后检查段内地址是否小于段长，最后用 `段基址 + 段内位移` 得到物理地址。

### 第五部分：LRU 页面置换

#### 原题

页面访问顺序为：

`4, 3, 2, 1, 4, 3, 5, 4, 3, 2, 1, 5`

采用 `LRU` 页面置换算法。

1. 分配 `4` 个存储块时，求缺页序列和缺页率。
2. 分配 `5` 个存储块时，求缺页序列和缺页率。
3. 以上结果说明什么？

参考答案：

`4` 个存储块：

- 缺页序列：`4, 3, 2, 1, 5, 2, 1, 5`
- 缺页次数：`8`
- 缺页率：`8/12 = 66.67%`

`5` 个存储块：

- 缺页序列：`4, 3, 2, 1, 5`
- 缺页次数：`5`
- 缺页率：`5/12 = 41.67%`

说明：

在 `LRU` 算法下，增加分配给进程的物理块数通常不会增加缺页次数。`LRU` 属于栈式算法，不会出现 `FIFO` 中可能发生的 Belady 异常。

解析：

`LRU` 需要在每次访问后更新“最近使用”顺序。淘汰时选择最久没有被访问的页。

### 第六部分：多级索引文件

#### 原题

某文件系统采用多级索引组织文件。`i_node` 中有 `13` 个地址项，其中直接索引项 `10` 项，一级间接、二级间接、三级间接各 `1` 项。数据块大小 `4KB`，磁盘地址 `4B`。

1. 文件系统允许的最大文件长度是多少？
2. 一个 `2G` 文件实际占用多少空间？不包括 `i_node` 占用空间。

参考答案：

每个索引块可存放地址数：

`4KB / 4B = 4096 / 4 = 1024`

最大可寻址数据块数：

`10 + 1024 + 1024^2 + 1024^3 = 1074791434`

最大文件长度：

`1074791434 * 4096 = 4402345713664B`

约为 `4TB` 多一点。

对 `2G` 文件，按 `2GiB` 计算：

- 数据块数：`2 * 1024^3 / 4096 = 524288`；
- 直接索引覆盖 `10` 块；
- 一级间接覆盖 `1024` 块；
- 二级间接还需覆盖 `524288 - 10 - 1024 = 523254` 块；
- 二级间接下需要一级索引块 `ceil(523254 / 1024) = 511` 个；
- 还需要一级间接索引块 `1` 个、二级间接根索引块 `1` 个；
- 三级间接未用到；
- 总块数 `= 524288 + 1 + 1 + 511 = 524801`；
- 实际占用空间 `= 524801 * 4096 = 2149584896B`。

解析：

索引文件的“实际占用空间”不仅包括数据块，还包括索引块。若题目写“不包括 inode”，只是不算 inode 自身，不能漏算一级/二级间接索引块。

### 第七部分：吸烟者问题

#### 原题

用四个进程模拟三个吸烟者和一个代理员。三个吸烟者分别拥有烟纸、烟草、火柴中的一种。代理员每次把另外两种材料放到桌上，能凑齐三种材料的吸烟者取走材料、卷烟、吸烟，然后通知代理员继续放材料。要求补全类 C 程序，并采用计数信号量实现同步。

参考答案：

设三个吸烟者：

- `SmokerA` 自带烟纸，需要烟草和火柴；
- `SmokerB` 自带烟草，需要烟纸和火柴；
- `SmokerC` 自带火柴，需要烟纸和烟草。

可设信号量：

```c
semaphore table = 1;      // 桌子是否空
semaphore needPaper = 0;  // 桌上有烟草和火柴，唤醒有烟纸者
semaphore needTobacco = 0;// 桌上有烟纸和火柴，唤醒有烟草者
semaphore needMatch = 0;  // 桌上有烟纸和烟草，唤醒有火柴者
```

同步程序：

```c
SmokerA() { // 自带烟纸
    while (1) {
        P(needPaper);
        Take_Tobacco_From_Table();
        Take_Matches_From_Table();
        Make_A_Cigarette();
        Smoking();
        V(table);
    }
}

SmokerB() { // 自带烟草
    while (1) {
        P(needTobacco);
        Take_Paper_From_Table();
        Take_Matches_From_Table();
        Make_A_Cigarette();
        Smoking();
        V(table);
    }
}

SmokerC() { // 自带火柴
    while (1) {
        P(needMatch);
        Take_Paper_From_Table();
        Take_Tobacco_From_Table();
        Make_A_Cigarette();
        Smoking();
        V(table);
    }
}

ServerAgent() {
    while (1) {
        P(table);
        ch = RandomNumberFrom_1_2_3();
        if (ch == 1) {
            Put_Tobacco_To_Table();
            Put_Matches_To_Table();
            V(needPaper);
        } else if (ch == 2) {
            Put_Paper_To_Table();
            Put_Matches_To_Table();
            V(needTobacco);
        } else {
            Put_Paper_To_Table();
            Put_Tobacco_To_Table();
            V(needMatch);
        }
    }
}
```

解析：

本题的关键是桌子一次只能放一组材料，所以用 `table` 控制代理员与吸烟者之间的交替；每种材料组合只唤醒唯一一个能吸烟的进程，所以再用三个同步信号量精确唤醒对应吸烟者。

### 第三部分：名词解释

#### 1. 原题：临界区

参考答案：

临界区是进程中访问临界资源的一段代码。多个进程不能同时进入访问同一临界资源的临界区，否则会产生竞态条件。

解析：

临界区管理原则包括空闲让进、忙则等待、有限等待、让权等待。

#### 2. 原题：死锁

参考答案：

死锁是指多个进程因竞争资源而互相等待，若无外力作用将永远不能继续执行的状态。

解析：

答题可补充四个必要条件：互斥、请求保持、不剥夺、循环等待。

#### 3. 原题：虚拟内存

参考答案：

虚拟内存是基于局部性原理，通过请求调页/调段和地址映射，让程序获得较大逻辑地址空间的技术。

解析：

它依赖页表、缺页中断、页面置换和外存交换区。

#### 4. 原题：进程调度

参考答案：

进程调度是操作系统从就绪队列中选择一个进程，把处理机分配给它运行的过程。

解析：

常见算法包括先来先服务、短作业优先、优先级、时间片轮转、多级反馈队列等。

#### 5. 原题：文件描述符

参考答案：

文件描述符是进程打开文件后由操作系统返回的一个整数句柄，用于在后续读、写、关闭等系统调用中标识该打开文件。

解析：

文件描述符对应进程的打开文件表项，而不是文件名本身。

### 第四部分：简答题

#### 1. 原题：描述上下文切换过程，并解释其对性能的影响。

参考答案：

上下文切换是 CPU 从一个进程/线程切换到另一个进程/线程执行的过程。操作系统需要保存当前进程的程序计数器、寄存器、栈指针、状态字等现场到 `PCB/TCB`，再装入下一个进程的现场，切换地址空间或内核数据结构，然后恢复执行。

上下文切换本身不完成用户计算，会带来额外开销；切换过于频繁会降低 CPU 有效利用率，并可能导致缓存、快表失效。

#### 2. 原题：解释时间片轮转调度，并讨论优缺点。

参考答案：

时间片轮转把就绪进程按队列排列，每个进程一次最多运行一个时间片；时间片用完仍未完成，则放回就绪队列尾部。

优点是公平、响应时间较好、适合分时系统；缺点是时间片过小会导致切换频繁，时间片过大则退化为先来先服务。

#### 3. 原题：操作系统如何实现文件权限控制？

参考答案：

操作系统可通过用户和用户组、访问控制位、访问控制表 `ACL`、文件所有者、系统调用权限检查等实现文件权限控制。进程打开或操作文件时，内核根据用户身份和文件权限判断是否允许读、写、执行或删除。

解析：

类 Unix 系统常见权限是所有者、同组用户、其他用户的 `rwx` 权限。

#### 4. 原题：讨论内存管理功能，包括内存分配和内存保护。

参考答案：

内存管理负责为进程分配和回收内存，维护空闲空间，完成地址变换，支持内存共享和虚拟存储。内存分配可采用连续分配、分页、分段或段页式管理。

内存保护通过界限寄存器、页表/段表权限位、用户态/内核态、访问异常等机制，防止进程越界访问或非法修改其他进程和内核空间。

#### 5. 原题：设备驱动程序的作用及其与硬件的交互方式。

参考答案：

设备驱动程序是操作系统中控制具体硬件设备的软件模块，向上为内核提供统一接口，向下通过设备寄存器、控制器命令、中断和 DMA 等机制与硬件交互。

解析：

驱动程序屏蔽设备差异，是实现设备独立性的基础之一。

### 第五部分：应用题

#### 1. 原题：设计一个简单文件系统，包括创建、删除、读取和写入操作，并解释数据结构。

参考答案：

可设计如下数据结构：

- 超级块：记录文件系统总块数、空闲块数、块大小等；
- inode 表：每个 inode 保存文件类型、大小、权限、时间戳、数据块地址；
- 目录项：保存文件名到 inode 号的映射；
- 空闲块位图：记录磁盘块是否空闲；
- 打开文件表：记录当前打开文件的 inode、偏移量、访问模式。

基本操作：

- 创建文件：分配 inode，建立目录项，初始化权限和长度；
- 删除文件：删除目录项，减少链接计数，必要时释放 inode 和数据块；
- 读取文件：按文件偏移找到数据块，读入缓冲区，更新偏移；
- 写入文件：必要时分配新块，把数据写入块，更新文件长度、时间戳和位图。

解析：

文件系统设计题要把“逻辑文件名”如何映射到“磁盘物理块”说清楚。

#### 2. 原题：描述如何实现一个简单进程调度器，并讨论调度策略。

参考答案：

简单调度器可维护一个就绪队列。进程创建或 I/O 完成后进入就绪队列；时钟中断、进程阻塞或进程结束时触发调度；调度器按某种策略选择下一个进程，保存当前进程上下文，恢复新进程上下文。

可选策略：

- `FCFS`：实现简单，但短进程可能等待太久；
- `SJF`：平均等待时间短，但长进程可能饥饿；
- 优先级调度：能体现任务重要性，但低优先级进程可能饥饿；
- 时间片轮转：公平、响应好，但需合理设置时间片；
- 多级反馈队列：综合响应时间与吞吐量，适合通用系统。

### 第六部分：综合题

#### 原题：设计一个带分页机制的 MMU，并解释其如何工作。

参考答案：

一个简单 `MMU` 可包括：

- 页表基址寄存器 `PTBR`：指向当前进程页表；
- 快表 `TLB`：缓存最近使用的页表项；
- 地址分解单元：把虚拟地址分为页号和页内偏移；
- 权限检查单元：检查读、写、执行权限和用户/内核态；
- 缺页异常逻辑：发现页不在内存时触发缺页中断。

工作过程：

1. CPU 给出虚拟地址；
2. `MMU` 分离页号和页内偏移；
3. 先查 `TLB`，命中则得到物理块号；
4. 未命中则根据页表基址查页表；
5. 若页表项有效且权限允许，将物理块号与页内偏移组合成物理地址；
6. 若页不在内存，产生缺页中断，由操作系统调页后重新执行访问。

解析：

分页 `MMU` 题要突出“地址分解、页表查询、权限检查、物理地址形成、缺页处理”五个环节。

### 第五部分：简答题

#### 1. 原题：简述操作系统的定义和作用。

参考答案：

操作系统是配置在计算机硬件上的系统软件，负责管理计算机系统中的处理机、存储器、设备、文件等资源，并为用户和应用程序提供方便、安全、高效的使用接口。

作用包括：作为资源管理者提高资源利用率；作为用户与硬件之间的接口简化使用；作为扩展机器屏蔽硬件细节。

#### 2. 原题：说明进程和线程的区别。

参考答案：

进程是资源分配基本单位，有独立地址空间；线程是 CPU 调度基本单位，同一进程内线程共享地址空间和资源。进程切换开销较大，线程切换开销较小；进程间通信较复杂，线程间共享数据方便但需要同步。

#### 3. 原题：简述分页存储管理和分段存储管理的区别。

参考答案：

分页按固定大小把逻辑地址空间划分为页，把内存划分为同样大小的块，便于离散分配，减少外部碎片；分段按程序逻辑模块划分为大小可变的段，便于共享和保护，但可能产生外部碎片。

地址结构上，分页地址由页号和页内位移组成；分段地址由段号和段内位移组成。

#### 4. 原题：什么是死锁？产生死锁的原因和必要条件是什么？

参考答案：

死锁是多个进程因竞争资源而互相等待，导致都无法继续推进的状态。原因通常是系统资源不足、资源分配策略不当、进程推进顺序不合理。必要条件为互斥、请求和保持、不剥夺、循环等待。

#### 5. 原题：简述 SPOOLing 技术的工作原理和优点。

参考答案：

`SPOOLing` 是假脱机技术，利用磁盘上的输入井和输出井，把独占设备改造成可共享的虚拟设备。用户进程把输出先写入磁盘缓冲区，由后台进程按顺序送往实际设备。

优点是提高独占设备利用率，减少进程等待设备的时间，实现设备共享，并改善系统吞吐量。

### 第六部分：计算题

#### 1. 原题：FIFO 页面置换

某请求分页系统采用 `FIFO`，为进程分配三个物理块，页面走向为：

`1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5`

求缺页次数和缺页率。

参考答案：

按 `FIFO` 模拟：

| 访问页 | 是否缺页 | 内存块状态 |
| ---: | --- | --- |
| 1 | 缺页 | 1 |
| 2 | 缺页 | 1,2 |
| 3 | 缺页 | 1,2,3 |
| 4 | 缺页 | 4,2,3 |
| 1 | 缺页 | 4,1,3 |
| 2 | 缺页 | 4,1,2 |
| 5 | 缺页 | 5,1,2 |
| 1 | 命中 | 5,1,2 |
| 2 | 命中 | 5,1,2 |
| 3 | 缺页 | 5,3,2 |
| 4 | 缺页 | 5,3,4 |
| 5 | 命中 | 5,3,4 |

缺页次数为 `9`，缺页率为 `9/12 = 75%`。

解析：

`FIFO` 淘汰最早进入内存的页，而不是最久未使用的页。

#### 2. 原题：磁盘调度

某磁盘共有 `200` 个柱面，编号 `0-199`。已完成 `68` 号柱面访问，正在为 `96` 号柱面服务，等待请求序列为：

`175, 52, 157, 36, 159, 106, 108, 72`

分别用 `FCFS`、`SSTF`、`SCAN` 确定服务次序，并计算平均寻道长度。

参考答案：

当前磁头在 `96`，刚从 `68` 到 `96`，因此 `SCAN` 当前方向按柱面号增大的方向。

`FCFS`：

- 服务次序：`175 -> 52 -> 157 -> 36 -> 159 -> 106 -> 108 -> 72`
- 移动距离：`79 + 123 + 105 + 121 + 123 + 53 + 2 + 36 = 642`
- 平均寻道长度：`642/8 = 80.25`

`SSTF`：

- 服务次序：`106 -> 108 -> 72 -> 52 -> 36 -> 157 -> 159 -> 175`
- 移动距离：`10 + 2 + 36 + 20 + 16 + 121 + 2 + 16 = 223`
- 平均寻道长度：`223/8 = 27.875`

`SCAN`，按真正扫描到端点 `199` 再折返的口径：

- 服务次序：`106 -> 108 -> 157 -> 159 -> 175 -> 72 -> 52 -> 36`
- 磁头移动：`96 -> 106 -> 108 -> 157 -> 159 -> 175 -> 199 -> 72 -> 52 -> 36`
- 总移动距离：`266`
- 平均寻道长度：`266/8 = 33.25`

若教材采用 `LOOK` 式扫描，即没有请求时不继续扫到端点，则总移动距离为 `218`，平均寻道长度为 `27.25`。考试若明确写 `SCAN` 且给出柱面范围，通常按扫到端点计算。

## 五、扫描版《操作系统》期末试题解析

### 第一部分：论述题

#### 1. 原题：磁盘中断驱动与 DMA

如果磁盘扇区大小为 `512B`，每个磁道有 `80` 个扇区，一共有 `4` 个可用盘面。磁盘转速为 `360rpm`。处理机使用中断驱动方式从磁盘读取数据，每字节产生一次中断；若处理中断需要 `2.5ms`，回答：

1. 处理机花费在处理 I/O 上的时间占整个磁盘访问时间的百分比是多少？忽略寻道时间。
2. 采用 `DMA` 方式，每个扇区产生一次中断，该百分比是多少？

参考答案：

磁盘转速：

`360rpm = 6rps`

每秒可经过的数据量：

`6 * 80 * 512 * 4 = 983040B/s`

中断驱动、每字节一次中断：

- 每秒中断次数：`983040`；
- CPU 处理中断时间：`983040 * 2.5ms = 2457600ms = 2457.6s`；
- 占 1 秒磁盘访问时间百分比：`2457.6 / 1 * 100% = 245760%`。

`DMA`、每扇区一次中断：

- 每秒扇区数：`6 * 80 * 4 = 1920`；
- CPU 处理中断时间：`1920 * 2.5ms = 4800ms = 4.8s`；
- 百分比：`4.8 / 1 * 100% = 480%`。

解析：

该结果超过 `100%`，说明在给定中断处理时间下，频繁中断会使 CPU 完全无法跟上磁盘数据传输，必须采用 DMA、缓冲和批量中断等方式降低 CPU 负担。若按单盘面计算，结果分别为 `61440%` 和 `120%`；但题目给出 `4` 个可用盘面，通常应计入总数据流量。

#### 2. 原题：磁盘调度

磁盘有 `200` 个柱面，编号 `0-199`。当前磁头位于 `143` 号柱面，刚完成 `125` 号柱面的服务请求。等待序列为：

`86, 147, 91, 177, 94, 150, 102, 175, 130`

求 `FCFS`、`SSTF`、`SCAN`、`C-SCAN` 的存取移动顺序和移动总量。

参考答案：

当前磁头由 `125` 到 `143`，方向为柱面号增大方向。

`FCFS`：

- 次序：`86 -> 147 -> 91 -> 177 -> 94 -> 150 -> 102 -> 175 -> 130`
- 总移动量：`57+61+56+86+83+56+48+73+45 = 565`

`SSTF`：

- 次序：`147 -> 150 -> 130 -> 102 -> 94 -> 91 -> 86 -> 175 -> 177`
- 总移动量：`4+3+20+28+8+3+5+89+2 = 162`

`SCAN`，按扫到端点 `199` 再反向：

- 服务次序：`147 -> 150 -> 175 -> 177 -> 130 -> 102 -> 94 -> 91 -> 86`
- 磁头路径：`143 -> 147 -> 150 -> 175 -> 177 -> 199 -> 130 -> 102 -> 94 -> 91 -> 86`
- 总移动量：`169`

`C-SCAN`，按增大方向扫到 `199`，再回到 `0`：

- 服务次序：`147 -> 150 -> 175 -> 177 -> 86 -> 91 -> 94 -> 102 -> 130`
- 磁头路径：`143 -> 147 -> 150 -> 175 -> 177 -> 199 -> 0 -> 86 -> 91 -> 94 -> 102 -> 130`
- 总移动量：`385`

解析：

磁盘调度题要先根据“刚完成的柱面”和“当前柱面”判断扫描方向。本题 `125 -> 143` 表示向上扫描。部分教材把不扫到端点的算法也称扫描，实际更接近 `LOOK/C-LOOK`；若按 `LOOK`，`SCAN` 总移动量为 `125`。

#### 3. 原题：为什么引入进程概念？操作系统在进程管理方面要做哪些工作？

参考答案：

引入进程是为了描述程序的并发执行过程，使操作系统能对并发活动进行管理和控制。程序本身是静态的，不能反映执行状态、资源占有、调度关系和并发制约；进程则能表示程序的一次动态执行。

进程管理的主要工作包括：

- 进程创建和撤销；
- 进程状态转换和调度；
- 进程同步与互斥；
- 进程通信；
- 死锁预防、避免、检测与解除；
- 进程资源分配与回收；
- 维护 `PCB` 和进程队列。

解析：

答题时先说“为什么程序概念不够”，再说“进程管理工作”。代表性关键词是动态性、并发性、资源分配、调度、同步通信、安全。

#### 4. 原题：段式地址转换

某段式存储管理系统段表如下：

| 段号 | 段长 | 段始址 |
| --- | ---: | ---: |
| 0 | 15KB | 40KB |
| 1 | 8KB | 80KB |
| 2 | 10KB | 100KB |

将逻辑地址 `[0,137]`、`[1,9000]`、`[2,3600]`、`[3,230]` 转换成物理地址。

参考答案：

按 `1KB = 1024B`：

| 逻辑地址 | 判断 | 物理地址 |
| --- | --- | --- |
| [0,137] | 段 0 长 15KB，137 未越界 | `40KB + 137 = 40960 + 137 = 41097B` |
| [1,9000] | 段 1 长 8KB = 8192B，9000 越界 | 段内地址越界 |
| [2,3600] | 段 2 长 10KB，3600 未越界 | `100KB + 3600 = 102400 + 3600 = 106000B` |
| [3,230] | 段表无段 3 | 段号越界或段不存在中断 |

解析：

分段地址合法性检查包括段号是否合法和段内地址是否小于段长。只有都合法时才能做 `段基址 + 段内位移`。

#### 5. 原题：设备分配中为什么可能出现死锁？

参考答案：

设备分配中可能出现死锁，是因为多个进程竞争独占设备或有限设备资源时，可能出现“占有一部分设备，同时等待其他设备”的循环等待。例如进程 `P1` 占有打印机等待磁带机，进程 `P2` 占有磁带机等待打印机，双方都不释放已有设备，就形成死锁。

解析：

设备死锁同样满足四个必要条件：互斥、请求和保持、不剥夺、循环等待。解决方法包括静态一次性分配、按序申请设备、银行家算法、安全分配、超时回收或死锁检测后撤销进程。

### 第二部分：简答题

#### 6. 原题：一般计算机的启动过程？

参考答案：

一般启动过程为：

1. 加电或复位后，CPU 从固定地址执行固件程序，如 `BIOS/UEFI`；
2. 固件进行硬件自检和初始化；
3. 按启动顺序查找可启动设备；
4. 读取引导程序或引导管理器到内存；
5. 引导程序装入操作系统内核；
6. 内核初始化内存、进程、设备驱动、文件系统等；
7. 启动系统服务并进入用户登录或桌面环境。

解析：

简答题不用展开到具体厂商实现，写出“固件自检、加载引导程序、加载内核、初始化系统”即可。

#### 7. 原题：何为死锁？产生死锁的原因和必要条件是什么？

参考答案：

死锁是多个进程因竞争资源而互相等待，导致都不能继续执行的状态。产生原因包括资源数量不足、资源分配策略不当、进程推进顺序不合理。

必要条件：

1. 互斥；
2. 请求和保持；
3. 不剥夺；
4. 循环等待。

解析：

这一题与设备死锁题可联动复习。死锁预防就是破坏四个必要条件之一，死锁避免是在分配前判断系统是否会进入不安全状态。

#### 8. 原题：什么是原语？它与广义指令有什么区别？

参考答案：

原语是由若干机器指令组成、完成特定系统功能、执行过程中不可被中断的操作序列，常用于进程同步、进程控制等关键操作，如 `P/V` 原语。

广义指令通常指系统调用，是用户程序请求操作系统服务的接口，如文件读写、进程创建、设备操作等。

区别：

- 原语强调原子性，执行期间不可分割；
- 广义指令强调用户程序进入内核请求服务；
- 原语多用于操作系统内部关键数据结构操作，广义指令是用户态与内核态之间的接口。

解析：

可简单记为：原语是“不可中断的内部基本操作”，广义指令是“系统调用接口”。

#### 9. 原题：将手工操作、单道批处理、多道批处理、多用户分时系统按 CPU 有效利用率由小到大排列。

参考答案：

由小到大一般为：

`手工操作 < 单道批处理 < 多用户分时系统 < 多道批处理`

解析：

手工操作时期 CPU 经常等待人工干预，利用率最低。单道批处理减少人工干预，但一个作业等待 I/O 时 CPU 仍可能空闲。多道批处理通过多个作业在内存中交替运行提高 CPU 利用率。多用户分时系统强调交互响应，频繁切换和等待终端输入会带来一定开销；若教材侧重“多道程序设计基础”，也可能把分时系统与多道批处理都视为较高利用率系统，但按“CPU 有效利用率”多道批处理通常最高。

#### 10. 原题：一般用户最常用的操作系统接口是什么接口？

参考答案：

一般用户最常用的操作系统接口是命令接口或图形用户接口。若题目强调“普通用户直接使用”，可答命令接口和图形界面接口；若强调“程序员在程序中使用”，则答程序接口，即系统调用。

解析：

操作系统接口常分为命令接口、程序接口和图形接口。普通用户日常使用最多的是图形界面或命令界面；应用程序访问操作系统服务靠系统调用。
