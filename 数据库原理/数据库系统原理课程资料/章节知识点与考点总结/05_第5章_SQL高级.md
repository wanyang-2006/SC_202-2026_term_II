# 第5章 SQL 高级

## 章节定位

本章关注 SQL 与程序语言的结合，以及数据库端程序能力，包括 JDBC、ODBC、嵌入式 SQL、函数、存储过程、触发器和递归查询。

## 核心知识点

1. 应用程序访问数据库
   - 应用通过 API 连接数据库服务器、发送 SQL、获取结果、提交或回滚事务。
   - 典型流程：建立连接、创建语句、执行查询或更新、处理结果集、关闭资源。

2. JDBC
   - Java 访问数据库的标准 API。
   - 支持查询、更新、结果集读取、元数据获取。
   - 常见对象：连接、语句、预编译语句、结果集。

3. ODBC
   - 开放数据库连接标准。
   - 提供跨语言、跨数据库的访问接口。
   - 应用通过 ODBC 驱动与数据库通信。

4. 嵌入式 SQL
   - 把 SQL 嵌入宿主语言。
   - 使用宿主变量在程序与 SQL 间传递值。
   - 游标用于逐行处理多元组查询结果。
   - 查询前需声明游标，执行后打开游标，逐条 `fetch`，最后关闭。

5. 函数与存储过程
   - 函数通常有返回值，可用于表达式。
   - 存储过程更强调完成一组操作，可有输入/输出参数。
   - 优点：封装业务逻辑、减少网络交互、便于权限控制和复用。
   - 缺点：可移植性较差，过度使用会增加数据库端负担。

6. 触发器
   - 由事件、条件、动作组成。
   - 触发事件可为 `insert`、`delete`、`update`。
   - 可在语句级或行级触发。
   - 可用于维护派生数据、审计日志、复杂完整性约束。
   - 不宜滥用触发器替代显式逻辑，避免递归触发、调试困难和隐藏副作用。

7. 递归查询
   - SQL 递归常用 `with recursive` 表达。
   - 适合处理传递闭包、层次结构、路径可达性等问题。
   - 固定点计算：不断加入新结果，直到结果不再变化。

## 考点整理

- 复习课件提示 SQL 考点包含函数、过程、触发器。
- 会说明 JDBC/ODBC/嵌入式 SQL 的作用和差别。
- 会理解游标为什么用于多行结果。
- 会写或读懂简单函数、存储过程、触发器。
- 会判断触发器的触发时机、粒度和用途。
- 了解递归查询的基本思想。

## 考点答案与范例

1. JDBC、ODBC、嵌入式 SQL 的区别
   - JDBC 是 Java 访问数据库的标准 API，程序通过连接对象、语句对象和结果集对象执行 SQL。
   - ODBC 是开放数据库连接标准，强调跨语言、跨数据库，通过驱动管理器和数据库驱动访问数据库。
   - 嵌入式 SQL 是把 SQL 语句写入宿主语言程序中，预编译后执行，常用宿主变量和游标传递数据。
   - 范例答案：Java 程序通常用 JDBC；C/C++ 跨数据库访问可用 ODBC；需要在宿主语言中直接嵌入 SQL 时用嵌入式 SQL。

2. 游标的作用与标准回答
   - SQL 查询结果可能有多行，而宿主语言一次通常处理一条记录。
   - 游标用于定位查询结果集中的当前行，并通过 `fetch` 逐行取出。
   - 标准流程：声明游标、打开游标、循环取记录、关闭游标。
   - 范例解释：查询所有选课超过某学分的学生时，结果可能有多名学生，程序需要游标逐条读取 `ID` 和 `name`。

3. 函数与存储过程的区别
   - 函数通常有返回值，可在 SQL 表达式中使用，强调计算一个值。
   - 存储过程是一组数据库操作的封装，可有输入、输出参数，强调完成业务流程。
   - 存储过程可减少网络通信、复用逻辑、集中权限控制，但可移植性较差。
   - 范例：计算教师年薪可写函数；完成选课、扣容量、记录日志的一组操作更适合写存储过程。

4. 存储过程范例
   ```sql
   create procedure raise_salary(in dept varchar(20), in rate numeric(4,2))
   begin
     update instructor
     set salary = salary * (1 + rate)
     where dept_name = dept;
   end;
   ```
   - 答题说明：该过程接收系名和涨薪比例，批量更新指定系教师工资。

5. 触发器标准答案
   - 触发器由触发事件、触发条件和触发动作组成。
   - 触发事件包括 `insert`、`delete`、`update`。
   - 触发粒度包括行级触发和语句级触发。
   - 用途：维护派生数据、审计日志、复杂完整性约束。
   - 范例：向 `takes` 插入及格成绩后，自动更新学生已获学分。

6. 触发器使用边界
   - 适合：必须在数据库层自动维护的一致性规则，如日志、派生字段、跨表约束。
   - 不适合：本可由外键、`check`、普通 SQL 明确表达的规则；复杂业务流程全部隐藏在触发器中；可能产生递归触发的操作。
   - 答题关键词：隐藏副作用、调试困难、可移植性差、可能影响性能。

7. 递归查询范例
   - 递归查询用于传递闭包，例如求所有直接或间接先修课。
   - 思路：先给出基础查询，再反复根据已有结果扩展，直到没有新结果。
   - 答题中即使不要求完整 SQL，也应说明“基础部分 + 递归部分 + 固定点停止”。

## 常见题型与答题抓手

- “程序如何读取多行查询结果”：答游标或结果集。
- “自动维护统计字段”：可用触发器，但要说明触发事件和动作。
- “封装一组数据库操作”：使用存储过程。
- “求先修课传递关系/组织上下级”：使用递归查询。
