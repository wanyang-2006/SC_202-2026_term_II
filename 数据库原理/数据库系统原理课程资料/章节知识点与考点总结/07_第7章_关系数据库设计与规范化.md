# 第7章 关系数据库设计与规范化

## 章节定位

本章是理论和计算题重点，围绕函数依赖、属性闭包、候选键、正则覆盖、无损连接、依赖保持、BCNF、3NF、多值依赖和分解算法展开。

## 核心知识点

1. 好的关系模式设计
   - 目标：减少冗余，避免插入异常、删除异常、更新异常。
   - 坏设计常由不恰当合并模式造成。
   - 分解需要兼顾无损连接和依赖保持。

2. 第一范式 1NF
   - 属性域必须是原子的。
   - 非原子域包括集合、复合值、可再拆分编码等。

3. 函数依赖
   - `α -> β` 表示任意两个元组若在 `α` 上相同，则在 `β` 上也相同。
   - 平凡函数依赖：`β ⊆ α`。
   - 函数依赖用于刻画业务规则和判断模式质量。

4. 函数依赖闭包
   - `F+` 是由 `F` 能推出的所有函数依赖。
   - Armstrong 公理：自反律、增广律、传递律。
   - 常用规则：合并、分解、伪传递。

5. 属性闭包
   - `α+` 是在 `F` 下由属性集 `α` 能推出的全部属性。
   - 用途：
     - 判断 `α` 是否为超键。
     - 判断函数依赖 `α -> β` 是否成立。
     - 求候选键。
     - 辅助 BCNF/3NF 判定。

6. 候选键计算
   - 必须出现在键中的属性：不能由其他属性推出且必须覆盖全关系的属性。
   - 通过属性闭包判断是否能推出全部属性。
   - 候选键必须最小，去掉任一属性后不再是超键。

7. 正则覆盖
   - 目标：得到与原依赖集等价、右部单属性、无多余属性、无多余依赖的依赖集。
   - 多余属性：删除后依赖集闭包不变。
   - 多余依赖：删除后依赖集闭包不变。
   - 3NF 分解常基于正则覆盖。

8. 无损连接分解
   - 分解后自然连接能恢复原关系，不产生伪元组。
   - 二元分解 `R1, R2` 对 `R` 无损的常用判定：
     - `(R1 ∩ R2) -> R1` 或 `(R1 ∩ R2) -> R2` 属于 `F+`。

9. 依赖保持
   - 分解后不用连接各子关系就能检查原函数依赖。
   - BCNF 分解可能不保持依赖。
   - 3NF 分解可做到无损连接且保持依赖。

10. BCNF
   - 对每个非平凡函数依赖 `α -> β`，`α` 必须是超键。
   - 优点：冗余更少。
   - 缺点：可能不保持依赖。
   - BCNF 分解：找到违反 BCNF 的依赖 `α -> β`，分解为 `α ∪ β` 和 `R - (β - α)`，递归处理。

11. 3NF
   - 对每个非平凡函数依赖 `α -> β`，至少满足：
     - `α` 是超键；或
     - `β - α` 中每个属性都是某个候选键的主属性。
   - 3NF 允许一定冗余以保持依赖。
   - 3NF 分解：求正则覆盖，为每个依赖建关系，必要时加入包含候选键的关系，删除被包含的冗余关系。

12. 多值依赖
   - `α ->-> β` 表示在给定 `α` 时，`β` 与其余属性独立组合。
   - 多值依赖用于进一步规范化到 4NF。
   - 本课程复习提示要求知道多值依赖概念。

13. 去规范化
   - 出于性能考虑，有时故意保留冗余。
   - 需要权衡查询效率、更新代价和一致性维护成本。

## 考点答案与范例

1. 函数依赖判断答案
   - `X -> Y` 的含义是：在任意合法关系实例中，若两个元组的 `X` 属性值相同，则它们的 `Y` 属性值也必须相同。
   - 判断时不能只看当前表中碰巧成立的数据，还要看业务语义是否必然成立。
   - 范例：若学号唯一确定姓名，则 `ID -> name` 成立；姓名通常不能唯一确定学号，所以 `name -> ID` 不一定成立。

2. 属性闭包计算范例
   - 已知 `R(A,B,C,D,E)`，`F={A->BC, CD->E, B->D, E->A}`，求 `A+`。
   - 初始：`A+ = {A}`。
   - 用 `A->BC`，得 `{A,B,C}`。
   - 用 `B->D`，得 `{A,B,C,D}`。
   - 用 `CD->E`，因已有 C、D，得 `{A,B,C,D,E}`。
   - 所以 `A+ = ABCDE`，`A` 是超键。若没有真子集也是超键，则 `A` 是候选键。

3. 候选键计算答案
   - 先找不出现在任何依赖右部的属性，这些属性通常必须包含在候选键中。
   - 再通过属性闭包试探能否推出全部属性。
   - 最后检查最小性，删除任一属性后不能再推出全部属性。
   - 答题必须写闭包过程，不能只写结果。

4. Armstrong 公理和推理范例
   - 自反律：若 `Y ⊆ X`，则 `X -> Y`。
   - 增广律：若 `X -> Y`，则 `XZ -> YZ`。
   - 传递律：若 `X -> Y` 且 `Y -> Z`，则 `X -> Z`。
   - 常用推论：合并律、分解律、伪传递律。
   - 范例：由 `A->B` 和 `B->C`，用传递律可得 `A->C`；由 `A->B` 和 `A->C`，用合并律可得 `A->BC`。

5. 正则覆盖答案
   - 目标是把依赖集化简为等价、右部单属性、无多余属性、无多余依赖的形式。
   - 步骤：右部分解为单属性；检查左部是否有多余属性；检查右部或整条依赖是否多余；可合并同左部依赖。
   - 检查多余属性时，核心方法是比较删除前后的闭包是否仍能推出相同依赖。
   - 3NF 分解通常先求正则覆盖，再按依赖建关系。

6. 无损连接证明范例
   - 对二元分解 `R` 分为 `R1`、`R2`，若 `(R1 ∩ R2) -> R1` 或 `(R1 ∩ R2) -> R2` 属于 `F+`，则分解无损。
   - 范例：`R(A,B,C)` 分解为 `R1(A,B)` 和 `R2(B,C)`，若 `B -> A` 或 `B -> C` 成立，则无损。
   - 如果公共属性不能决定任一子模式，则自然连接可能产生伪元组，是有损分解。

7. 依赖保持判断答案
   - 依赖保持要求原依赖集中的约束可通过各子关系上的依赖检查完成，不需要把子关系连接起来。
   - 方法：把原依赖投影到各子模式，求这些投影依赖的闭包，看能否推出原依赖集。
   - 结论：3NF 分解算法可保证无损且保持依赖；BCNF 分解保证无损，但不一定保持依赖。

8. BCNF 判定与分解范例
   - BCNF 条件：每个非平凡函数依赖 `X -> Y` 的左部 `X` 都必须是超键。
   - 若发现 `X -> Y` 违反 BCNF，则分解为：
     ```text
     R1 = X ∪ Y
     R2 = R - (Y - X)
     ```
   - 范例：`R(A,B,C)`，`F={A->B, B->C}`。`A+ = ABC`，A 是键；`B+ = BC`，B 不是超键，所以 `B->C` 违反 BCNF。分解为 `R1(B,C)` 和 `R2(A,B)`。

9. 3NF 判定与分解范例
   - 3NF 条件：对每个非平凡依赖 `X -> A`，至少满足 `X` 是超键，或 `A` 是主属性。
   - 主属性是出现在某个候选键中的属性。
   - 3NF 分解步骤：求正则覆盖；对每个 `X->A` 建关系 `XA`；若没有任何关系包含候选键，则补一个候选键关系；删除被其他关系包含的关系。
   - 3NF 相比 BCNF 的优势是能保持依赖，代价是可能保留少量冗余。

10. 多值依赖答案
   - `X ->-> Y` 表示在给定 `X` 时，`Y` 的取值集合与其余属性取值集合相互独立。
   - 范例：某课程有多个教师、多个教材，教师和教材彼此独立，则可能有 `course ->-> teacher` 和 `course ->-> book`。
   - 多值依赖会造成组合冗余，是 4NF 讨论的基础。本课程一般要求理解概念，不一定要求复杂分解。

## 常见题型与答题抓手

- 求属性闭包：从给定属性开始，反复应用左部已包含的依赖，直到不再增加属性。
- 求候选键：先找必须包含属性，再试探最小属性组合。
- 判 BCNF：检查每个非平凡依赖左部是否为超键。
- 判 3NF：若左部不是超键，看右部属性是否为主属性。
- BCNF 分解：每次用一个违反依赖分解，最后检查每个子模式。
- 3NF 分解：先求正则覆盖，再按依赖建表，补候选键，删包含表。
- 无损连接：二元分解用公共属性能否推出某一子模式。
- 依赖保持：把依赖投影到各子模式，闭包能否推出原依赖。
