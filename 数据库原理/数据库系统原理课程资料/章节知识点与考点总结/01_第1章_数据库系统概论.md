# 第1章 数据库系统概论

## 章节定位

本章是全课程的概念入口，重点理解数据库系统为什么产生、DBMS 的作用、数据抽象层次、数据模型、数据库语言、数据库体系结构，以及存储、查询、事务等系统组件的基本分工。

## 核心知识点

1. 数据库系统基本概念

   - 数据库：长期存储在计算机内、有组织、可共享的数据集合。
   - DBMS：管理数据库的软件，提供数据定义、查询更新、事务管理、安全控制、并发控制、恢复等功能。
   - 数据库系统：数据库、DBMS、应用程序、用户、数据库管理员和运行环境的整体。
   - 数据独立性：应用程序与数据存储细节解耦，分为物理数据独立性和逻辑数据独立性。
2. ***文件系统的局限***

   - 数据冗余和不一致。
   - 数据访问困难，新增需求常需重新编程。
   - 数据孤立，格式不统一。
   - 完整性约束难维护。
   - 原子性、并发访问、安全控制和故障恢复能力不足。
3. 数据管理的发展阶段

   - 人工管理阶段：数据不保存或保存能力弱，程序与数据高度耦合。
   - 文件系统阶段：文件可长期保存，但共享性差、冗余高。
   - 数据库系统阶段：统一管理、共享性高、冗余低、独立性强。
4. 数据抽象与模式

   - 物理层：描述数据实际如何存储。
   - 逻辑层：描述存储哪些数据以及数据间关系。
   - 视图层：面向不同用户的局部数据表示。
   - 实例：某一时刻数据库中的实际数据。
   - 模式：数据库的总体逻辑结构。
5. 数据模型

   - 层次模型、网状模型、关系模型、对象关系模型、XML、NoSQL 等。
   - 关系模型用表表示数据和联系，是后续关系代数、SQL、规范化的基础。
   - NoSQL 常见类型包括键值、文档、列族、图数据库，强调扩展性、灵活模式和高可用。
6. 数据库语言

   - DDL：定义数据库模式和约束，如 `create table`。
   - DML：查询和更新数据，如 `select`、`insert`、`delete`、`update`。
   - SQL：集 DDL、DML、DCL 等于一体的关系数据库标准语言。
7. 数据库系统组件

   - 存储管理器：管理文件、索引、缓冲区、页面、记录。
   - 查询处理器：解析、优化、执行查询。
   - 事务管理器：保证 ACID、并发控制和故障恢复。
   - DBA：负责模式定义、权限、安全、备份恢复、性能调优。
8. 数据库体系结构与应用

   - 集中式、客户机/服务器、分布式数据库、云托管数据库、云原生数据库。
   - OLTP 偏事务处理，OLAP 偏分析查询，HTAP 兼顾事务与分析。

## 考点答案与范例

1. DB、DBMS、数据库系统、模式、实例、数据独立性的标准回答
   - 数据库 DB：按一定数据模型组织、长期存储、可被多个用户共享的数据集合。例：学校的学生、课程、选课、成绩数据集合。
   - DBMS：数据库管理系统，是位于用户和数据库之间的软件系统，负责定义、存储、查询、更新、并发控制、恢复、安全和完整性控制。
   - 数据库系统 DBS：由数据库、DBMS、应用程序、用户、数据库管理员以及硬件/软件环境组成的整体。
   - 模式 schema：数据库的逻辑结构和约束的描述，较稳定。例如 `student(ID, name, dept_name, tot_cred)`。
   - 实例 instance：某一时刻数据库中的实际数据，随插入、删除、更新而变化。
   - 数据独立性：应用程序不需要因数据组织方式变化而频繁修改。物理数据独立性指存储结构变化不影响逻辑模式和应用；逻辑数据独立性指逻辑模式适度变化不影响外模式和应用。

2. 文件系统与数据库系统比较题答案
   - 文件系统的主要问题是数据冗余和不一致、数据共享困难、数据孤立、访问不灵活、完整性约束难保证、原子性和并发控制不足、安全性和恢复机制弱。
   - 数据库系统通过统一的数据模型、统一管理、共享访问、完整性约束、事务机制、权限控制和恢复机制解决上述问题。
   - 范例答案：文件系统中学生基本信息可能在学籍科、房产科、劳资科多份保存，导致冗余和不一致；数据库系统把公共数据集中管理，各应用通过 DBMS 访问，从而减少冗余并提高一致性。

3. 三级抽象及作用
   - 物理层回答“数据怎样存”：文件、页、记录、索引、磁盘布局。
   - 逻辑层回答“存什么数据以及数据间有什么关系”：表、属性、联系、约束。
   - 视图层回答“不同用户看到什么”：不同用户只看到与自己相关的数据视图。
   - 作用：隐藏复杂性，保护数据，支持数据独立性。考试答题时可写“物理层隐藏存储细节，逻辑层描述整体结构，视图层提供局部表示”。

4. DDL、DML、DCL 的区别
   - DDL 数据定义语言：定义和修改数据库对象，如 `create table`、`alter table`、`drop table`。
   - DML 数据操纵语言：查询和更新数据，如 `select`、`insert`、`delete`、`update`。
   - DCL 数据控制语言：授权和回收权限，如 `grant`、`revoke`。
   - 范例：建学生表用 DDL，查询学生成绩用 DML，允许某用户查询学生表用 DCL。

5. DBMS 组件分工
   - 存储管理器负责磁盘空间、文件组织、缓冲区、索引和记录管理。
   - 查询处理器负责 SQL 解析、关系代数转换、查询优化和执行。
   - 事务管理器负责并发控制和恢复，保证 ACID。
   - 权限和完整性管理模块负责安全控制和约束检查。
   - 范例答案：执行 `select` 时，查询处理器生成执行计划，存储管理器读取页和索引；若执行更新，事务管理器还要保证并发正确性和故障恢复。

6. 基本概念题答题模板
   - 名词解释用“定义 + 作用 + 例子”。
   - 简答比较题用“问题/特点 + 原因 + 后果 + 数据库系统如何解决”。
   - 如问“数据库系统的特点”，可答：数据结构化、共享性高、冗余低、独立性高、由 DBMS 统一管理和控制，支持安全性、完整性、并发控制和恢复。

## 常见题型与答题抓手

- 名词解释：用“定义 + 作用 + 例子”回答。
- 简答：文件系统缺点、数据库系统特点、数据独立性的意义。
- 判断/选择：模式与实例、逻辑层与物理层、DDL 与 DML 的区别。
