# 第17章 事务

## 章节定位

本章是事务处理基础，重点是事务概念、ACID、事务状态、调度、冲突可串行化、视图可串行化、可恢复调度、无级联调度和 SQL 隔离级别。

## 核心知识点

1. 事务概念
   - 事务是访问并可能更新数据库的程序执行单元。
   - 典型例子：账户转账，必须保证扣款和加款作为整体完成。

2. ACID 特性
   - Atomicity 原子性：事务操作要么全做，要么全不做。
   - Consistency 一致性：事务单独执行应保持数据库一致性。
   - Isolation 隔离性：并发事务之间不应看到彼此的中间状态。
   - Durability 持久性：事务提交后结果应永久保存，即使系统故障也不丢失。

3. 事务状态
   - Active：事务正在执行。
   - Partially committed：最后一条语句已执行，等待提交。
   - Failed：事务无法继续正常执行。
   - Aborted：事务回滚，数据库恢复到事务开始前状态。
   - Committed：事务成功完成，结果永久写入。

4. 调度
   - 调度表示多个事务中读写操作的执行顺序。
   - 串行调度：事务一个接一个执行。
   - 并发调度：不同事务操作交错执行。
   - 正确并发执行的目标是效果等价于某个串行调度。

5. 冲突与冲突可串行化
   - 两个操作冲突需满足：属于不同事务、访问同一数据项、至少一个是写操作。
   - 冲突类型：`read-write`、`write-read`、`write-write`。
   - 通过交换不冲突操作可变成串行调度，则为冲突可串行化。

6. 优先图
   - 顶点表示事务。
   - 若事务 Ti 的操作与 Tj 的操作冲突且 Ti 先于 Tj，则加边 Ti -> Tj。
   - 调度冲突可串行化当且仅当优先图无环。
   - 若无环，拓扑序给出等价串行顺序。

7. 可恢复性
   - 可恢复调度：若 Tj 读了 Ti 写的数据，则 Tj 必须在 Ti 提交后才能提交。
   - 级联回滚：一个事务回滚导致读过其未提交数据的事务也必须回滚。
   - 无级联调度：事务只能读取已提交数据，避免级联回滚。

8. 隔离性实现与隔离级别
   - 并发异常：丢失修改、脏读、不可重复读、幻读。
   - SQL 隔离级别：Read Uncommitted、Read Committed、Repeatable Read、Serializable。
   - 隔离级别越高，并发异常越少，但并发性能可能越低。

9. SQL 中事务定义
   - `commit` 提交。
   - `rollback` 回滚。
   - 可设置事务隔离级别。

10. 视图可串行化
   - 比冲突可串行化更宽松。
   - 关注读从关系、最终写关系是否与某个串行调度相同。
   - 判定更复杂，课程重点通常是理解概念和能处理简单例子。

## 考点整理

- 复习课件提示：事务概念、模型、ACID、冲突/视图可串行化判别、事务隔离性和原子性、可恢复调度、无级联调度都是重点。
- 会画优先图并判断冲突可串行化。
- 会找冲突操作。
- 会判断调度是否可恢复、是否无级联。
- 会说明 SQL 隔离级别能避免哪些异常。

## 考点答案与范例

1. 事务和 ACID 标准答案
   - 事务是数据库操作的逻辑工作单位，要么完整执行，要么不执行。
   - 原子性：转账中扣 A 和加 B 必须作为整体完成，不能只扣不加。
   - 一致性：转账前后 A+B 总额不变。
   - 隔离性：并发事务不能看到转账中间状态。
   - 持久性：转账提交后，即使系统崩溃，结果也不能丢失。

2. 事务状态题答案
   - Active：事务正在执行读写操作。
   - Partially committed：最后一条语句执行完，但提交尚未最终完成。
   - Committed：提交成功，结果持久化。
   - Failed：发现错误，不能继续正常执行。
   - Aborted：事务回滚，数据库恢复到事务开始前状态。
   - 常见状态转换：Active -> Partially committed -> Committed；Active/Partially committed -> Failed -> Aborted。

3. 冲突操作判定答案
   - 两个操作冲突必须同时满足：不同事务、同一数据项、至少一个写。
   - `read_i(X)` 与 `read_j(X)` 不冲突。
   - `read_i(X)` 与 `write_j(X)` 冲突。
   - `write_i(X)` 与 `read_j(X)` 冲突。
   - `write_i(X)` 与 `write_j(X)` 冲突。

4. 冲突可串行化判定范例
   - 调度：
     ```text
     r1(A), w1(A), r2(A), w2(A), r1(B), w1(B), r2(B), w2(B)
     ```
   - A 上 `w1(A)` 在 `r2(A)、w2(A)` 前，得边 `T1 -> T2`。
   - B 上 `w1(B)` 也在 `r2(B)、w2(B)` 前，仍是 `T1 -> T2`。
   - 优先图无环，所以冲突可串行化，等价串行顺序为 `T1, T2`。

5. 优先图有环范例
   - 若 A 上产生 `T1 -> T2`，B 上又产生 `T2 -> T1`，优先图有环。
   - 有环说明不能通过交换非冲突操作变成串行调度，所以不是冲突可串行化。
   - 答题时必须写出边的来源，不能只画结论。

6. 可恢复调度答案
   - 若 `Tj` 读了 `Ti` 写的数据，则 `Tj` 必须在 `Ti` 提交之后才能提交。
   - 若 `Tj` 在 `Ti` 提交前提交，而之后 `Ti` 回滚，则 `Tj` 已提交了基于脏数据的结果，调度不可恢复。
   - 判断方法：找所有“读了谁写的数据”的关系，再检查提交顺序。

7. 无级联调度答案
   - 无级联要求事务只能读取已提交事务写入的数据。
   - 如果 `Tj` 读取了 `Ti` 未提交写入的数据，则可能因 `Ti` 回滚导致 `Tj` 也回滚，形成级联回滚。
   - 无级联调度一定是可恢复调度，但可恢复调度不一定无级联。

8. 隔离级别和异常答案
   - Read Uncommitted：可能脏读、不可重复读、幻读。
   - Read Committed：避免脏读，但可能不可重复读、幻读。
   - Repeatable Read：避免脏读和不可重复读，但标准意义下可能幻读。
   - Serializable：效果等价于串行执行，避免上述异常。
   - 丢失修改：两个事务同时更新同一数据，后写覆盖前写。
   - 脏读：读到未提交数据。
   - 不可重复读：同一事务两次读同一行结果不同。
   - 幻读：同一事务两次按条件查询，出现新增或消失的满足条件行。

9. 视图可串行化答案
   - 视图等价要求读从关系一致、最终写一致。
   - 视图可串行化比冲突可串行化更宽松，所有冲突可串行化调度都是视图可串行化，但反过来不一定。
   - 含盲写的调度可能视图可串行化但非冲突可串行化。

## 常见题型与答题抓手

- 判断冲突可串行化：列冲突，画优先图，看有无环。
- 判断可恢复：看读脏数据的事务是否在被读事务提交后再提交。
- 判断无级联：看是否读取未提交数据。
- 隔离异常题：先识别读写顺序，再判断是脏读、不可重复读、幻读还是丢失修改。
