# 第4章 SQL 中级

## 章节定位

本章继续扩展 SQL，重点是连接表达式、视图、事务语句、完整性约束、SQL 类型与模式、索引、授权，以及关系代数与 SQL 的转换。

## 核心知识点

1. 连接表达式
   - `inner join ... on ...`：按连接条件匹配。
   - `natural join`：按同名属性自动连接。
   - `left outer join`：保留左表不匹配元组。
   - `right outer join`：保留右表不匹配元组。
   - `full outer join`：保留两侧不匹配元组。
   - 外连接用 `null` 填充缺失一侧的属性。

2. 连接写法
   - 逗号连接加 `where`。
   - 显式 `join ... on ...`。
   - 自然连接 `natural join`。
   - 考试中应明确连接条件，避免自然连接误按无关同名属性连接。

3. 视图
   - `create view v as <query>` 定义虚关系。
   - 视图可隐藏复杂查询、提供逻辑独立性、限制用户访问范围。
   - 视图展开：查询视图时可替换为其定义查询。
   - 物化视图存储查询结果，需要维护刷新。

4. 视图更新
   - 简单单表视图较可能可更新。
   - 含聚集、分组、去重、集合运算、连接、表达式列的视图通常不可直接更新或更新含义不唯一。
   - 可更新视图的核心问题是：视图上的修改能否唯一翻译成基表修改。

5. 事务
   - SQL 中事务由一组语句组成。
   - `commit work` 提交，`rollback work` 回滚。
   - 更完整的事务性质在第 17 章展开。

6. 完整性约束
   - 单关系约束：`not null`、`unique`、`check`。
   - 参照完整性：`foreign key ... references ...`。
   - 级联动作：`on delete cascade`、`on update cascade`，以及置空、拒绝等策略。
   - 复杂约束可用断言或触发器表达。

7. SQL 数据类型、模式与索引
   - 内置类型、用户自定义类型、域、LOB。
   - `create index` 用于建立索引，加速查询但会增加更新维护成本。
   - B+ 树索引是数据库常见索引结构。

8. 授权
   - 权限包括 `select`、`insert`、`update`、`delete`、`references`、`usage` 等。
   - `grant` 授权，`revoke` 回收权限。
   - `with grant option` 允许被授权者继续授权。
   - 角色 `role` 便于统一管理权限。
   - 视图可用于行列级访问控制。

## 考点整理

- 会写内连接、外连接，并说明外连接为什么引入 `null`。
- 会判断视图是否可更新，说明不能更新的原因。
- 会写完整性约束，尤其主键、外键、`check`、级联。
- 会写授权和回收权限语句。
- 会进行关系代数与 SQL 的相互转换。

## 考点答案与范例

1. 内连接、自然连接、外连接答案
   - 内连接只保留满足连接条件的匹配元组：
     ```sql
     select course.course_id, title, prereq_id
     from course inner join prereq
       on course.course_id = prereq.course_id;
     ```
   - 自然连接自动按同名属性连接，若存在语义无关的同名属性，可能产生错误结果。
   - 左外连接保留左表所有元组：
     ```sql
     select course.course_id, title, prereq_id
     from course left outer join prereq
       on course.course_id = prereq.course_id;
     ```
   - 若某课程没有先修课，左外连接仍保留该课程，`prereq_id` 为 `null`。

2. 三种连接写法的等价范例
   - 逗号连接：
     ```sql
     select name, title
     from instructor, teaches, course
     where instructor.ID = teaches.ID
       and teaches.course_id = course.course_id;
     ```
   - 显式连接：
     ```sql
     select name, title
     from instructor join teaches on instructor.ID = teaches.ID
       join course on teaches.course_id = course.course_id;
     ```
   - 若同名属性语义一致，可用自然连接，但考试中更推荐写显式 `on` 条件，答案更稳。

3. 视图定义与作用答案
   - 视图是由查询定义的虚关系，本身通常不直接存储数据。
     ```sql
     create view faculty as
     select ID, name, dept_name
     from instructor;
     ```
   - 作用：简化复杂查询、提供逻辑数据独立性、隐藏敏感列、限制用户访问范围。
   - 查询视图时，DBMS 可把视图名展开为其定义查询。

4. 可更新视图判断答案
   - 可更新的关键是：对视图的插入、删除、更新能否唯一翻译成对基表的修改。
   - 通常可更新：基于单个表、没有聚集、没有 `group by`、没有 `distinct`、没有集合运算、没有复杂表达式列，并保留必要键属性的简单视图。
   - 通常不可更新：连接视图、聚集视图、分组视图、去重视图、集合运算视图、只含表达式而无法定位基表列的视图。
   - 范例：`select ID, name from student` 较可能可更新；`select dept_name, avg(salary) from instructor group by dept_name` 不可更新，因为无法把平均工资修改唯一映射到某个教师工资。

5. 完整性约束范例
   - 唯一和非空：
     ```sql
     name varchar(20) not null,
     email varchar(50) unique
     ```
   - 检查约束：
     ```sql
     check (salary > 0)
     ```
   - 外键和级联：
     ```sql
     foreign key (dept_name) references department(dept_name)
       on delete set null
       on update cascade
     ```
   - 答题解释：删除被引用系时，学生或教师中的 `dept_name` 可按策略置空、级联删除或拒绝删除，具体取决于约束定义。

6. 授权语句范例
   - 授权：
     ```sql
     grant select on student to user1;
     grant update(salary) on instructor to user2;
     ```
   - 带转授权：
     ```sql
     grant select on course to user1 with grant option;
     ```
   - 回收权限：
     ```sql
     revoke select on student from user1;
     ```
   - 角色：先给角色授权，再把角色授予用户，便于批量管理。

7. 关系代数与 SQL 转换范例
   - 关系代数：
     ```text
     πname(σdept_name='Comp. Sci.'(instructor))
     ```
   - SQL：
     ```sql
     select name
     from instructor
     where dept_name = 'Comp. Sci.';
     ```
   - 转换要点：投影对应 `select`，选择对应 `where`，连接对应 `from/join/on`。

## 常见题型与答题抓手

- “保留未匹配记录”：使用外连接。
- “只允许用户看部分列/行”：定义视图并授权视图。
- “删除父表记录对子表影响”：看外键的级联/置空/拒绝策略。
- “视图更新是否合法”：检查是否存在唯一对应的基表元组和属性。
