# 第3章 SQL 基础

## 章节定位

本章从 SQL 的数据定义和基本查询开始，逐步覆盖集合运算、空值、聚集、分组、嵌套子查询以及数据库修改，是考试中最容易出编程题和查询题的部分。

## 核心知识点

1. DDL 与表定义
   - `create table` 定义关系名、属性名、数据类型和完整性约束。
   - 常见类型：`char(n)`、`varchar(n)`、`int`、`smallint`、`numeric(p,d)`、`real`、`double precision`、日期时间类型等。
   - 常见约束：`not null`、`primary key`、`foreign key ... references ...`。
   - `drop table` 删除表，`alter table` 修改表结构。

2. 基本查询结构
   - 基本形式：`select ... from ... where ...`。
   - `select` 指定输出属性，可使用表达式和别名。
   - `from` 指定查询来源，多个表默认形成笛卡尔积。
   - `where` 指定筛选条件。
   - SQL 默认保留重复行，去重需用 `distinct`。
   - 排序用 `order by`，可指定升序/降序。

3. 字符串与谓词
   - `like` 进行模式匹配。
   - `%` 匹配任意长度字符串，`_` 匹配单个字符。
   - `between`、比较运算符、逻辑运算 `and/or/not`。

4. 集合运算
   - `union`、`intersect`、`except` 默认去重。
   - `union all`、`intersect all`、`except all` 保留重复。
   - 参与集合运算的查询结果需列数和类型相容。

5. 空值与三值逻辑
   - `null` 参与比较的结果通常是 `unknown`。
   - SQL 条件判断存在 `true`、`false`、`unknown`。
   - 判断空值必须用 `is null` 或 `is not null`，不能用 `= null`。
   - 聚集函数通常忽略 `null`，但 `count(*)` 统计元组数。

6. 聚集与分组
   - 聚集函数：`avg`、`min`、`max`、`sum`、`count`。
   - `group by` 按属性分组。
   - `having` 筛选分组，`where` 筛选分组前的元组。
   - `select` 中非聚集属性通常必须出现在 `group by` 中。

7. 嵌套子查询
   - `in` / `not in`：集合成员测试。
   - `some` / `all`：集合比较。
   - `exists` / `not exists`：判断子查询结果是否为空。
   - `not exists` 常用于表达“所有”或“不存在反例”。
   - 子查询可出现在 `where`、`from`、`select` 中。
   - `with` 子句可定义临时关系，增强复杂查询可读性。

8. 数据库修改
   - `delete from ... where ...` 删除元组。
   - `insert into ... values ...` 或插入查询结果。
   - `update ... set ... where ...` 更新属性值。
   - `case when ... then ... else ... end` 可实现条件更新。

## 考点答案与范例

1. DDL 建表与约束范例
   - 建表题要写清属性、类型、主键、外键和非空约束。
     ```sql
     create table student (
       ID varchar(5),
       name varchar(20) not null,
       dept_name varchar(20),
       tot_cred numeric(3,0),
       primary key (ID),
       foreign key (dept_name) references department(dept_name)
     );
     ```
   - 主键自动要求唯一且非空；外键要求引用值在被引用表中存在，或按具体 DBMS 规则允许为空。

2. 基本查询题答案模板
   - SQL 查询优先按“输出列、数据来源、连接条件、筛选条件、分组条件、分组筛选、排序”组织。
     ```sql
     select name
     from instructor
     where dept_name = 'Comp. Sci.' and salary > 80000;
     ```
   - 多表查询必须写连接条件，否则会产生笛卡尔积。

3. 集合运算准确答案
   - `union` 求并集，默认去重；`union all` 保留重复。
   - `intersect` 求交集；`except` 求差集。
   - 两边查询结果必须列数相同、对应列类型相容。
   - 范例：查询既当教师又当学生的姓名：
     ```sql
     select name from instructor
     intersect
     select name from student;
     ```

4. 空值与三值逻辑答案
   - `null` 表示未知或缺失，不等于 0，也不等于空字符串。
   - 与 `null` 的普通比较结果通常为 `unknown`，不能写 `salary = null`。
   - 正确写法：
     ```sql
     where salary is null
     where salary is not null
     ```
   - `where` 只保留条件为 `true` 的元组，`false` 和 `unknown` 都不会保留。
   - 聚集函数除 `count(*)` 外通常忽略 `null`。

5. 聚集函数、`group by`、`having` 范例
   - 查询每个系的平均工资：
     ```sql
     select dept_name, avg(salary) as avg_salary
     from instructor
     group by dept_name;
     ```
   - 查询平均工资大于 70000 的系：
     ```sql
     select dept_name, avg(salary) as avg_salary
     from instructor
     group by dept_name
     having avg(salary) > 70000;
     ```
   - `where` 在分组前筛选行，`having` 在分组后筛选组。凡是涉及聚集结果的条件，一般放在 `having`。

6. 嵌套子查询与 `not exists` 标准范例
   - `in` 用于判断某值是否属于子查询结果：
     ```sql
     select name
     from student
     where ID in (select ID from takes where course_id = 'CS-101');
     ```
   - “选修所有课程”的经典双重 `not exists`：
     ```sql
     select S.ID, S.name
     from student as S
     where not exists (
       select *
       from course as C
       where not exists (
         select *
         from takes as T
         where T.ID = S.ID and T.course_id = C.course_id
       )
     );
     ```
   - 含义：不存在一门课程 C，使得该学生没有选 C。

7. 数据修改范例
   - 删除某系所有教师：
     ```sql
     delete from instructor
     where dept_name = 'Finance';
     ```
   - 插入一名学生：
     ```sql
     insert into student(ID, name, dept_name, tot_cred)
     values ('12345', 'Zhang', 'Comp. Sci.', 0);
     ```
   - 给工资低于 70000 的教师涨薪 5%：
     ```sql
     update instructor
     set salary = salary * 1.05
     where salary < 70000;
     ```

## 常见题型与答题抓手

- “每个系/每门课/每个学生的数量或平均值”：`group by` + 聚集函数。
- “分组后满足条件”：`having`。
- “选修了所有课程”：双重 `not exists`。
- “没有选某课/没有记录”：`not exists` 或 `not in`，但含 `null` 时优先 `not exists`。
- “排名/排序输出”：`order by`。
