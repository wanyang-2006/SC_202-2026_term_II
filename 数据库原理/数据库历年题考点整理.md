# 数据库历年题知识点与考点整理

资料来源：`数据库系统原理/数据库系统原理资料/数据库历年题.pdf`。该 PDF 为扫描版，以下为逐页阅读后按章节归纳的考点。

## 总体高频结论

- 高频主线：E-R 建模与关系模式转换、关系代数与 SQL、函数依赖与范式、事务并发控制。
- 高频操作题：画 E-R 图、转换为关系模式并标主键/外键、写 SQL/关系代数、求闭包/候选码/正则覆盖、判断 3NF/BCNF 并分解、判断冲突可串行化并画优先图。
- 中频考点：视图、授权、完整性约束、触发器/存储过程/函数、索引与散列、查询优化。
- 低频但出现过：RAID、数据仓库/数据挖掘、信息安全模型、SeaView、多级安全数据库、数据清洗、决策树信息增益。

## 第 1 章 数据库系统概述

### 概念与名词解释

- 数据、信息、数据库、数据库系统 DBS、数据库管理系统 DBMS。
- 数据库系统组成：数据库、DBMS、应用系统、用户/DBA。
- 数据库模式与实例：schema 是逻辑结构，instance 是某时刻数据库内容。
- 数据模型：概念数据模型、逻辑数据模型、物理数据模型；层次模型、网状模型、关系模型、E-R 模型。
- 数据独立性：物理数据独立性、逻辑数据独立性。
- 数据字典：集中保存数据库对象定义、模式、约束、权限等元数据。
- 数据库系统优点：减少冗余、共享、统一管理、数据独立性、安全性和完整性、并发控制、恢复。
- 文件系统与数据库系统比较：冗余、不一致、访问困难、完整性和安全控制弱、并发和恢复困难。

### 常考问法

- DB、DBMS、DBS 三者关系。
- 数据库系统基本功能：定义、操纵、控制、组织存储、维护。
- 数据库设计阶段：需求分析、概念设计、逻辑设计、物理设计、实施、运行维护。
- B/S 与 C/S 模式特点。

## 第 2 章 关系模型与关系代数

### 基本概念

- 关系、元组、属性、域、关系模式、关系实例。
- 主键、候选键、超键、外键、参照关系、被参照关系。
- 1NF：属性域原子性。
- 实体完整性、参照完整性、用户定义完整性。
- 自然连接、等值连接、外连接、内连接、除法。

### 关系代数

- 基本运算：选择 sigma、投影 pi、并、差、笛卡尔积、重命名。
- 扩展运算：交、连接、自然连接、外连接、除法、聚集。
- 关系代数表达式与元组关系演算/域关系演算之间的转换。
- 关系代数等价变换：选择串接、投影下推、连接选择分配等。
- 关系代数表达式证明：如 R 与 S 无相同属性时，关系连接/笛卡尔积相关结论。

### 常考题型

- 给定小关系表，计算投影、选择、连接、笛卡尔积、差、交、除法结果。
- 把自然语言查询翻译为关系代数。
- 把关系代数翻译为 SQL。
- 判断表达式是否等价。
- “查询所有满足全部条件的对象”常用除法或 `NOT EXISTS`。

## 第 3 章 SQL

### DDL 与约束

- DDL、DML、DCL 的含义和区别。
- `CREATE TABLE`、`ALTER TABLE`、`DROP TABLE`。
- 数据类型：`char` 与 `varchar` 区别，`number`、`date` 等。
- 约束：`PRIMARY KEY`、`FOREIGN KEY`、`NOT NULL`、`UNIQUE`、`CHECK`、`DEFAULT`。
- MySQL 外键与标准 SQL 的差异曾被问到。

### 查询

- `SELECT-FROM-WHERE` 基本结构和逻辑执行顺序。
- `DISTINCT` 去重。
- 模糊查询 `LIKE`，范围查询，空值 `NULL` 判断。
- 多表连接、自然连接、外连接。
- 分组与聚集：`COUNT`、`SUM`、`AVG`、`MAX`、`MIN`、`GROUP BY`、`HAVING`。
- 排序：`ORDER BY ASC/DESC`。
- 子查询：标量子查询、集合子查询、相关子查询、`IN`、`NOT IN`、`EXISTS`、`NOT EXISTS`、`ALL`、`ANY/SOME`。
- 集合运算：`UNION`、`INTERSECT`、`EXCEPT`；MySQL 不支持 `INTERSECT` 时用连接或 `EXISTS` 替代。

### 更新与权限

- `INSERT`、`UPDATE`、`DELETE`。
- `GRANT` 授权，常考 `SELECT`、`UPDATE`、`DELETE`、`INSERT` 权限。
- 视图：`CREATE VIEW`、基于视图查询、视图更新限制。

### 过程化 SQL

- 存储过程：按时间、范围、统计需求计算年度/期间销售额、购买次数等。
- 函数：按参数计算受欢迎程度、销售旺季等。
- 触发器：插入/更新前后检查空值、业务规则、票据有效性、数量上限下限等。

### 常考应用场景

- 银行数据库：branch/customer/loan/account/borrower/depositor。
- 大学数据库：department/course/instructor/section/teaches/student/takes。
- 商品/订单/图书/音乐/游戏/仓储/保险/票务系统。
- 常见要求：创建表、查询、更新、删除、建视图、授权、触发器、存储过程。

## 第 4 章 E-R 模型与数据库设计

### E-R 基本元素

- 实体、属性、联系、实体集、联系集。
- 实体集与属性、实体集与联系集的区别。
- 强实体集、弱实体集、标识性联系、部分键。
- 单值/多值属性、复合属性、派生属性。
- 联系类型：1:1、1:n、m:n。
- 角色名、参与约束、基数约束。
- ISA/泛化/特化，`disjoint` 与 `overlap`，`total` 与 `partial`。

### E-R 到关系模式转换

- 强实体集转换为关系。
- 弱实体集转换：加入所属强实体主键，组合成主键。
- 1:1、1:n 联系合并规则。
- m:n 联系单独建表，主键通常为参与实体主键组合。
- 多值属性单独建表。
- ISA 继承结构的转换策略。
- 转换后标注主键、外键和完整性约束。

## 第 5 章 关系数据库规范化

### 函数依赖基础

- 函数依赖、平凡/非平凡函数依赖、完全/部分函数依赖、传递依赖。
- Armstrong 公理：自反律、增广律、传递律。
- 属性集闭包 X+。
- 函数依赖集闭包 F+。
- 正则覆盖/最小依赖集。
- 候选码、超码、主属性、非主属性。

### 范式

- 1NF：属性不可再分。
- 2NF：消除非主属性对候选码的部分依赖。
- 3NF：对每个非平凡依赖 X -> A，X 是超码，或 A 为主属性。
- BCNF：对每个非平凡依赖 X -> Y，X 必须是超码。
- 3NF 与 BCNF 的区别：BCNF 更强；BCNF 分解可能不保持依赖，3NF 分解通常可保持依赖。

### 分解

- 无损连接分解判断。
- 保持函数依赖判断。
- 3NF 分解算法。
- BCNF 分解算法。
- 判断分解后模式是否合理：无损连接、保持依赖。

### 高频题型

- 给 R(U,F)，求某个属性集闭包。
- 求所有候选码。
- 求正则覆盖。
- 判断 R 是否满足 3NF/BCNF，并说明理由。
- 若不满足，分解到 3NF 或 BCNF。
- 证明 BCNF 蕴含 3NF。

## 第 6 章 存储、文件组织与索引

### 文件与索引概念

- 顺序文件、散列文件。
- 主索引、辅助索引、聚集索引、非聚集索引。
- 稠密索引、稀疏索引。
- 索引的优点和代价：查询快，维护和空间开销增加。
- 同一关系上能否建多个查找键的主索引：通常不能，因为物理顺序只能按一个查找键组织。

### B 树与 B+ 树

- B+ 树定义、节点结构、叶子节点链表、范围查询优势。
- B+ 树插入、删除、节点分裂、合并、借位。
- B 树/2-3 树插入删除。
- 判断 B+ 树叶子深度是否一致。

### 散列与位图索引

- 静态散列、可扩充散列、桶、目录、全局深度、局部深度。
- 散列溢出处理。
- 位图索引适合低基数属性；位图检索过程。

### 其他

- RAID 定义与作用。
- 易失/非易失存储。
- 物理存储、缓冲区、日志文件、后援副本。

## 第 7 章 查询处理与查询优化

### 查询处理

- 查询处理步骤：解析与翻译、优化、执行。
- 执行计划表达式。
- 表达式执行方式：物化、流水线。
- 连接算法：嵌套循环、索引嵌套循环、排序归并、哈希连接。

### 查询优化

- 代价估计：表大小、选择率、连接大小。
- 优化目标：在等价计划中选择代价最低者。
- 关系代数等价规则用于优化。
- 选择下推、投影下推、连接顺序优化。
- 给定执行计划图，写出对应关系代数表达式。

## 第 8 章 事务管理

### 事务概念

- 事务定义：访问并可能更新数据项的程序执行单元。
- ACID：原子性、一致性、隔离性、持久性。
- 事务状态：Active、Partially committed、Failed、Aborted、Committed。
- 引入事务的原因：保证并发、故障环境下的数据正确性。
- 非正常结束对数据库正确性的影响。

### 并发异常

- 丢失修改。
- 脏读/脏数据。
- 不可重复读。
- 幻读。
- 不一致分析。

### 可串行化

- 调度、串行调度、并发调度。
- 冲突操作：不同事务、同一数据项、至少一个写。
- 冲突等价、冲突可串行化。
- 优先图/前驱图：有环则不可冲突可串行化；无环则可串行化，拓扑序给出等价串行次序。
- 视图可串行化及其与冲突可串行化的关系。

### 封锁协议

- 共享锁 S、排他锁 X。
- 锁相容矩阵。
- 两阶段封锁协议 2PL：增长阶段、收缩阶段。
- 严格两阶段封锁、强两阶段封锁。
- 三级封锁协议。
- 封锁能保证并发正确性，但可能产生死锁。
- 死锁判断：等待图有环。

### 恢复

- 日志、检查点、后援副本。
- 故障类型：事务故障、系统故障、介质故障等。

## 第 9 章 完整性、安全性与权限

### 完整性

- 实体完整性、参照完整性、用户定义完整性。
- 完整性约束在 SQL 中的定义。
- 触发器用于复杂业务完整性。

### 安全性

- 数据库安全目标：保密性、完整性、可用性。
- 用户标识与鉴别。
- 存取控制：自主访问控制 DAC、强制访问控制 MAC。
- 角色、授权、回收权限。
- 视图机制实现安全保护。
- 审计追踪、日志审计。
- 数据加密。
- SQL 注入攻击及防范：参数化、输入验证、修补漏洞。
- 隐蔽通道、推理控制。
- 多级安全模型：Bell-LaPadula、Biba、Clark-Wilson、SeaView。
- 访问控制三要素：主体、客体、访问控制策略/权限。

## 第 10 章 数据仓库与数据挖掘

### 数据仓库

- 数据仓库定义与特点：面向主题、集成、相对稳定、反映历史变化。

### 数据挖掘

- 数据挖掘过程：数据选择、预处理、模式发现、解释评估。
- 数据清洗方法：填补缺失数据、平滑噪声、去除孤立点、解决不一致。
- 数据挖掘基本方法：分类、聚类、关联规则、时间序列。
- 关联规则：支持度、置信度。
- 应用：预测、寻找关联。
- 决策树：划分纯度、信息增益、信息增益率、信息内容。

## 复习优先级建议

1. 必须熟练：SQL、关系代数、E-R 转关系模式、函数依赖与范式、冲突可串行化。
2. 必须会写步骤：闭包、候选码、3NF/BCNF 分解、优先图、`NOT EXISTS` 表达“所有/没有”。
3. 概念题重点背：DB/DBMS/DBS、数据独立性、视图、事务 ACID、锁、索引类型、B+ 树、数据安全机制。
4. 可作为提分项：触发器、存储过程、函数、授权、查询优化、可扩充散列、位图索引、数据挖掘。
