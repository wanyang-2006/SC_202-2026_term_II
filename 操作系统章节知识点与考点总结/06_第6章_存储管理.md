# 第6章 存储管理

> 整理依据：`课件/操作系统原理课件 2026/06第六章 存储管理.pptx`。

## 章节定位

本章讲主存如何分配、保护、共享和地址转换，是虚拟存储的前置基础。重点掌握内存分配算法、碎片、界地址管理、覆盖、分页、分段、段页式管理、多级页表、反置页表和快表 TLB。考试常考地址转换、有效访问时间和分页分段比较。

## 核心知识点

1. ==存储管理功能==
   - 内存分配与回收：为进程分配主存空间，结束后回收。
   - 地址转换：把逻辑地址转换为物理地址。
   - 存储保护：防止越界访问和非法访问。
   - 存储共享：多个进程共享代码段、库或共享内存。
   - 存储扩充：通过覆盖、交换、虚拟存储等技术扩大可运行程序规模。

2. 内存资源管理
   - 位示图：用位表示页面或块是否空闲，适合固定大小块管理。
   - 空闲页面表：记录空闲页框号。
   - 空闲页面链：把空闲块链接起来，分配回收方便。
   - 动态异长分区：按进程需要分配连续可变大小分区。

3. 动态分区分配算法

| 算法 | 思想 | 优点 | 缺点 |
| --- | --- | --- | --- |
| First Fit | 从低地址开始找第一个足够大的空闲区 | 快，保留高地址大块 | 低地址碎片多 |
| Next Fit | 从上次位置继续找 | 分布较均匀 | 可能破坏大空闲区 |
| Best Fit | 找最小但足够大的空闲区 | 单次剩余最小 | 易产生难以利用的小碎片 |
| Worst Fit | 找最大的空闲区 | 剩余块仍较大 | 大块消耗快 |

4. 碎片处理
   - 内部碎片：分配给进程但进程用不到的空间，如固定分区或分页最后一页。
   - 外部碎片：空闲空间总量足够但不连续，如动态分区。
   - 紧凑：移动进程使空闲区合并，需要动态重定位支持。

5. 界地址管理
   - 通过基址寄存器和限长寄存器实现连续分配下的地址转换与保护。
   - 逻辑地址 d 合法条件：0 <= d < limit；物理地址 = base + d。
   - 越界访问触发地址越界中断。

6. 覆盖技术
   - 把一个程序划分为若干功能模块，互斥执行的模块不同时装入内存。
   - 优点：可在较小内存中运行较大程序。
   - 缺点：需要程序员或编译系统设计覆盖结构，管理复杂。

7. 分页式存储管理
   - 逻辑地址空间划分为页，物理内存划分为同样大小的页框。
   - 逻辑地址 (p,d)：p 为页号，d 为页内位移。
   - 页表记录页号到页框号的映射，物理地址为 (f,d)。
   - 分页消除外部碎片，但可能有内部碎片。
   - 快表 TLB 缓存页表项，加速地址转换。

8. 多级页表和反置页表
   - 多级页表把页表分页，只为实际使用的页表页分配内存，适合大地址空间。
   - 反置页表按物理页框建立表项，每个页框一个表项，减少页表空间，但查找需要散列或辅助结构。

9. 分段式存储管理
   - 逻辑地址按程序自然单位划分为段，如代码段、数据段、栈段。
   - 逻辑地址 (s,d)：s 为段号，d 为段内位移。
   - 段表项记录段长、段基址和保护信息。
   - 分段便于共享和保护，但会产生外部碎片。

10. 段页式存储管理
   - 先按逻辑意义分段，再把每段分页。
   - 逻辑地址 (s,p,d)，先查段表得到该段页表，再查页表得到页框号。
   - 兼有分段的逻辑保护共享和分页的离散分配优点，但地址转换层次更复杂。

## 考点答案与范例

1. 页式与段式比较

| 项目 | 分页 | 分段 |
| --- | --- | --- |
| 划分依据 | 固定大小 | 程序逻辑单位 |
| 地址形式 | 页号 + 页内位移 | 段号 + 段内位移 |
| 碎片 | 内部碎片 | 外部碎片 |
| 共享保护 | 不如分段直观 | 按段共享和保护自然 |
| 用户可见性 | 通常用户不可见 | 用户可感知逻辑段 |

2. 分页地址转换例子
   - 页大小 1KB，逻辑地址 2500。
   - 页号 p = 2500 / 1024 = 2，页内位移 d = 452。
   - 若页表中页 2 对应页框 8，则物理地址 = 8 * 1024 + 452 = 8644。

3. 分段地址转换例子
   - 段表项：段 1 基址 4000，段长 1000。
   - 逻辑地址 (1,600) 合法，物理地址 = 4600。
   - 逻辑地址 (1,1200) 越界，因为 1200 >= 1000。

4. TLB 有效访问时间
   - 设访问内存时间为 m，TLB 查找时间为 t，命中率为 h。
   - 命中：t + m。
   - 未命中：t + 2m，因为先访问页表再访问数据。
   - EAT = h(t+m) + (1-h)(t+2m)。

5. 动态分区算法例子
   - 空闲区大小为 100、500、200、300，申请 180。
   - First Fit 选择 500。
   - Best Fit 选择 200。
   - Worst Fit 选择 500。
   - Next Fit 从上次位置继续找，选择从当前位置之后遇到的第一个足够大分区。

## 常见题型与答题抓手

- “地址转换题”：先拆逻辑地址，再查表，最后检查越界或有效位。
- “比较题”：分页重物理分配，分段重逻辑结构。
- “碎片题”：固定大小分配多内部碎片，连续可变分区多外部碎片。
- “TLB 题”：明确命中是否仍需访问一次内存取数据。
