## 第1章 操作系统概述

> 整理依据：`课件/操作系统原理课件 2026/01第一章 操作系统概述.pptx`。  
> 复习方式：先掌握定义、特征和类型，再把运行环境中的硬件机制与后续章节联系起来。

### 章节定位

本章回答“操作系统是什么、为什么需要操作系统、操作系统依赖哪些硬件机制、用户怎样使用操作系统”。重点理解操作系统既是资源管理者，也是用户与硬件之间的接口，还把复杂硬件抽象成进程、文件、虚拟内存和设备等易用对象。

### 核心知识点

1. ==操作系统定义==
   - 操作系统是管理计算机硬件与软件资源、控制程序执行、改善人机界面并为用户和应用程序提供服务的一组系统软件。
   - 向下管理处理机、存储器、I/O 设备和文件；向上提供命令、图形界面和系统调用。
   - 例子：用户执行 `read` 读文件时，并不直接操作磁盘，而是通过系统调用进入内核，由文件系统、设备驱动和磁盘调度协同完成。

2. 操作系统的作用
   - 处理机管理：进程创建、撤销、调度、同步和通信。
   - 存储管理：内存分配、地址转换、存储保护、虚拟存储。
   - 设备管理：设备分配、驱动、中断处理、缓冲、磁盘调度。
   - 文件管理：文件组织、目录、共享、保护和外存空间管理。
   - 用户接口：命令解释程序、系统调用、GUI、作业控制语言。

3. 操作系统的发展
   - 手工操作阶段：人工装入程序，CPU 等待时间长。
   - 批处理阶段：批量连续处理作业，减少人工干预。
   - 多道程序阶段：多个作业同时进入内存，通过并发提高 CPU 与设备利用率。
   - 分时系统阶段：多个用户交互使用系统，强调响应时间。
   - 实时、网络、分布式、嵌入式、多处理机系统：面向不同场景扩展。

4. 操作系统四大特征
   - 并发：多个程序在同一时间段内交替推进。
   - 共享：多个进程共同使用系统资源，包括互斥共享和同时共享。
   - 虚拟：把一个物理资源抽象为多个逻辑资源，如虚拟处理机、虚拟内存。
   - 异步：进程推进速度不可预知，执行顺序受调度和外部事件影响。

5. 操作系统类型

| 类型 | 主要目标 | 典型特点 | 例子 |
| --- | --- | --- | --- |
| 多道批处理系统 | 提高吞吐量和资源利用率 | 交互性差，周转时间较长 | 离线工资结算 |
| 分时系统 | 多用户交互与短响应 | 时间片轮转，用户感觉独占机器 | 多终端系统 |
| 实时系统 | 在截止时间前响应 | 硬实时不能错过时限，软实时允许偶尔延迟 | 工业控制、订票系统 |
| 网络操作系统 | 网络资源共享 | 远程访问、网络服务 | 文件服务器 |
| 分布式操作系统 | 多机统一管理 | 位置、迁移、复制透明 | 分布式计算平台 |
| 嵌入式操作系统 | 可靠、实时、资源受限 | 面向专用设备 | 智能终端、控制器 |

6. 操作系统运行环境
   - 定时装置：产生时钟中断，支持时间片和抢占调度。
   - 系统栈：处理中断、陷入和系统调用时保存现场。
   - 寄存器：保存程序计数器、状态字、基址限长等执行状态。
   - 特权指令与非特权指令：I/O、关中断、修改状态字等敏感操作必须由内核执行。
   - 用户态与核心态：用户程序在用户态运行，系统调用、中断或异常使 CPU 进入核心态。
   - 地址映射和存储保护：把逻辑地址映射为物理地址并阻止越界访问。
   - 中断、DMA、通道：让外设与 CPU 并行工作，提高 I/O 效率。

7. 操作系统界面
   - 命令接口：用户输入命令，由 shell 或命令解释程序执行。
   - 程序接口：应用通过系统调用请求内核服务。
   - 图形界面：以窗口、菜单、鼠标等方式提供交互。

### 考点整理

- 会从“资源管理者、控制程序、扩展机器”三个角度解释操作系统。
- 会比较批处理、分时和实时系统的目标与适用场景。
- 会说明并发、共享、虚拟、异步四大特征，其中并发和共享是最基本特征。
- 会区分用户态和核心态、特权指令和非特权指令。
- 会解释中断、系统调用、DMA、通道在操作系统运行中的作用。

### 考点答案与范例

1. 四大特征辨析

| 特征 | 定义 | 例子 | 易错点 |
| --- | --- | --- | --- |
| 并发 | 多个事件在同一时间段内发生 | 单核 CPU 上多个进程轮流运行 | 并发不等于并行 |
| 共享 | 多个进程共同使用资源 | 多进程共享打印机、文件 | 互斥共享需要同步 |
| 虚拟 | 把物理资源变成逻辑资源 | 每个进程拥有独立虚拟地址空间 | 虚拟依赖硬件和 OS 配合 |
| 异步 | 进程推进速度不可预知 | I/O 完成时间不固定 | 异步要求 OS 保证结果正确 |

2. 用户态和核心态题
   - 用户态：普通程序运行状态，不能执行特权指令。
   - 核心态：操作系统内核运行状态，可以执行全部指令。
   - 转换：用户程序执行系统调用、发生中断或异常时进入核心态；内核处理完毕后恢复用户态。

3. DMA 与通道题
   - DMA：由 DMA 控制器在内存和设备之间直接传输数据，CPU 只负责初始化和完成中断处理。
   - 通道：更高级的 I/O 处理部件，可执行通道程序，适合复杂 I/O 系统。

### 常见题型与答题抓手

- “为什么需要操作系统”：从提高效率、方便用户、管理资源、提供抽象四点答。
- “并发与并行区别”：并发是时间段内交替推进，并行是同一时刻真正同时执行。
- “系统调用为什么要陷入内核”：因为用户程序不能直接执行特权指令。
- “实时系统和分时系统区别”：实时重截止期，分时重响应时间和交互体验。

## 第2章 进程、线程与作业

> 整理依据：`课件/操作系统原理课件 2026/02第二章 进程线程与作业1.pptx`。

### 章节定位

本章是进程管理的基础。重点掌握多道程序设计为什么引入进程，进程的定义、状态、PCB、上下文、队列、创建撤销，以及线程和作业的概念。考试常把进程、程序、线程、作业放在一起比较，也常考状态转换图和 fork 类程序的进程数。

### 核心知识点

1. ==多道程序设计==
   - 单道程序设计中，程序等待 I/O 时 CPU 空闲，系统利用率低。
   - 多道程序设计把多个作业同时装入内存，当一个程序等待 I/O 时切换到另一个程序运行。
   - 目标：提高 CPU 利用率、提高吞吐量、改善资源利用。
   - 新问题：执行失去封闭性和可再现性，需要进程、同步、互斥、调度和保护。

2. 进程的概念
   - 进程是程序在某个数据集合上的一次执行过程，是资源分配和调度的基本单位。
   - 进程由程序段、数据段和 PCB 组成。
   - 进程具有动态性、并发性、独立性、异步性和结构性。
   - 例子：同一个编辑器程序打开多个窗口，程序文件是静态的，运行起来后才形成进程。

3. 进程状态及状态转换
   - 三态模型：运行、就绪、阻塞。
   - 五态模型：新建、就绪、运行、阻塞、终止。
   - 就绪到运行：被调度程序选中。
   - 运行到就绪：时间片用完或被更高优先级进程抢占。
   - 运行到阻塞：等待 I/O、等待信号量、等待事件。
   - 阻塞到就绪：等待事件完成。

4. PCB 与进程上下文
   - PCB 是进程存在的唯一标志，内核通过 PCB 管理和调度进程。
   - PCB 内容：进程标识、状态、程序计数器、寄存器、调度信息、内存信息、打开文件、资源清单等。
   - 进程上下文包括用户级上下文、寄存器上下文和系统级上下文。
   - 上下文切换：保存当前进程现场，恢复下一个进程现场，存在时间开销。

5. 进程队列
   - 就绪队列：等待 CPU 的进程。
   - 阻塞队列：等待某类事件的进程，通常按事件或设备分队列。
   - 作业队列：后备作业或进入系统的作业集合。

6. 进程创建与撤销
   - 创建：申请 PCB，分配地址空间和资源，初始化 PCB，插入就绪队列。
   - 撤销：回收资源，释放地址空间和 PCB，处理父子进程关系。
   - UNIX 中 fork 创建子进程，父子进程从 fork 返回处继续执行，返回值不同；exec 用新程序映像覆盖当前进程。

7. 线程与轻进程
   - 线程是进程内的一个执行流，是处理机调度的基本单位。
   - 同一进程内线程共享地址空间、全局变量、打开文件等资源，但有各自的程序计数器、寄存器和栈。
   - 用户级线程：切换快，但一个线程阻塞可能导致整个进程阻塞。
   - 核心级线程：内核可独立调度，适合多处理机，但切换开销大。
   - 混合线程：把多个用户级线程映射到若干内核实体，兼顾效率和并行能力。

8. 作业
   - 作业是用户提交给系统的一个完整任务，通常包括程序、数据和作业说明书。
   - 批处理作业由作业控制语言描述，交互式作业由命令解释程序接收命令。
   - 作业调度决定哪些后备作业进入内存，进程调度决定哪个就绪进程占用 CPU。

### 考点整理

- 会画三态或五态进程状态转换图，并解释每条边的触发条件。
- 会说明 PCB 的作用和典型内容。
- 会比较程序与进程、进程与线程、作业与进程。
- 会分析 fork 程序产生多少进程，注意父子进程都会继续执行 fork 后面的语句。
- 会区分用户级线程、内核级线程、混合线程。

### 考点答案与范例

1. 进程与程序比较

| 比较项 | 程序 | 进程 |
| --- | --- | --- |
| 性质 | 静态代码和数据 | 动态执行过程 |
| 生命周期 | 可长期保存 | 有创建、运行、撤销 |
| 并发性 | 本身不能并发 | 可与其他进程并发 |
| 资源 | 不直接占有运行资源 | 占有地址空间、文件等资源 |
| 标志 | 文件名或代码段 | PCB |

2. 进程与线程比较

| 比较项 | 进程 | 线程 |
| --- | --- | --- |
| 资源拥有 | 资源分配基本单位 | 通常不独立拥有资源 |
| 调度 | 传统 OS 中是调度单位 | 现代 OS 中常是调度单位 |
| 地址空间 | 进程间隔离 | 同进程线程共享 |
| 切换开销 | 大 | 小 |
| 通信 | 需要 IPC | 可直接共享变量，但需同步 |

3. 状态转换范例
   - 正在运行的进程执行 read 等待磁盘数据，运行变阻塞。
   - 磁盘中断到来，数据准备好，阻塞变就绪。
   - 调度程序选择该进程，就绪变运行。
   - 时间片用完，运行变就绪。

4. fork 计数范例

~~~c
fork();
fork();
~~~

第一次 fork 后有 2 个进程，第二次 fork 由这 2 个进程各执行一次，最终有 4 个进程。若 fork 在 if 中，要分别分析父进程返回子进程 PID、子进程返回 0 的分支。

### 常见题型与答题抓手

- “PCB 为什么重要”：答“PCB 是进程存在的唯一标志”。
- “运行、就绪、阻塞区别”：运行有 CPU，就绪只缺 CPU，阻塞在等事件。
- “线程共享什么”：共享进程资源，不共享执行现场和栈。
- “作业调度与进程调度区别”：前者决定作业能否进入内存，后者决定谁占 CPU。

## 第3章 中断与处理机调度

> 整理依据：`课件/操作系统原理课件 2026/03第三章 中断与处理机调度.pptx`。

### 章节定位

本章连接硬件事件处理和 CPU 分配策略。中断部分要掌握中断源、中断向量、中断嵌套、优先级和处理流程；调度部分要能计算周转时间、带权周转时间、等待时间和响应时间，并比较 FCFS、SJF、SRTN、HRN、HPF、RR、多级队列、反馈队列、EDF、RMS 等算法。

### 核心知识点

1. ==中断的概念==
   - 中断是 CPU 执行当前程序时，由内部或外部事件引起，暂停当前程序转去执行相应处理程序，处理完后再返回原程序的机制。
   - 中断使操作系统能响应 I/O 完成、时钟、异常、系统调用和硬件故障。
   - 自愿性中断通常指系统调用，程序主动请求内核服务。

2. 中断系统
   - 中断源：产生中断的事件或设备。
   - 中断字：记录中断源、状态和屏蔽信息。
   - 中断向量：保存中断处理程序入口地址的表项。
   - 中断嵌套：高优先级中断可打断低优先级中断处理。
   - 中断屏蔽：临时禁止某些中断，保护关键处理过程。
   - 系统栈：保存被中断程序的现场和嵌套中断现场。

3. 中断响应与处理流程
   - 识别中断源。
   - 保护断点和现场，如 PC、PSW、寄存器。
   - 切换到核心态和系统栈。
   - 根据中断向量转入中断处理程序。
   - 执行具体处理，如唤醒等待 I/O 的进程。
   - 恢复现场并返回，必要时触发调度。

4. 调度指标
   - CPU 利用率：CPU 忙碌时间比例。
   - 吞吐量：单位时间完成作业数。
   - 周转时间：完成时间减提交时间。
   - 带权周转时间：周转时间除以运行时间。
   - 等待时间：在就绪队列中等待 CPU 的总时间。
   - 响应时间：从提交请求到首次响应的时间。

5. 处理机调度算法

| 算法 | 基本思想 | 优点 | 缺点 |
| --- | --- | --- | --- |
| FCFS | 先来先服务 | 简单、公平 | 短作业可能等待很久 |
| SJF | 运行时间短者优先 | 平均等待时间小 | 需预知运行时间，长作业可能饥饿 |
| SRTN | 剩余时间最短者优先 | 抢占式 SJF | 切换频繁，长作业不利 |
| HRN | 响应比最高者优先 | 兼顾等待时间和运行时间 | 每次调度需计算响应比 |
| HPF | 优先级最高者优先 | 体现紧急程度 | 低优先级可能饥饿 |
| RR | 时间片轮转 | 适合分时系统 | 时间片过大退化为 FCFS，过小切换开销大 |
| MLQ | 多个固定队列 | 区分前台后台任务 | 队列间可能不公平 |
| FB | 多级反馈队列 | 兼顾交互与批处理 | 参数多，分析复杂 |

6. 调度级别
   - 高级调度：作业调度，从后备队列选择作业进入内存。
   - 中级调度：交换调度，把进程换入或换出内存。
   - 低级调度：进程调度，从就绪队列选择进程占用 CPU。

7. 实时调度
   - EDF：最早截止期优先，动态优先级。
   - RMS：速率单调调度，周期越短优先级越高，静态优先级。
   - 周期任务常用参数：执行时间 C、周期 T、截止期 D。

8. 多处理机调度
   - 自调度：多个处理机从公共就绪队列取任务，简单但队列竞争大。
   - 组调度：相关线程作为一组同时调度，适合并行程序。
   - 负载均衡：避免某些 CPU 空闲而其他 CPU 过载。

### 考点整理

- 会写中断处理的基本步骤，并说明现场保护和恢复的意义。
- 会区分中断、异常、系统调用。
- 会计算各调度算法下的完成时间、周转时间、带权周转时间和平均等待时间。
- 会判断抢占式与非抢占式调度。
- 会比较 RR 时间片大小对响应时间和切换开销的影响。
- 会解释 EDF 与 RMS 的优先级依据。

### 考点答案与范例

1. 中断与系统调用比较

| 项目 | 中断 | 系统调用 |
| --- | --- | --- |
| 触发来源 | 外设、时钟、故障或异常 | 用户程序主动请求 |
| 目的 | 响应异步或异常事件 | 使用内核服务 |
| 状态变化 | 通常进入核心态 | 由陷入指令进入核心态 |
| 例子 | 磁盘 I/O 完成中断 | open、read、write |

2. HRN 公式

响应比 R = (等待时间 + 要求服务时间) / 要求服务时间 = 1 + 等待时间 / 要求服务时间。

例子：A 需要 10ms、已等 20ms，R=3；B 需要 2ms、已等 2ms，R=2。HRN 选择 A。

3. RR 时间片分析
   - 时间片太大：交互响应变差，近似 FCFS。
   - 时间片太小：响应较快，但上下文切换次数增加，系统开销大。
   - 合理时间片应使大多数交互请求能较快得到响应，又不造成过多切换。

4. 调度计算模板
   - 画时间轴，标出每个进程的开始和完成时间。
   - 周转时间 = 完成时间 - 到达时间。
   - 等待时间 = 周转时间 - 实际运行时间。
   - 带权周转时间 = 周转时间 / 实际运行时间。
   - 平均值 = 所有进程对应指标求和 / 进程数。

### 常见题型与答题抓手

- “给到达时间和服务时间求平均周转”：先画甘特图，再套公式。
- “判断是否抢占”：看到时间片、更高优先级抢占、最短剩余时间，一般是抢占式。
- “实时调度选谁”：EDF 看截止期，RMS 看周期。
- “为什么需要时钟中断”：支持时间片轮转、计时、超时处理和抢占。

## 第4章 互斥、同步与通讯

> 整理依据：`课件/操作系统原理课件 2026/04第四章 互斥同步与通讯（1）.pptx` 与 `04第四章 互斥同步与通讯（2）.pptx`。

### 章节定位

本章是并发程序正确性的核心。重点掌握临界区、互斥、同步、信号量与 P/V 操作，以及生产者-消费者、读者-写者、哲学家就餐等经典问题。课件还涉及软件互斥算法、硬件原语、会合机制和消息传递通信。考试常要求解释概念、判断算法是否正确、补全 P/V 顺序或分析死锁。

### 核心知识点

1. ==并发程序与时间有关的错误==
   - 顺序程序具有顺序性、封闭性和可再现性。
   - 并发程序的执行速度和交错顺序不可预知，若共享变量访问没有同步，可能产生与时间有关的错误。
   - 例子：两个进程同时执行 count = count + 1，若读、改、写交错，最终 count 可能只增加 1。

2. 互斥与临界区
   - 临界资源：一次只允许一个进程使用的资源，如共享变量、打印机、缓冲区指针。
   - 临界区：访问临界资源的代码段。
   - 互斥：多个进程不能同时进入访问同一临界资源的临界区。
   - 临界区管理原则：空闲让进、忙则等待、有限等待、让权等待。

3. 软件互斥算法
   - 单标志法、双标志法常用来说明互斥算法的缺陷。
   - Dekker 算法和 Peterson 算法是两个进程互斥的经典软件算法。
   - Lamport 面包店算法解决 n 个进程互斥，思想是“先取号，号码小者先进入，号码相同按进程号决定”。
   - Eisenberg/McGuire 算法使用 idle、want_in、in_cs 状态和 turn 指针解决 n 进程互斥。
   - 注意：面包店算法是软件互斥算法，不是信号量算法，但常与信号量例题一起考查。

4. 硬件互斥原语
   - 关中断：单处理机上可防止进程切换，但不适合多处理机和长临界区。
   - Test-and-Set：原子测试并置位。
   - Swap：原子交换两个变量值。
   - 硬件原语可实现自旋锁；自旋锁会忙等，适合临界区很短的场景。

5. 信号量与 P/V 操作
   - 信号量是一个整型变量及其等待队列，只能通过 P、V 原语访问。
   - P(S)：申请资源或等待条件。S 减 1，若 S < 0，则进程阻塞进入 S 的等待队列。
   - V(S)：释放资源或发出信号。S 加 1，若 S <= 0，则唤醒一个等待进程。
   - 互斥信号量初值通常为 1。
   - 资源计数信号量初值通常为可用资源数。
   - 同步信号量初值由先后关系决定，若事件尚未发生通常为 0。

6. 生产者-消费者问题
   - 问题：生产者向有界缓冲区放产品，消费者从缓冲区取产品。
   - 信号量：mutex=1 保护缓冲区；empty=n 表示空缓冲区数；full=0 表示满缓冲区数。
   - 正确顺序：生产者先 P(empty) 再 P(mutex)，消费者先 P(full) 再 P(mutex)，退出临界区后先 V(mutex)，再 V(full) 或 V(empty)。

~~~text
Producer:
while true:
  produce item
  P(empty)
  P(mutex)
  put item into buffer
  V(mutex)
  V(full)

Consumer:
while true:
  P(full)
  P(mutex)
  remove item from buffer
  V(mutex)
  V(empty)
  consume item
~~~

7. 读者-写者问题
   - 多个读者可同时读，写者写时必须独占。
   - 读者优先方案可能导致写者饥饿。
   - 写者优先方案避免写者长期等待，但读者可能延迟。
   - 公平方案通常引入队列或 turnstile，按到达顺序服务。

8. 哲学家就餐问题
   - 五个哲学家共享五只筷子，每人需要左右两只筷子才能进餐。
   - 若每人先拿左筷子再拿右筷子，可能所有人都拿到左筷子后互相等待，发生死锁。
   - 常见解法：限制最多 4 人同时拿筷子；规定奇偶哲学家拿筷子顺序相反；使用管程或服务员统一分配。

9. 会合与高级通信
   - 会合 Rendezvous：发送方和接收方在某一同步点相遇，常见于 Ada 任务通信。
   - 直接通信：send/receive 直接指定进程。
   - 间接通信：通过信箱或端口传递消息。
   - 有缓冲通信：发送者把消息放入缓冲区即可继续。
   - 无缓冲通信：发送和接收必须同步，类似会合。

### 考点整理

- 会说明临界资源、临界区、互斥、同步之间的关系。
- 会写 P/V 操作的语义，并根据题意设置初值。
- 会补全生产者-消费者、读者-写者、哲学家就餐算法。
- 会判断 P/V 顺序错误导致死锁或互斥失效的原因。
- 会比较忙等锁、阻塞信号量、消息传递、会合。
- 会解释面包店算法的取号思想和有限等待性质。

### 考点答案与范例

1. 互斥与同步比较

| 项目 | 互斥 | 同步 |
| --- | --- | --- |
| 目的 | 防止同时访问临界资源 | 保证进程按某种先后关系推进 |
| 典型信号量初值 | 1 | 常为 0 或资源数量 |
| 例子 | 修改共享计数器 | 先生产后消费 |
| 错误后果 | 数据竞争、结果错误 | 顺序颠倒、读空缓冲区 |

2. P/V 初值判断
   - 互斥访问一个共享变量：mutex = 1。
   - 有 n 个同类资源：resource = n。
   - A 必须在 B 前完成：设置 s = 0，A 完成后 V(s)，B 开始前 P(s)。

3. 生产者-消费者顺序易错点
   - 不能先 P(mutex) 再 P(empty)。若缓冲区满，生产者拿着 mutex 等 empty，消费者无法进入临界区取走产品，可能死锁。
   - 不能忘记 mutex，否则多个生产者或消费者可能同时修改缓冲区指针。

4. 读者优先算法骨架

~~~text
semaphore rw = 1, mutex = 1
int readcount = 0

Reader:
  P(mutex)
  readcount++
  if readcount == 1: P(rw)
  V(mutex)
  read
  P(mutex)
  readcount--
  if readcount == 0: V(rw)
  V(mutex)

Writer:
  P(rw)
  write
  V(rw)
~~~

5. 面包店算法抓手
   - choosing[i] 表示进程 i 正在取号。
   - number[i] 表示进程 i 的号码。
   - 进程 i 等待所有其他进程取号完成，并等待比自己号码小的进程先进入。
   - 比较规则是二元组 (number, pid)，号码小优先，号码相同 pid 小优先。

6. 直接通信与间接通信比较

| 项目 | 直接方式 | 间接方式 |
| --- | --- | --- |
| 通信对象 | 明确指定接收进程 | 通过信箱或端口 |
| 耦合程度 | 强 | 弱 |
| 例子 | send(P, msg) | send(mailbox, msg) |
| 适用场景 | 进程关系固定 | 生产者消费者、客户服务器 |

### 常见题型与答题抓手

- “P/V 题”：先找互斥资源，再找同步先后关系，最后确定初值和 P/V 位置。
- “算法是否死锁”：看是否出现循环等待，特别是拿着 mutex 等待其他条件。
- “读者写者”：明确题目要求读者优先、写者优先还是公平。
- “哲学家”：破坏死锁四条件之一即可，如破坏循环等待或占有并等待。

## 第5章 死锁与饥饿

> 整理依据：`课件/操作系统原理课件 2026/05第五章 死锁与饥饿.pptx`。

### 章节定位

本章讨论并发系统中的资源竞争失效。重点掌握死锁定义、死锁必要条件、资源分配图、死锁预防、避免、检测与恢复，以及死锁、饥饿、活锁的区别。银行家算法和资源分配图约简是最常见的计算考点。

### 核心知识点

1. ==死锁定义==
   - 一组进程中的每个进程都在等待只能由该组其他进程释放的事件或资源，导致所有进程都无法继续推进，称为死锁。
   - 例子：P1 已占有打印机，等待扫描仪；P2 已占有扫描仪，等待打印机。两者互相等待，形成死锁。

2. 死锁类型
   - 资源死锁：多个进程竞争不可剥夺资源。
   - 通信死锁：进程之间等待对方发送或接收消息。
   - 存储死锁：进程等待内存或缓冲区，释放条件又依赖其他等待进程。
   - 进程推进顺序不当也可能形成死锁。

3. 死锁的四个必要条件
   - 互斥条件：资源一次只能被一个进程使用。
   - 占有并等待：进程已占有资源，又申请新的资源。
   - 不可剥夺：资源不能被强行抢占，只能由占有进程主动释放。
   - 循环等待：存在进程资源等待环。
   - 四个条件同时成立才可能死锁，破坏任一条件可预防死锁。

4. 资源分配图
   - 进程节点用圆表示，资源节点用方框表示，资源中可有多个实例。
   - 请求边：进程指向资源。
   - 分配边：资源实例指向进程。
   - 若每类资源只有一个实例，图中有环等价于死锁。
   - 若资源有多个实例，有环只是可能死锁，不一定死锁。

5. 死锁定理与图约简
   - 找一个请求能被当前可用资源满足的非孤立进程，假设它运行完成并释放资源，删除相关边。
   - 若所有边都能删去，系统无死锁。
   - 若最后仍有无法删除的边，剩余进程集合处于死锁。

6. 死锁预防
   - 预先分配法：进程运行前一次申请全部资源，破坏占有并等待。
   - 有序分配法：给资源编号，进程只能按递增顺序申请，破坏循环等待。
   - 剥夺资源法：对某些资源允许强行回收，破坏不可剥夺。
   - 缺点：资源利用率低，限制强，可能增加等待时间。

7. 死锁避免
   - 避免不是强行破坏条件，而是在分配前判断是否会进入不安全状态。
   - 安全状态：存在一个安全序列，使所有进程都能依次完成。
   - 不安全状态不一定死锁，但可能导致死锁。
   - 银行家算法使用 Available、Max、Allocation、Need 矩阵判断资源请求是否安全。

8. 银行家算法步骤
   - 检查 Request_i <= Need_i，否则出错。
   - 检查 Request_i <= Available，否则进程等待。
   - 试探分配：Available 减 Request_i，Allocation_i 加 Request_i，Need_i 减 Request_i。
   - 执行安全性算法，若安全则正式分配，否则回滚。

9. 死锁检测与恢复
   - 检测：允许系统进入死锁，周期性或在资源请求失败时运行检测算法。
   - 恢复：撤销进程、回滚到检查点、剥夺资源。
   - 选择牺牲进程时考虑优先级、已运行时间、占有资源、回滚代价等。

10. 饥饿与活锁
   - 饥饿：进程长期得不到所需资源或 CPU，但系统整体仍在运行。
   - 活锁：进程不断改变状态响应对方，却没有实质进展。
   - 死锁是所有相关进程阻塞不动，饥饿是个别进程长期等待。

### 考点整理

- 会准确写出死锁定义和四个必要条件。
- 会判断资源分配图是否死锁，特别注意单实例和多实例差异。
- 会执行资源分配图约简。
- 会用银行家算法判断一个请求能否立即满足。
- 会比较死锁预防、避免、检测恢复、鸵鸟算法。
- 会区分死锁、饥饿、活锁。

### 考点答案与范例

1. 死锁与饥饿比较

| 项目 | 死锁 | 饥饿 | 活锁 |
| --- | --- | --- | --- |
| 状态 | 相关进程互相等待而阻塞 | 某进程长期得不到资源 | 进程持续动作但无进展 |
| 系统整体 | 局部或全部停滞 | 其他进程可能正常完成 | CPU 可能被消耗 |
| 原因 | 循环等待等条件同时成立 | 调度或分配策略不公平 | 过度礼让或重试策略 |
| 例子 | 两进程互等两资源 | 低优先级长期不运行 | 两人让路同时左右移动 |

2. 死锁处理策略比较

| 策略 | 思想 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 预防 | 破坏死锁必要条件 | 理论简单 | 资源利用率低 |
| 避免 | 分配前保持安全状态 | 比预防灵活 | 需要最大需求等先验信息 |
| 检测恢复 | 允许死锁，发现后恢复 | 资源利用率较高 | 检测和恢复有代价 |
| 鸵鸟算法 | 忽略低概率死锁 | 实现简单 | 死锁时需人工或重启处理 |

3. 银行家算法安全性检测模板

~~~text
Work = Available
Finish[i] = false
while exists i such that Finish[i] == false and Need[i] <= Work:
    Work = Work + Allocation[i]
    Finish[i] = true
if all Finish[i] == true:
    safe, found safe sequence
else:
    unsafe
~~~

4. 安全状态例子
   - Available = (3,3,2)。若存在 P1、P3、P0、P2、P4 的顺序，使每个 Pi 的 Need_i 都不超过当前 Work，则该顺序是安全序列。
   - 若找不到任何可完成进程，当前试探分配不安全，应拒绝本次请求。

5. 资源分配图例子
   - 单实例资源：P1 等 R2，R2 分给 P2；P2 等 R1，R1 分给 P1。存在环 P1 -> R2 -> P2 -> R1 -> P1，因此死锁。
   - 多实例资源：即使有环，也可能还有空闲实例或环外进程释放资源，因此需要约简判断。

### 常见题型与答题抓手

- “是否死锁”：先看资源实例数，单实例看环，多实例做约简或检测。
- “银行家算法”：先检查合法性，再试分配，最后找安全序列。
- “预防方法属于破坏哪个条件”：预先分配破坏占有并等待，有序分配破坏循环等待。
- “不安全是否死锁”：不安全不一定死锁，只是存在进入死锁的可能。

## 第6章 存储管理

> 整理依据：`课件/操作系统原理课件 2026/06第六章 存储管理.pptx`。

### 章节定位

本章讲主存如何分配、保护、共享和地址转换，是虚拟存储的前置基础。重点掌握内存分配算法、碎片、界地址管理、覆盖、分页、分段、段页式管理、多级页表、反置页表和快表 TLB。考试常考地址转换、有效访问时间和分页分段比较。

### 核心知识点

1. ==存储管理功能==
   - 内存分配与回收：为进程分配主存空间，结束后回收。
   - 地址转换：把逻辑地址转换为物理地址。
   - 存储保护：防止越界访问和非法访问。
   - 存储共享：多个进程共享代码段、库或共享内存。
   - 存储扩充：通过覆盖、交换、虚拟存储等技术扩大可运行程序规模。

2. 内存资源管理
   - 位示图：用位表示页面或块是否空闲，适合固定大小块管理。
   - 空闲页面表：记录空闲页框号。
   - 空闲页面链：把空闲块链接起来，分配回收方便。
   - 动态异长分区：按进程需要分配连续可变大小分区。

3. 动态分区分配算法

| 算法 | 思想 | 优点 | 缺点 |
| --- | --- | --- | --- |
| First Fit | 从低地址开始找第一个足够大的空闲区 | 快，保留高地址大块 | 低地址碎片多 |
| Next Fit | 从上次位置继续找 | 分布较均匀 | 可能破坏大空闲区 |
| Best Fit | 找最小但足够大的空闲区 | 单次剩余最小 | 易产生难以利用的小碎片 |
| Worst Fit | 找最大的空闲区 | 剩余块仍较大 | 大块消耗快 |

4. 碎片处理
   - 内部碎片：分配给进程但进程用不到的空间，如固定分区或分页最后一页。
   - 外部碎片：空闲空间总量足够但不连续，如动态分区。
   - 紧凑：移动进程使空闲区合并，需要动态重定位支持。

5. 界地址管理
   - 通过基址寄存器和限长寄存器实现连续分配下的地址转换与保护。
   - 逻辑地址 d 合法条件：0 <= d < limit；物理地址 = base + d。
   - 越界访问触发地址越界中断。

6. 覆盖技术
   - 把一个程序划分为若干功能模块，互斥执行的模块不同时装入内存。
   - 优点：可在较小内存中运行较大程序。
   - 缺点：需要程序员或编译系统设计覆盖结构，管理复杂。

7. 分页式存储管理
   - 逻辑地址空间划分为页，物理内存划分为同样大小的页框。
   - 逻辑地址 (p,d)：p 为页号，d 为页内位移。
   - 页表记录页号到页框号的映射，物理地址为 (f,d)。
   - 分页消除外部碎片，但可能有内部碎片。
   - 快表 TLB 缓存页表项，加速地址转换。

8. 多级页表和反置页表
   - 多级页表把页表分页，只为实际使用的页表页分配内存，适合大地址空间。
   - 反置页表按物理页框建立表项，每个页框一个表项，减少页表空间，但查找需要散列或辅助结构。

9. 分段式存储管理
   - 逻辑地址按程序自然单位划分为段，如代码段、数据段、栈段。
   - 逻辑地址 (s,d)：s 为段号，d 为段内位移。
   - 段表项记录段长、段基址和保护信息。
   - 分段便于共享和保护，但会产生外部碎片。

10. 段页式存储管理
   - 先按逻辑意义分段，再把每段分页。
   - 逻辑地址 (s,p,d)，先查段表得到该段页表，再查页表得到页框号。
   - 兼有分段的逻辑保护共享和分页的离散分配优点，但地址转换层次更复杂。

### 考点整理

- 会区分内部碎片和外部碎片。
- 会模拟 First Fit、Next Fit、Best Fit、Worst Fit 的分配结果。
- 会进行分页、分段、段页式地址转换。
- 会计算 TLB 命中率下的有效访问时间。
- 会比较页式、段式、段页式管理。
- 会解释多级页表和反置页表的目的。

### 考点答案与范例

1. 页式与段式比较

| 项目 | 分页 | 分段 |
| --- | --- | --- |
| 划分依据 | 固定大小 | 程序逻辑单位 |
| 地址形式 | 页号 + 页内位移 | 段号 + 段内位移 |
| 碎片 | 内部碎片 | 外部碎片 |
| 共享保护 | 不如分段直观 | 按段共享和保护自然 |
| 用户可见性 | 通常用户不可见 | 用户可感知逻辑段 |

2. 分页地址转换例子
   - 页大小 1KB，逻辑地址 2500。
   - 页号 p = 2500 / 1024 = 2，页内位移 d = 452。
   - 若页表中页 2 对应页框 8，则物理地址 = 8 * 1024 + 452 = 8644。

3. 分段地址转换例子
   - 段表项：段 1 基址 4000，段长 1000。
   - 逻辑地址 (1,600) 合法，物理地址 = 4600。
   - 逻辑地址 (1,1200) 越界，因为 1200 >= 1000。

4. TLB 有效访问时间
   - 设访问内存时间为 m，TLB 查找时间为 t，命中率为 h。
   - 命中：t + m。
   - 未命中：t + 2m，因为先访问页表再访问数据。
   - EAT = h(t+m) + (1-h)(t+2m)。

5. 动态分区算法例子
   - 空闲区大小为 100、500、200、300，申请 180。
   - First Fit 选择 500。
   - Best Fit 选择 200。
   - Worst Fit 选择 500。
   - Next Fit 从上次位置继续找，选择从当前位置之后遇到的第一个足够大分区。

### 常见题型与答题抓手

- “地址转换题”：先拆逻辑地址，再查表，最后检查越界或有效位。
- “比较题”：分页重物理分配，分段重逻辑结构。
- “碎片题”：固定大小分配多内部碎片，连续可变分区多外部碎片。
- “TLB 题”：明确命中是否仍需访问一次内存取数据。

## 第7章 虚拟存储管理

> 整理依据：`课件/操作系统原理课件 2026/07第七章 虚拟存储管理.pptx`。

### 章节定位

本章在分页、分段基础上引入虚拟存储。重点掌握请求分页、缺页中断、页表扩展位、页面调入和淘汰策略、页面置换算法、Belady 异常、颠簸、工作集模型、页故障率反馈，以及虚拟段式和虚拟段页式系统。考试常给页面访问序列，要求计算缺页次数和置换过程。

### 核心知识点

1. ==虚拟存储系统概念==
   - 虚拟存储利用程序局部性原理，只把当前需要的部分装入内存，其余放在外存。
   - 程序看到的地址空间可大于实际物理内存。
   - 需要硬件地址转换、中断机制、外存和操作系统置换策略支持。

2. 请求分页
   - 页表项通常包含页框号、有效位、访问位 R、修改位 M、保护位、外存地址等。
   - 有效位为 0 时访问该页触发缺页中断。
   - 缺页处理：保护现场，判断访问合法性，寻找空闲页框，必要时淘汰页面，调入所需页，修改页表，恢复执行。

3. 页面调入与页框分配
   - 请求调页：访问到某页才调入，启动快，缺页次数可能多。
   - 预调页：根据预测提前调入，可能减少缺页，也可能调入无用页。
   - 固定分配：每个进程页框数固定。
   - 可变分配：根据缺页率或工作集变化调整页框数。
   - 局部置换：只在本进程页框中置换。
   - 全局置换：可从全系统页框中选择牺牲页。

4. 页面置换算法

| 算法 | 思想 | 特点 |
| --- | --- | --- |
| OPT | 淘汰未来最长时间不用的页 | 理论最优，用作比较基准 |
| FIFO | 淘汰最早进入内存的页 | 简单，可能有 Belady 异常 |
| LRU | 淘汰最长时间未使用的页 | 接近 OPT，实现成本较高 |
| NRU | 根据访问位和修改位分类淘汰 | 近似 LRU，开销较低 |
| LFU | 淘汰访问次数最少的页 | 可能保留过时热点页 |
| Second Chance | FIFO 基础上给访问位为 1 的页第二次机会 | 实用近似算法 |
| Clock | 用环形指针实现 Second Chance | 常用，高效 |
| 改进 Clock | 同时考虑访问位和修改位 | 优先淘汰未访问且未修改页 |

5. Belady 异常
   - 在某些算法中，分配更多页框反而缺页次数增加，称为 Belady 异常。
   - FIFO 可能出现 Belady 异常。
   - 栈算法如 LRU、OPT 不会出现 Belady 异常。

6. 颠簸与工作集
   - 颠簸：系统把大量时间花在页面换入换出上，CPU 利用率急剧下降。
   - 原因：进程获得页框少于其局部性需求，频繁缺页。
   - 工作集 WS(t, Δ)：时刻 t 前 Δ 时间窗口内进程访问过的页面集合。
   - 若为进程分配不少于工作集大小的页框，可减少颠簸。

7. 页故障率反馈模型
   - 页故障率过高：说明页框不足，应增加页框或降低多道程度。
   - 页故障率过低：说明页框可能过多，可回收部分页框。

8. 虚拟段式与虚拟段页式
   - 虚拟段式：以段为单位装入、置换和保护，便于动态连接和共享。
   - 动态连接：调用某段时再连接并调入，节省内存。
   - 虚拟段页式：段内分页，缺页中断和地址转换更复杂，但兼有逻辑分段和离散分配优点。

9. 系统实例
   - Linux 常用伙伴系统管理物理页框，用多级页表支持虚拟地址转换。
   - Windows 使用工作集、页面文件和多级页表管理虚拟内存。

### 考点整理

- 会写缺页中断处理流程。
- 会根据访问串计算 FIFO、LRU、OPT、Clock 的缺页次数。
- 会解释 Belady 异常，并指出 FIFO 可能出现，LRU/OPT 不会出现。
- 会比较局部置换和全局置换、固定分配和可变分配。
- 会解释颠簸、工作集、页故障率反馈。
- 会说明访问位和修改位在置换中的作用。

### 考点答案与范例

1. 缺页中断流程
   - CPU 访问页面，页表有效位为 0。
   - 硬件产生缺页中断，陷入内核。
   - OS 判断地址是否合法；非法则终止进程。
   - 找空闲页框；无空闲页框则选择牺牲页。
   - 若牺牲页修改位为 1，先写回外存。
   - 从外存读入所需页，更新页表和 TLB。
   - 重新执行被中断的指令。

2. FIFO 与 LRU 示例
   - 访问串：1 2 3 1 4，页框数 3。
   - FIFO：装入 1、2、3 后访问 1 命中，访问 4 淘汰最早进入的 1。
   - LRU：访问 1 后，最近最久未用的是 2，访问 4 淘汰 2。
   - 抓手：FIFO 看进入内存时间，LRU 看最近访问时间。

3. Clock 算法模板

~~~text
while true:
  if R[hand] == 0:
      replace page at hand
      advance hand
      break
  else:
      R[hand] = 0
      advance hand
~~~

4. 改进 Clock 的淘汰优先级

| 类别 | R | M | 含义 | 优先级 |
| --- | --- | --- | --- | --- |
| 1 | 0 | 0 | 最近未访问且未修改 | 最高 |
| 2 | 0 | 1 | 最近未访问但已修改 | 次高，需写回 |
| 3 | 1 | 0 | 最近访问未修改 | 较低 |
| 4 | 1 | 1 | 最近访问且已修改 | 最低 |

5. 工作集例子
   - 访问序列到时刻 t 为 5,7,1,6,2,1，窗口 Δ=5。
   - 最近 5 次访问为 7,1,6,2,1，工作集为 {7,1,6,2}，大小为 4。
   - 若只分配 2 个页框，很可能频繁缺页。

### 常见题型与答题抓手

- “页面置换计算”：画页框表，每访问一页更新命中/缺页和淘汰页。
- “FIFO 与 LRU 混淆”：FIFO 不因再次访问改变队列顺序，LRU 会改变最近使用顺序。
- “颠簸原因”：工作集大于分配页框数，多道程度过高。
- “改进 Clock”：优先找 R=0、M=0，因为无需写回且近期未用。

## 第8章 文件系统

> 整理依据：`课件/操作系统原理课件 2026/08第八章 文件系统1.pptx`。

### 章节定位

本章讨论文件如何抽象、组织、查找、共享、保护和落盘。重点掌握文件逻辑结构、物理组织、目录、FCB/inode、文件共享、保护、安全、外存空间管理、UNIX 文件系统调用、日志结构文件系统、内存映射文件和 VFS。考试常考文件组织方式比较、目录查找、链接共享、空闲块管理和 UNIX inode。

### 核心知识点

1. ==文件概念==
   - 文件是具有符号名的一组相关信息集合，是操作系统管理外存信息的基本单位。
   - 文件属性包括文件名、类型、长度、位置、所有者、权限、时间戳等。
   - UNIX 常把普通文件、目录文件、设备文件等统一为文件。

2. 文件访问方式
   - 顺序访问：按记录或字节顺序读写，适合文本、日志。
   - 直接访问：按相对块号访问，适合数据库和随机访问文件。
   - 索引访问：通过索引表或关键字查找记录。

3. 文件逻辑组织
   - 无结构字节流：如 UNIX 普通文件。
   - 记录式文件：由定长或变长记录组成。
   - 关键字文件：按关键字组织和查找。

4. 文件物理组织

| 组织方式 | 思想 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 连续结构 | 文件占连续磁盘块 | 顺序和随机访问快 | 外部碎片，扩展困难 |
| 链接结构 | 每块指向下一块 | 无外部碎片，易扩展 | 随机访问慢，指针损坏影响大 |
| 索引结构 | 索引块保存数据块地址 | 支持随机访问和动态扩展 | 索引块有空间开销 |
| Hash 结构 | 按关键字散列定位 | 查找快 | 冲突处理复杂 |

5. 文件目录与 FCB
   - FCB 文件控制块记录文件元数据和物理位置信息。
   - 目录是文件名到 FCB 或 inode 的映射。
   - 单级目录简单但命名冲突严重。
   - 两级目录为每个用户设置目录，减少冲突。
   - 树形目录支持层次路径，是现代系统常用结构。
   - 目录查找可用缓存、散列和当前目录加速。

6. UNIX inode
   - inode 保存文件类型、权限、属主、大小、时间、数据块地址等。
   - 目录项通常保存文件名和 inode 号。
   - UNIX 常采用直接索引、一次间接索引、二次间接索引、三次间接索引组合，兼顾小文件效率和大文件容量。

7. 文件共享
   - 硬链接：多个目录项指向同一 inode，共享同一文件内容。
   - 符号链接：保存目标路径的特殊文件，可跨文件系统。
   - 引用计数用于判断 inode 是否仍被目录项引用。

8. 文件保护、保密与安全
   - 存取控制矩阵：行表示主体，列表示客体，元素表示权限。
   - 访问控制表 ACL：每个文件列出哪些用户拥有哪些权限。
   - 能力表 Capability：每个用户或进程列出可访问对象和权限。
   - UNIX 权限：属主、同组用户、其他用户分别有读 r、写 w、执行 x 权限。
   - 保密强调防止未授权读取，可使用加密。
   - 安全强调防止故障和破坏，可使用备份、日志、校验和。

9. 文件系统实现与外存空间管理
   - 位示图：每位表示一个磁盘块是否空闲，查找连续空闲块方便。
   - 空闲块链：把空闲块链接起来，空间开销小但查找连续块困难。
   - 成组链接：UNIX 常用，把空闲块地址分组保存，兼顾链式和批量管理。
   - 超级块记录文件系统总体信息，如块大小、空闲块、空闲 inode 等。

10. 文件系统界面
   - creat/open/close：创建、打开、关闭文件。
   - read/write：读写文件。
   - seek：改变当前文件读写位置。
   - link/unlink：建立和删除目录项链接。
   - mount/umount：安装和卸载文件系统。
   - chdir/chown/chmod/stat：改变目录、属主、权限和读取状态。

11. 日志结构文件系统与内存映射文件
   - 日志结构文件系统把修改顺序写入日志，提高写性能并便于恢复。
   - 内存映射文件把文件映射到进程地址空间，可像访问内存一样访问文件。
   - VFS 虚拟文件系统为不同具体文件系统提供统一接口。

### 考点整理

- 会比较连续、链接、索引三种物理组织方式。
- 会解释 FCB、目录项、inode 的关系。
- 会区分硬链接和符号链接。
- 会说明 UNIX 权限位和文件类型。
- 会比较位示图、空闲块链、成组链接。
- 会解释 open/read/write/seek/link/unlink 等系统调用的功能。

### 考点答案与范例

1. inode 与目录项关系
   - 目录项把文件名映射到 inode 号。
   - inode 保存文件元数据和数据块地址。
   - 因此同一个 inode 可对应多个文件名，这就是硬链接。

2. 硬链接与符号链接比较

| 项目 | 硬链接 | 符号链接 |
| --- | --- | --- |
| 本质 | 多个目录项指向同一 inode | 一个保存路径名的特殊文件 |
| 是否跨文件系统 | 通常不能 | 可以 |
| 目标删除后 | 只要链接计数不为 0 内容仍在 | 可能变成悬空链接 |
| 目录链接 | 通常限制 | 可以指向目录 |

3. 文件物理组织例子
   - 连续结构适合只读大文件，如顺序播放的视频。
   - 链接结构适合频繁扩展但主要顺序访问的文件。
   - 索引结构适合需要随机访问的文件，如可执行文件、数据库文件。

4. UNIX 权限例子
   - rwxr-x--- 表示属主可读写执行，同组用户可读执行，其他用户无权限。
   - 对目录而言，x 表示可进入目录，r 表示可列目录项，w 表示可创建或删除目录项。

5. open 与 read 的关系
   - open 根据路径查目录，建立打开文件表项，返回文件描述符 fd。
   - read(fd, buf, count) 根据 fd 找到打开文件表项和当前偏移，再读取数据并推进偏移。

### 常见题型与答题抓手

- “目录项和 inode 各存什么”：目录项存名字到 inode，inode 存属性和块地址。
- “删除文件发生什么”：unlink 删除目录项并减少链接计数，计数为 0 且无人打开时才释放数据块。
- “随机访问效率”：连续和索引较好，链式较差。
- “权限题”：先分属主、同组、其他，再解释 r/w/x。

## 第9章 设备与 I/O 管理

### 章节定位

本章讲操作系统如何管理外部设备和 I/O。重点掌握设备分类、I/O 传输方式、设备分配、设备驱动、磁盘调度、缓冲技术、UNIX 缓冲、RAID、虚拟设备和稳定存储器。考试常考 I/O 方式比较、磁盘访问时间计算、磁盘调度序列、缓冲与 SPOOLing。

### 核心知识点

1. ==设备分类==
   - 按使用特性：输入设备、输出设备、存储设备、通信设备。
   - 按共享属性：独占设备、共享设备、虚拟设备。
   - 按传输单位：字符设备、块设备。
   - 按速度：低速设备、中速设备、高速设备。

2. 磁盘基本结构
   - 磁盘由盘面、磁道、柱面和扇区组成。
   - 一次磁盘 I/O 时间 = 寻道时间 + 旋转延迟 + 数据传输时间。
   - 磁盘调度主要优化寻道时间。

3. I/O 传输方式

| 方式 | 工作方式 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 程序查询 | CPU 反复检查设备状态 | 实现简单 | CPU 忙等，效率低 |
| 中断驱动 | 设备完成后中断 CPU | CPU 可并行执行其他任务 | 每次传输仍需较多 CPU 参与 |
| DMA | DMA 控制器直接在设备和内存间传输块数据 | CPU 开销小，适合高速块设备 | 需要 DMA 控制器 |
| 通道 | 通道执行通道程序管理 I/O | I/O 并行能力强 | 硬件复杂 |

4. 设备分配与去配
   - 设备控制表 DCT：描述设备状态、类型、队列等。
   - 控制器控制表 COCT：描述控制器状态。
   - 通道控制表 CHCT：描述通道状态。
   - 系统设备表 SDT：系统所有设备总表。
   - 独占设备分配要检查设备、控制器和通道是否可用，并把请求进程挂入等待队列或完成分配。

5. 设备驱动
   - 设备驱动程序屏蔽硬件细节，向上提供统一接口。
   - 驱动通常负责初始化设备、启动 I/O、处理中断、错误处理和维护请求队列。
   - I/O 软件层次：用户 I/O 软件、设备无关软件、设备驱动、中断处理、硬件。

6. 磁盘调度算法

| 算法 | 思想 | 优点 | 缺点 |
| --- | --- | --- | --- |
| FCFS | 按请求到达顺序服务 | 公平简单 | 平均寻道距离可能大 |
| SSTF | 选择距离当前磁头最近请求 | 平均寻道较短 | 远端请求可能饥饿 |
| SCAN | 磁头像电梯一样单向移动，到端点后反向 | 避免过度来回 | 两端等待时间可能不同 |
| C-SCAN | 只按一个方向服务，到端点后回到起点 | 等待时间更均匀 | 回扫有额外移动 |
| LOOK | 类似 SCAN，但到最远请求处即反向 | 减少无效移动 | 实现需知道请求边界 |
| C-LOOK | 类似 C-SCAN，只在请求范围内循环 | 更少无效移动 | 仍有循环跳转开销 |

7. 缓冲技术
   - 单缓冲：一个缓冲区连接设备和进程，减少速度不匹配。
   - 双缓冲：一个缓冲区给设备使用，另一个给进程处理，提高并行性。
   - 循环缓冲：多个缓冲区按环形队列组织，适合连续流式 I/O。
   - 缓冲池：系统维护多个缓冲区，可动态分配给不同设备或文件。
   - UNIX 块缓冲使用 buf 头部、设备队列和延迟写等机制提高磁盘效率。

8. 输入输出进程
   - 把部分 I/O 管理功能放入专门进程，由它统一接受请求、启动设备并通知用户进程。
   - 好处是简化同步和设备管理，但增加进程通信开销。

9. RAID 技术
   - RAID 0：数据分条，无冗余，性能高但可靠性低。
   - RAID 1：镜像，可靠性高，空间利用率约 50%。
   - RAID 2：位级分条和汉明码校验，较少实际使用。
   - RAID 3：字节或位级分条，专用奇偶校验盘。
   - RAID 4：块级分条，专用奇偶校验盘，校验盘可能瓶颈。
   - RAID 5：块级分条，奇偶校验分布在各盘，读写和可靠性较均衡。

10. 虚拟设备与 SPOOLing
   - 虚拟设备把独占设备改造成可共享的逻辑设备。
   - SPOOLing 使用磁盘输入井、输出井和守护进程模拟脱机输入输出。
   - 例子：多个进程同时“打印”到磁盘队列，由打印守护进程按顺序输出到打印机。

11. 稳定存储器
   - 通过冗余副本、校验和、写入协议等技术，使数据在单点故障后仍能恢复到一致状态。
   - 常用于事务日志、文件系统元数据和关键系统信息。

### 考点整理

- 会比较程序查询、中断、DMA、通道四种 I/O 方式。
- 会计算磁盘 I/O 时间，尤其是平均旋转延迟和传输时间。
- 会根据请求队列模拟 FCFS、SSTF、SCAN、C-SCAN、LOOK 的磁头移动。
- 会说明缓冲的目的和单缓冲、双缓冲、循环缓冲区别。
- 会解释 SPOOLing 如何把独占设备虚拟成共享设备。
- 会比较 RAID 0、1、4、5 的性能、冗余和空间利用率。

### 考点答案与范例

1. I/O 方式比较答法
   - 程序查询让 CPU 忙等，适合简单低速设备。
   - 中断驱动使 CPU 与设备并行，适合字符设备和中低速 I/O。
   - DMA 按块传输，适合磁盘等高速设备。
   - 通道能独立执行 I/O 程序，适合大型复杂 I/O 系统。

2. 磁盘访问时间公式
   - 总时间 = 寻道时间 + 旋转延迟 + 传输时间。
   - 平均旋转延迟通常为磁盘转一圈时间的一半。
   - 例子：7200 RPM 磁盘每分钟 7200 转，每转 8.33ms，平均旋转延迟约 4.17ms。

3. 磁盘调度抓手
   - FCFS：按给定队列原样走。
   - SSTF：每步选离当前位置最近的柱面。
   - SCAN：先确定移动方向，一直服务到该方向最后请求再反向。
   - C-SCAN：只在一个方向服务，到末端后跳回另一端继续。
   - LOOK：不必走到物理端点，只走到该方向最远请求。

4. 缓冲技术比较

| 技术 | 缓冲区数 | 特点 |
| --- | --- | --- |
| 单缓冲 | 1 | 简单，改善速度不匹配有限 |
| 双缓冲 | 2 | 设备传输和进程处理可并行 |
| 循环缓冲 | 多个 | 适合连续输入输出流 |
| 缓冲池 | 多个共享 | 动态利用率高，管理复杂 |

5. RAID 比较

| 级别 | 冗余 | 性能 | 典型特点 |
| --- | --- | --- | --- |
| RAID 0 | 无 | 读写高 | 任一盘坏数据丢失 |
| RAID 1 | 镜像 | 读好、写一般 | 可靠，空间成本高 |
| RAID 4 | 专用校验盘 | 读好、写有瓶颈 | 校验盘热点 |
| RAID 5 | 分布校验 | 综合较好 | 常用，能容忍单盘故障 |

### 常见题型与答题抓手

- “磁盘调度题”：画数轴，逐步记录磁头位置和移动距离。
- “DMA 为什么快”：数据块传输不需 CPU 逐字节搬运。
- “SPOOLing 本质”：用磁盘缓冲和守护进程把独占设备逻辑共享。
- “RAID 选择题”：RAID 0 重性能无可靠性，RAID 1 重可靠性，RAID 5 折中。
