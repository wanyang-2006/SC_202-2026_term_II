# 第8章 文件系统

> 整理依据：`课件/操作系统原理课件 2026/08第八章 文件系统1.pptx`。

## 章节定位

本章讨论文件如何抽象、组织、查找、共享、保护和落盘。重点掌握文件逻辑结构、物理组织、目录、FCB/inode、文件共享、保护、安全、外存空间管理、UNIX 文件系统调用、日志结构文件系统、内存映射文件和 VFS。考试常考文件组织方式比较、目录查找、链接共享、空闲块管理和 UNIX inode。

## 核心知识点

1. ==文件概念==
   - 文件是具有符号名的一组相关信息集合，是操作系统管理外存信息的基本单位。
   - 文件属性包括文件名、类型、长度、位置、所有者、权限、时间戳等。
   - UNIX 常把普通文件、目录文件、设备文件等统一为文件。

2. 文件访问方式
   - 顺序访问：按记录或字节顺序读写，适合文本、日志。
   - 直接访问：按相对块号访问，适合数据库和随机访问文件。
   - 索引访问：通过索引表或关键字查找记录。

3. 文件逻辑组织
   - 无结构字节流：如 UNIX 普通文件。
   - 记录式文件：由定长或变长记录组成。
   - 关键字文件：按关键字组织和查找。

4. 文件物理组织

| 组织方式 | 思想 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 连续结构 | 文件占连续磁盘块 | 顺序和随机访问快 | 外部碎片，扩展困难 |
| 链接结构 | 每块指向下一块 | 无外部碎片，易扩展 | 随机访问慢，指针损坏影响大 |
| 索引结构 | 索引块保存数据块地址 | 支持随机访问和动态扩展 | 索引块有空间开销 |
| Hash 结构 | 按关键字散列定位 | 查找快 | 冲突处理复杂 |

5. 文件目录与 FCB
   - FCB 文件控制块记录文件元数据和物理位置信息。
   - 目录是文件名到 FCB 或 inode 的映射。
   - 单级目录简单但命名冲突严重。
   - 两级目录为每个用户设置目录，减少冲突。
   - 树形目录支持层次路径，是现代系统常用结构。
   - 目录查找可用缓存、散列和当前目录加速。

6. UNIX inode
   - inode 保存文件类型、权限、属主、大小、时间、数据块地址等。
   - 目录项通常保存文件名和 inode 号。
   - UNIX 常采用直接索引、一次间接索引、二次间接索引、三次间接索引组合，兼顾小文件效率和大文件容量。

7. 文件共享
   - 硬链接：多个目录项指向同一 inode，共享同一文件内容。
   - 符号链接：保存目标路径的特殊文件，可跨文件系统。
   - 引用计数用于判断 inode 是否仍被目录项引用。

8. 文件保护、保密与安全
   - 存取控制矩阵：行表示主体，列表示客体，元素表示权限。
   - 访问控制表 ACL：每个文件列出哪些用户拥有哪些权限。
   - 能力表 Capability：每个用户或进程列出可访问对象和权限。
   - UNIX 权限：属主、同组用户、其他用户分别有读 r、写 w、执行 x 权限。
   - 保密强调防止未授权读取，可使用加密。
   - 安全强调防止故障和破坏，可使用备份、日志、校验和。

9. 文件系统实现与外存空间管理
   - 位示图：每位表示一个磁盘块是否空闲，查找连续空闲块方便。
   - 空闲块链：把空闲块链接起来，空间开销小但查找连续块困难。
   - 成组链接：UNIX 常用，把空闲块地址分组保存，兼顾链式和批量管理。
   - 超级块记录文件系统总体信息，如块大小、空闲块、空闲 inode 等。

10. 文件系统界面
   - creat/open/close：创建、打开、关闭文件。
   - read/write：读写文件。
   - seek：改变当前文件读写位置。
   - link/unlink：建立和删除目录项链接。
   - mount/umount：安装和卸载文件系统。
   - chdir/chown/chmod/stat：改变目录、属主、权限和读取状态。

11. 日志结构文件系统与内存映射文件
   - 日志结构文件系统把修改顺序写入日志，提高写性能并便于恢复。
   - 内存映射文件把文件映射到进程地址空间，可像访问内存一样访问文件。
   - VFS 虚拟文件系统为不同具体文件系统提供统一接口。

## 考点答案与范例

1. inode 与目录项关系
   - 目录项把文件名映射到 inode 号。
   - inode 保存文件元数据和数据块地址。
   - 因此同一个 inode 可对应多个文件名，这就是硬链接。

2. 硬链接与符号链接比较

| 项目 | 硬链接 | 符号链接 |
| --- | --- | --- |
| 本质 | 多个目录项指向同一 inode | 一个保存路径名的特殊文件 |
| 是否跨文件系统 | 通常不能 | 可以 |
| 目标删除后 | 只要链接计数不为 0 内容仍在 | 可能变成悬空链接 |
| 目录链接 | 通常限制 | 可以指向目录 |

3. 文件物理组织例子
   - 连续结构适合只读大文件，如顺序播放的视频。
   - 链接结构适合频繁扩展但主要顺序访问的文件。
   - 索引结构适合需要随机访问的文件，如可执行文件、数据库文件。

4. UNIX 权限例子
   - rwxr-x--- 表示属主可读写执行，同组用户可读执行，其他用户无权限。
   - 对目录而言，x 表示可进入目录，r 表示可列目录项，w 表示可创建或删除目录项。

5. open 与 read 的关系
   - open 根据路径查目录，建立打开文件表项，返回文件描述符 fd。
   - read(fd, buf, count) 根据 fd 找到打开文件表项和当前偏移，再读取数据并推进偏移。

## 常见题型与答题抓手

- “目录项和 inode 各存什么”：目录项存名字到 inode，inode 存属性和块地址。
- “删除文件发生什么”：unlink 删除目录项并减少链接计数，计数为 0 且无人打开时才释放数据块。
- “随机访问效率”：连续和索引较好，链式较差。
- “权限题”：先分属主、同组、其他，再解释 r/w/x。
